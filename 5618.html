On 24 Jul 2002 02:28:25 +0200, Wolfgang Pfeiffer wrote:
>
> I'm running FVWM 2.5.2 on a RedHat 6.2. with fvwm-themes-base-0.6.0-1
>
> When starting fvwm via fvmw-themes
> (entry in .xintrc:
> exec /usr/bin/fvwm-themes-start
> ) I see this error message on the console:
>
> "[FVWM-Themes]: Starting FVWM under the main fvwm-themes session
> FvwmTheme is obsolete, see the Colorset section of the fvwm(1) man
> page [FVWM][executeModule]: <<ERROR>> No such module 'FvwmNetHints' in
> ModulePath '/u sr/libexec/fvwm/2.5.2'"
>
> 1:
> How do I get rid of this error?

The second warning should only happen if you switched on the old-style
EWMH support (replaced in 2.5.x). It is disabled by default.

> 2:
> Is there a chance that there will be a new RPM for
> fvwm-themes-base to get rid of the error?

No, sorry, the first FvwmTheme warning is not going to disappear.

fvwm-themes-0.6.0 normally runs without a single warning with both
fvwm-2.4.8 and fvwm-2.5.2, this is the only warning that you get on
the 2.5.x startup. There is nothing you can do about it, except for:

  1) Ignore this warning (preferable)
  2) Don't run alpha/beta fvwm versions
  3) Patch FvwmTheme not to issue this warning
  4) Wait for 0.6.1 where FvwmTheme will be started only with 2.4.x

Regards,
Mikhael.
