I've recently compiled fvwm2 and really like the package. I'm getting
ready to roll this out to the user community and ran into a problem
getting the system-wide defaults to be picked up.
In the man page it states,
"First, fvwm will search for a file named .fvwm2rc (or .fvwmrc based
on how it was compiled - .fvwm2rc is the default) in the users home
directory. Failing that, it will look for /usr/lib/X11/fvwm/.fvwm2rc
for system-wide defaults. If that file is not found, fvwm will be
basically useless."
So I copied a generic .fvwm2rc file to /usr/lib/X11/fvwm/.fvwm2rc. I
moved my working .fvwm2rc out of the way and then reinitiated fvwm2.
The restart did not pick up the system-wide defaults as stated. Is
there some work around for this. It's really important that there be
a system default file, but as it stands right now it doesn't seem to
work.
