[...]
| Otherwise, to get interactive, you use a MODULE... like FvwmTalk,
| or FvwmCommand, or whatnot.

 There are some things in fvwm2 that cannot be controlled through a
module. For example, if you want to modify how fvwm2 places windows,
you currently have to write/modify C code.

 I am not sure if it is desirable to make fvwm2 larger and more
complex so that people can avoid modifying C code. Supporting all
possible features and modifications just gets one a large muddy
mess, and so at some point it is better to say 'this is what it
does, and this is how it does it, and if you don't like that here's
the source (or there's other window managers)'.
