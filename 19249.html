i've got a couple question for things i'd like to do in my .fvwm2rc

1) i'd like be able to bind actions of my mouse to buttons on a
   specific window or to a specific icon. e.g. when i right click
   on the netscape mail icon, i'd like to get a special menu with
   Get New Mail

2) i guess being able to do 1) would let me do this, but...
   another thing i'd like to be able to do is have an icon (the
   netscape mail one) be sticky, but the opened window not be.
   that is, i'd like to see the icon on every "page" but when i
   open the window, i don't want that following me around.

4) i think this is more of a request than a question, becuase
   i;m pretty sure fvwm2 doesn't currently support this.
   it'd be really nice if i could have setting so if i start a
   process on a certain "page" then every other window spawned by
   that process would appear on that "page" without causing me
   to switch. e.g. the Dotfile Generator pops up heaps of
   different windows. i'd like to be able to start an action and
   go to another window to do work while the Dotfile program is
   working--it's a bit slow--without having all the different
   window is spawns popping up on me. (i know you can set windows
   to appear on a specific desktop and not jump there, but i'd
   like a general setting on a process level so i don't need to
   know the names of every window the application might spawn.

thanks in advance


--mark
