Drew Lebauer <asl_at_evolving.com> writes:
> I just pulled down and successfully built (making both static and shared
> libraries) xpm 3.4k. It is installed in /u/asl/opt/usr/openwin (/bin include
> include/X11 include/X11/X11 and /lib). When I run configure it can't find the
> xpm stuff. It says:
>
> > Have XPM support? no: Xpm library or header not found!
> >
> I'm on an ultra-sparc running SunOS 5.6 (== solaris 2.6). This is a box where
> I
> cannot become root or install anywhere except under my home directory (/u/asl
> ).
>
> I've tried the following command line:
> ./configure --with-xpm-library=/u/asl/opt/usr/openwin/lib \
> --with-xpm-includes=/u/asl/opt/usr/openwin/include/X11 --prefix=/u/asl
> --enable-extras
>
> What can I do to make it work?

You would have to normally show us the errors.

Leave the X11 part off of the include path.

You're going to need "-R/u/asl/opt/usr/openwin/lib" in your link editor
flags. The whole process is explained in the INSTALL.fvwm file.
