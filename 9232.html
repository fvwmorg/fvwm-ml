On 13 Jun 2003 15:05:52 -0600, Dameron, Gregg wrote:
>
> > I'm running fvwm-2.4.16 on Solaris 8 and am running into the
> > following problem:
> >
> > There is one app that opens some new windows outside the physical
> > screen area. I can't get to the title bar to move it into view. I've
> > turned off multi screen mode and edge scroll mode and have set my
> > desktop to 1x1 like so:
> >
> > EdgeScroll 0 0
> > DeskTopSize 1x1
> >
> > Can I force apps to always open windows on the physical desktop and
> > if so, how?
>
> We have some Java apps that tend to do that. FAQ 3.18 (FvwmEvent) works
> great for us.

Try also press Alt-Tab to bring problematic windows to the screen.
If a window is WindowListSkip, it will not be shown on Alt-Tab, then do
"WindowList OnlyListSkip" (you may bind this to Alt-Shift-Tab like in
fvwm-themes).

Try also:

  Style MyBadWindow NoPPosition, NoUSPosition

Search for "Position" in the fvwm man page.
