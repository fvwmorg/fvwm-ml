patelj_at_mindspring.com writes:
> I am trying to install FVWM 2.4.3 onto a local area. I am a user and don't ha
> ve priviledges to put files in the main areas. I installed FVWM and when I ru
> n it, this pops up:
>
> [FVWM][Read]: <<ERROR>> file '/usr/local/share/fvwm/ConfigFvwmDefaults' not f
> ound in /export/home/jpatel/.fvwm or /usr/local/share/fvwm
>
> Any idea on how to fix it?

When you run the configure step, use:

./configure --prefix=/export/home/jpatel/fvwm

(or any other location you have write access to.)

If you are using a prebuilt binary, its going to try to read
ConfigFvwmDefaults from one of the 2 locations shown.
