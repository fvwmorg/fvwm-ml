It looks like rox tries to put itself in the background when run
from a terminal. It has this option:

-n, --new

    start a new filer, even if one already seems to be running. This
    also prevents the filer from forking (running in the
    background). This option is mainly useful for debugging.

You might want to use "-n" to start the first one, or make sure there aren't
any instances currently hanging that might confuse it.
