First, I would like to join the chorus of thanks to all those who
spend time on the wm. It is really great, we all love it here.

The problem shows itself when I first log in when bringing up the first
new window or two from my .xsession file. These windows sometimes do not
have any of my styles; the color is some default light grey, and I have
no buttons in my menu bar. All other windows (most of which are opened
by fvwm in some manor) are fine. Doing a restart fixes things, but
I would like this not to happen. FYI fvwm is one of the first things
started in my .xsession, and it is in the background. I use another
gizmo for the last command in my .xsession.

I am running fvwm2 pl37 under several OS's. I see it from
time to time under HP-UX 9.05, but it happens frequently
under Solaris 2.4. I built it with gcc, and
I use the FvwmCpp module to preprocess my .fvwm2rc file.

As long as I am on the subject, my $.02 about the
config file is that I would not like to see fvwm depend on a
copy dropped in /tmp for runtime. We run tmp cleaners
that wack old stuff, so if I am logged in for several days
this file goes away. (I just seem to remember some rumor floating
about. Donno, maybe I am nuts.)
