jgaither_at_aur.alcatel.com (Justin L. Gaither) writes:
>
>
> I noticed a small bug in FVWM 2.0.42 if i enable num lock. I can no
> longer use the programmed F- keys, highlight in an rxvt terminal, or
> use my middle button to close windows. I am running solaris on an
> UltraSparc with type 5 keyboard.
>

Its not a bug, its a feature.

Num_lock sets the Mod 3 bit (something like caps_lock).

The first line below brings up the "systems" menu for mouse button 1,
regardless of the num_lock setting. The second line brings up the
Window-Ops menu only when num_lock is on.

Mouse 1 R A Menu "Systems" Nop
Mouse 1 R 3 Menu Window-Ops Nop

Arrange your config file accordingly.

If for some reason, your results vary, check out your .xmodmap file,
if you use one. Failing that, get hold of xkeycaps to see what
num_lock is set to do.
