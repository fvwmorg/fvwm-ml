I've noticed a strange behavior (which seems new to 2.0.45) with my
menus since upgrading.

I invoke my "main" menu for calling applications via a keystroke, and
like to poke around the menu using the arrow keys or start apps via a
single letter key stroke. But "sometimes" (I know that isn't much help,
and I've tried to narrow this down to a specific event, but have been
unable to thus far) pressing the key invokes the menu, but then it seems
the keyboard is "disjoint" from the menu. By disjoint I mean: no keys
have any effect on the menu. I can not scroll down the menu, nor can
any of my letter keys (corresponding to an application name) start any
apps, nor does the ESC key get rid of the menu. . . the only thing that
can operate on the menu is the mouse.

If I can be of any assistance in narrowing down the possible causes of
this behavior please let me know.
