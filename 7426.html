BTW, if I configure fvwm 2.5.5. using

./configure --with-gnome=no --disable-ewmh --disable-gnome-hints

then the problem is gone. Using just '--with-gnome=no'
was not sufficient to make the problem go away, so I would
expect that there is something wrong with the WMH communication
between fvwm2 and Mozilla.
