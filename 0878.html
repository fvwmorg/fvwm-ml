Doug Wandell <dwandell_at_UU.NET> writes:
> I keep a newer version of fvwm in my home directory at work (company
> version is like 2.0.42 or something). When I restart fvwm from a popup
> menu, it restarts the globally installed (older) version. Does fvwm
> have a variable to select an fvwm executable to run when restarting, or
> do I need to modify my search path, or what?

Just plain "Restart" executes the first "fvwm2" in your path.
2.3.x versions of fvwm let you restart using the full path name
of an fvwm2 version.

Look up "Restart" in your man page.
