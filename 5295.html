So you have the following message at the end of the configure
script:

With Xft aa font support? no: Failed to detect freetype2 ...

may be you can help the config script with the --with-ft-prefix=PFX
and --with-ft-exec-prefix=PFX options. Do you have other messages
during configure about FreeType 2? As I used only a 2.0.x FreeType
version there is may be a pb in the configure script. I will check.
 
Any way, the aa support used the Xft library which is specific to
XFree-4.x. I do not know if MAC OS X X server has such a library.
However, recently Keith Packard has released independent package for
Xft2, fontconfig and XRender (http://keithp.com/fonts).
Maybe these packages can be used with MAC OS X (I am bit sceptical,
but I know nothing about the internal of these libs). You may
ask such a question to the render mailing list: render_at_XFree86.Org
(search Xft2 with xfree86.org as domain in the Google advanced
search engine for the archive).

Regards, Olivier
