On Aug 27, 1997 - Broc Stirton spammed my mailbox.
Those 803 random chars looked somewhat like this..

> Steve-
> This may not work perfectly, but what I do is use the geometry option on
> the command line or in the fvwm config (menu, buttons, whatever). E.g.
> 'rxvt -geometry XxY+1032+7' gives me an rxvt on my second Screen (my
> resolution is 1024x768). This should have a similar (identical?) effect
> to a "StartsOnScreen" option. Remember, with a virtual screen, it is
> just treated as a huge screen (2048x1536 in my case: 2x2) of which you
> see only part at a time.
Not really..
If you have 2x2, and you are on the first (top,left) screen and start with
+1032+7, it will come to screen 2 (top,right). This is fine.
If you are on screen 2, and start with +1032+7, it'll end up right
_outside_ screen 2. The +X+Y is _relative_ from where you are now, not
from the top right corner.. (Haven't looked to deeply in the fvwm2 code,
is it X or fvwm[2] behaviour?)
