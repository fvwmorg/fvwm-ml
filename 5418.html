On 03 Jul 2002 15:47:14 +1000, Shao Zhang wrote:
>
> I am trying to get fvwm to display chinese characters on the windows
> title bar.
>
> I installed the latest alpha version 2.5.2, and changed my .fvwm2rc
> config file to have the following line:
>
> Style "*" Font StringEncoding=gb2312-1980-0:"-isas-song ti-medium-r-normal--0-0-72-72-c-0-gb2312.1980-0"

It should be "StringEncoding=gb2312.1980-0". A dot, not a dash.
It is critical to specify it correctly.

> Also tried without the StringEncoding bit.

Yes, please don't specify StringEncoding, in this case gb2312.1980-0 is
assumed automatically from a font charset.

> When I use mozilla to go to a Chinese website, only chars like "?????
> ??? ???" are shown on the title bar.

I don't have this font, but I have another similar one, so I do:

  Style * Font '-isas-song ti-medium-r-normal--16-160-72-72-c-160-gb2312.1980-0'

Then I open mozilla or opera on some Chinese page, say:

  http://www.randburg.com/is/nas_ch.html

choose GB2312 (Simplified) from a View / Encoding menu and see Chinese.
Note, for window title to be refreshed after changing Encoding from both
browser menus, you may need to switch to another tab and return back.
