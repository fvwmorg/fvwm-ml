> On Fri, Jan 09, 2004 at 05:38:34PM +0200, Micha Feigin wrote:
> > I have a strange behavior in mozilla-firebird. If I change the title
> > theme to noTitle before I load a webpage then mozilla-firebird stops
> > responding to keyboard input. If I first load a webpage (any webpage)
> > and only then change the theme to noTitle then everything is fine.
> > Any idea to whats causing this?
> > (I tried doing it both from the keyboard short cut and a menu entry)
>
> Mozilla stops responding to my keyboard input sometimes. It seems
> to happen to me randomly, I can't reproduce it with your instructions..
>
> To get keyboard input back, I just click Edit -> Find in this page...
> in the mozilla menu bar, and type something. after that keyboard input
> is restored...
>
> Can you try to send more detailed instructions, like the actual fvwm
> commands you used to to reproduce this.?
>

I use the following function:

DestroyFunc title-bar-toggle
AddToFunc title-bar-toggle
+ "I" ThisWindow (State 11) WindowStyle Title
+ "I" TestRc (!Match) WindowStyle !Title
+ "I" State 11 toggle

Assigned to a kb shortcut A-C-Ins and to a menu item to remove the
title from a window (maximize screen usage, small laptop 14" lcd).

When used to remove the title from mozilla-firebird when it first comes
up (just started, including quit and restart) then the keyboard stops
responding. Restoring the title bar restores the keyboard.
I set it up to start by default with a blank page, no home page. If
after starting I load a page and only then remove the title bar then
everything works fine.
fvwm version is 2.5.9 mozilla-firebird version 1.5 system is debian
unstable.
