>>>>> Leo Weijs writes:

  Leo> So, it seems there is a problem opening libXpm, the library is in
  Leo> /usr/openwin/lib.
  Leo> My fvwm2 is in /usr/local/bin.

(1) Start fvwm with the right LD_LIBRARY_PATH, like so:
    env LD_LIBRARY_PATH=/usr/openwin/lib:/usr/local/lib fvwm2 &
    Of course, you might need to change your LD_LIBRARY_PATH. The
    output of "ldd /usr/local/bin/fvwm2" might give you a clue.

(2) Use the -R option to ld so that the library search path is
    recorded in the executable. This needs to be done in the final
    linking step to produce the fvwm executable.
