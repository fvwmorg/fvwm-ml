> > I have a feeling that fvwm cannot help me select the font for my xterms.
> > Am I correct?
>
> Correct, for all applications. In general the window manager does the
> following things:
>
> 1. Draw borders around windows.
> 2. Allow resizing and movement of windows.
> 3. Allow iconifying of windows.
> 4. Provide other services, such as menus, button bars, virtual desks, etc.
>
> The window manager doesn't affect what happens *inside* the window. That
> is up to the application itself.
>
> > That is, the only way to specify what fonts an xterm will use is to
> > give the xterm/remoteXterm command the right argument.
>
> Or set the X resource. I have the following in .Xdefaults:
>
> XTerm*font: -b&h-lucidatypewriter-medium-r-normal-sans-12-120-*-*-m-*-iso8859-1
>
>
> This is read when X starts up, or can be reloaded by:
>
> xrdb -load .Xdefaults
