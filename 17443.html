>>>>> "BFJ" == Bruce F Juntti <bjuntti_at_uswnvg.com> writes:

    BFJ> Hi all,
    BFJ> When using fvwm2 as my window manager on my Linux box,
    BFJ> when invoking Seyon I get complaints from that app like:
    BFJ> "/dev/modem: no such device". Seyon works fine, however,
    BFJ> when using the fvwm that cam with my 1.2.1 Linux
    BFJ> distribution. Any ideas?

Have checked that /dev/modem exsists ? It should be a link to the
device file you are using for your modem, i.e.
/dev/modem --->/dev/cua0 . This has nothing to do with FVWM at all.
Seyon must default to look for a device file called /dev/modem.
Check and see where your modem is connected, such as use dmesg
and read the output and look for the serial ports.

Hope this helps,
