Further to the previous discussions about Themes for Fvwm2, I have written a
form to edit FvwmThemeColorset [options] on the 'fly'. It saves these options
to a ${FVWM_USERHOME}/.fvwm2_pallette file, in a comma separated format
as specified in the man page, using techniques from the examples in
${EXEC_PREFIX}/etc/fvwm - 2 options for colorsets numbered 1 to 8 can be
edited, and multiple colorset options are allowed within a 30 character limit.
e.g.
# This file last created by ColorsetOptions on: Thu Nov 18 13:15:42 CST 1999.
*FvwmThemeColorset 1 fg cyan3, bg rgb:b4/aa/94, hi rgb:a1/b2/c8, sh orange4
*FvwmThemeColorset 2 fg DarkGreen, bg grey60
<--snip-->
The default colors were randomly chosen (I don't use any kind of color-chart)
so you'll have to set them properly for yourself. There's no documentation for
it yet, not even embedded comments. (sorry) It seems to work for me, but as I've
only recently upgraded to fvwm-2.3.9, I still have a lot of Colorset options to
place into my now fairly extensive configuration to get the full benefits of
this new module. This means I also haven't fully 'bug-tested' it yet.
?Gradient, Pixmap and Shape options should work but need testing as I have yet
to implement them into my "themes". If anyone would care to 'try it out' and
let me know of 'bugs' or just 'what they think', I would appreciate it greatly.
This file is 90 lines or so long, so please email me (awol_at_dove.net.au)
off-list and I'll attach it to my reply. I only have a 'dial-in' account so
don't expect an instantaneous reply, but I do check my email several times a
day, so it shouldn't be too long a wait.
