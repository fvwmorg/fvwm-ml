B> I changed the builtin WindowList to use the icons, and added some stuff to
B> FvwmIconMan to use the miniicon to denote an iconified window instead of the
B> raised square thing. The FvwmIconMan code is rather rough proof-of-concept
B> type code (you can't turn off displaying icons mainly), but it seems to work.
B> Also, the version of FvwmIconMan that I hacked is the one that's due to come
B> out in the next release of fvwm, and is significantly different than the one
B> in 2.0.43. Read the CHANGES file and the manpage for info (nothing on
B> miniicons is mentioned though).

B> I didn't do anything to FvwmWinList, since somebody else indicated that they
B> were willing to do that.

I've reworked WindowList. (not FvwmWinList).
I've included your compile-time option into my patches, which I'll mail to
chuck, as soon as they're ready (tiny bug left, and it's late)
(well I wanted to check out your patch :-).

just to make your mouth watery:
I ripped out the [...] handling code from Circulate/Prev/Next/None and
used it for the new windowlist parameter handling.
Then I enhanced it, since this effort now paid at five different locations
:)

Windowlist [!sticky !OnTop maximized CirculateSkip]
Windowlist [.iconic .sticky Xterm CurrentDesk]
