I had this same problem compiling 2.0.42 on HPUX 10.01. The problem seemed to be
the '-Aa' option for gcc. It has been a while since I compiled fvwm, but I think
if you change your compiler in Fvwm.tmpl to cc, it should take care of the
problem. If it doesn't, there is probably a way to tell the Imakefile or Fvwm.tmpl
not to use the -Aa option.
