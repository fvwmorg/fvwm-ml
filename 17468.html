| [FVWM][Done]: <<ERROR>> Call of 'fvwm
| ' failed!!!!

That's a bug in either the FvwmButtons module or the code in fvwm2 that
parses input for the Restart directive; I've reported it to the list
previously, but gotten no acknowledgement. The problem is that the
configuration line that gets sent to the module ("*FvwmButtons [...]")
has a newline on the end; FvwmButtons doesn't strip the newline, and
when it sends the Restart command back to fvwm2, the newline's still on
the end. Unfortunately, fvwm2 also doesn't seem to remove the trailing
whitespace, so it tries to exec "fvwm\n" (which causes the error).

A brute force workaround is to insert a null character at the end of
the line in your .fvwm2rc file, but it's probably not worth the trouble.

(Is this bug fixed for 2.1?)
