Dominik Vogt writes:
> As far as I know CDE even adapts the actual colours it uses to your
> display hardware. That is why CDE's colours look the same
> regardless of the machine you have. This is far beyond everything
> we tried to code yet.

AFAIK the matching of device independent colours to the display
hardware is done in Xcms rather than CDE. That is in the core X11
library and so is used by FVWM already. Of course it needs to have a
file set up for the particular video card and monitor being used.


If you really want to get the same colours as CDE, link in the Motif
library and call XmGetColors(). That returns foreground, select, and
shadow colours for a given background. In my experience the
underlying algorithm is different in each of Motif 1.1, 1.2 (CDE 1.0),
and 2.1 (CDE 2.1).
