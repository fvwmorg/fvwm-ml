> > # Button 2 with Meta (diamond), moves window: dje
> > Mouse 2 W M Move

> Cool. Works like a charm.

Small problem with this recipe.

In a xterm, when I do this with my mouse anywhere in the window, the move
works fine, but when I release the mouse buttons (1+3 since I am using a 2
button mouse to simulate Button 2), the current primary selection gets
pasted into the xterm!

I understand that it is happening because Mouse 2 in a xterm is mapped to
paste the current selection, but this is certainly a unexpected side
effect and could be dangerous if I am running a curses-based application
that interprets the pasted characters as commands!

Comments?
