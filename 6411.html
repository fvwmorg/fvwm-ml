On 04 Oct 2002 18:32:11 -0300, Christian Lyra wrote:
>
> I just got another videocard+monitor and configure a dual-head Xfree (not
> xinerama as I want to run two distinct resolutions). From fvwm2 manual I saw
> that I can call fvwm2 with parameter -d <display>. but fvwm seems to be not
> respecting this! I tried to run icewm together with fvwm2 (one on each screen
> of course), but fvwm2 insist in running on both displays!
>
> I tried the following:
>
> 1 - start X with only a xterm in .xsession, so I got no WM
> 2 - from the xterm launch icewm with: icewm -display :0.1 (ok... it works)
> 3 - launch fvwm2 with: fvwm2 -d :0.0 (them fvwm just launched on both
> displays!)
>
> I'm running fvwm2 snapshot from last week (cant remember the day...).
>
> This is a bug or am I doing something wrong?

Try "fvwm -s -d :0.0" instead. The -s means manage only one screen.
