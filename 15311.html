Kim> Is it possible to bind a key to mouse buttons? I know it is easy
Kim> to bind e.g. Ctrl-Meta-Left to move the mouse pointer to left,
Kim> but I would like to have a key combination that would press a
Kim> mouse button too! It would be really nice in Netscape, cut&paste
Kim> in xterms etc. when mouse isn't in reach of the hand.. Maybe it
Kim> is possible even now, I am not certain, but if it isn't, please
Kim> add it!

We've discussed this here before, but I guess I should put this in the
FAQ as well.

Making fvwm generate ButtonPress events is ok in theory, but a lot of
applications ignore events marked SYNTHETIC, which these would be. If
I get a chance, I'll experiment with it, but I can't promise anything
(it is on the TO-DO list, BTW).

I hate making all these negative responses, so I'd better redeem
myself with a positive one: If you'd like to do cut-n-paste via
keystrokes though, you can do that now via xcb. Grab it from
ftp.x.org or your favorite mirror.
