Instead of making fvwm able to have a bazillion of user interface,
would it be possible to have modules that handle windows decoration
and menus ? If the inter-modules communication is not too slow, it
would allow to have a win95 module, a macOS module, a warp module or
whatever, and keeping fvwm core small. The current window decorations
and menus could be kept as default, but possibly overridden by some
module routines.

If it is feasible in terms of speed, I'd rather see that than a lot of
new functions.
