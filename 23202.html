On 17 Dec, Fabien Villard wrote:

>>I've just installed fvwm 2.1.5 on solaris and all my pixmaps disappiared
> I have the same problem.
> And also, the line in .fvwmrc :
>
> PixmapPath path1:path2
>
> is not recognized anymore. The error message is : Module PixmapPath not
> found in path <module path>.

I've had this happen to me, also.

I found out the cause... for some reason, the XPM library isn't found
unless you use the --with-xpm-library= and --with-xpm-includes=
parameters on the configure script (that is, if the XPM library isn't
in the standard include location).

For example, if your XPM library and includes are in /usr/local, then
it won't be found unless you specify their locations explicitly.

I would guess that the logic to find XPM could be made a bit robust...
