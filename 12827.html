On Fri, Jul 16, 2004 at 04:09:35PM -0700, Moore, Garron wrote:
> What you have explained below is not an option for us. We can't assume the location that they are putting it. I can't even assume that they have root access or write permissions in /usr/local or /opt. Given this, do you have any recommendations? I would be willing to create a script that goes into any file where the path is stored and replaces it.
>
> I realize that what I'm asking is somewhat unique. I would really appreciate any advice that you could give that would help me accomplish what I need to.
>

I haven't seen the sources, but if fvwm uses hardcoded directories then
the end users will either need to have root access to install it or have
a compiler so that they can recompile it for their system requirements,
or like someone else suggested, let them install wherever they want and
make temporary links from /tmp each time you want to run it (may be a
problem though letting multiple users run fvwm at the same time).
