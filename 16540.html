>I have the lastest fvwm on my DEC Alpha running Digital UNIX (OSF/1).
>I am also running tkgoodstuff w/ fvwm. In this tkgoodstuff I am running

Hmm. Since tkgoodstuff can't "swallow" the FvwmPager, perhaps you are
using tkgoodstuff's Pager client? If the problem is with tkgoodstuff,
which version of tkgoodstuff are you using?

>The problem is I cannot use the mouse to switch virtual
>screens i can use like control-next, control-prev to advance it but, the
>mouse doesn't switch the screens. I tried using FvwmConfig to fix this,
>and I added the line Read .fvwm-config in my .fvwm2rc file. FvwmConfig
>doesn't seem to save the virtual information in it's file. Any

I haven't heard of this problem with tkgoodstuff's pager; but I'd be
happy to help you debug it if you send me details. Which version of
tkgoodstuff are you using? Does the tkgoodstuff pager correctly track
which virtual screen you are on (the pager button corresponding to the
current page should become sunken) and which screens have windows on
them (the corresponding button should be blue). I.e., I'm wondering
if the tkgoodstuff/fvwm interface is working for you at all.
