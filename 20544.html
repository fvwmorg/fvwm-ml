How can I set up a window manager other than twm with XFree86
when X is started with xdm? (I prefer xdm to startx or xinit scripts
because of xsm's checkpoint feature).
Other window manager's ressource files (such as .fvwmrc) don't help
much as long as twm isn't replaced by this other window manager.
Since two weeks I'm looking for a solution - I really need some hints.

While the installation of XFree86 3.2 on a FreeBSD 2.2.2 system was easy,
the Doc files/ Readme's could however be improved in some respects
(I refer to QuickStart.doc from Aug. 26, 1996 by Joe Moss):

1.) Item "3.5 Ending text" is too short, as it only says that the server
could now be started by typing "xdm" or "startx" (I had already realized
that just typing "X" leaves me with a server without any xterm or other
windows - even worse, I didn't know that pressing Ctrl-Alt-Fx would
allow me to switch back to the text console windows...).
Some more hints about starting and customizing X would be greatly
appreciated! Links to other documents would last - at this time, I
still have great difficulties in setting up fvwm2 or afterstep as my
default window manager - just took .xinitrc from the man page and
replaced twm with fvwm2, not yet that satisfying).

2.) For some reason, X doesn't use csh's default environment. This is
to say, applications in /usr/local/bin are not found as they are not in
the path. Had to add a new "setenv PATH ..." line to /root/.cshrc.
The Backspace key didn't work in xterm with csh (difficult to type
longer command lines without making a mistake... - problem fixed
since I made tcsh my default shell).

3.) XF86Config and XF86Setup do not "know" 1152x864 resolutions
which are optimal for nowadays 17" monitors. After a look into the
86C864's config file for DOS/Win, I managed to set up this resolution.
As the desktop background is still displayed in dithered colours/ shades
of grey, I doubt if XF86_S3 really uses 8bpp (DefaultColorDepth is set
to this value).
BTW: had had some difficulties in setting up the screen blank
timeout correctly; it seems that changes to XF86Config do not
take into effect until next system reboot.
