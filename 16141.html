On Mon, 15 Apr 1996, Stephen Riehm wrote:

> Is there a logical reason why starting applications from the window
> manager seems to be different than starting them from the command
> line? Can someone explain it in a way which might lead to a solution
> for this problem?

The problem comes from a difference in the environnement seen by the fvwm
process and all child. Quite often we execute xterm with the -ls option,
forcing a read of .profile (or whatever your shell is using).

I have experienced difference between using "startx" on Linux for example
and starting using xdm. Ultimatly, some PATH may not be setuped properly
for fvwm.
