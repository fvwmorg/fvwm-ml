Yea!

>I downloaded scwm (touted as the successor to fvwm) but could not get it
>to do button bars or a pager. Then I inquired about Enlightenment but
>was told it was resource hungry compared to fvwm. Now I see conversation
>about Gnome and how to do gnome-like things with fvwm. What is Gnome and
>what are itst strenths & weaknesses?

I'm not sure about SCVWM except that it exists,

Enlightenment is a Fvwm Hack ( a very nice one ) that
allow custom window frames shaped icons etc etc etc, resource hungry
but very pretty, als some say that it can be hard to configure.
www.enlightenment.org

Gnome is not a window manager, it is a Desktop Suite, consisting of a
"taskbar", file manager
admin applications and productivity applications, ( they have ported fun
ones too :), it
provides a consistant look and feel between apps, session management,
applets. It has
no specific window manager base, Miquel uses olvwm, raster use
Enlightenment, I use fvwm with it.
www.gnome.org

>More generally is there a path forward from fvwm2 that incorporates the
>critical functions (button bars, pagers, menus) but does not chew up
>more resources than fvwm2? Or should I ignore the static and just stick
>with fvwm2?

Fvwm2 provides all of these, FvwmPager, FvwmButtons, and the menus are
configurable from the .fvwm2rc file..
