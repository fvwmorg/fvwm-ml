I'm using fvwm 2.0 pl 33 on Linux. All I have to report is a very
minor bug: whenever I run "xemacs -f vm" and go to an empty screen
before it comes up, the window appears at the upper left hand corner.
(I'm using SmartPlacement or whatever it's called.) In the middle of
the loading process, the window self-expands slightly in width. Then
after the program is finished loading, I can't scroll across the top
or left boundaries of the screen with the mouse pointer anywhere
inside the Xemacs window. This situation is also very temporary,
since any virtual desktop scrolling as well as many window manager
activities end it.

Also, I have a suggestion for an improvement: it would be nice if
there were an option to make SmartPlacement look through the whole
virtual desktop for free space for a window, so I wouldn't have to
switch to an empty screen before a program had time to start up.
