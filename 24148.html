Andrew Morton wrote:
>
> Bug reports on fvwm-snap-19990511:
>
> 1: I get "Some command line modules have not quit, Starting up after
> timeout.`" and a long hang at startup
>
It's not a bug, it's a feature. What's the command line you start fvwm
with? If it has any modules on it other than FvwmM4 or FvwmCpp then you
should be getting a pause. Modules other than FvwmM4 and FvwmCpp should
be started in the .fvwm2rc file (preferably in the InitFunction so you
can re-read the file to pick up any changes without starting a second
copy of the modules). See question 4.3 in the FAQ that comes with the
snapshot. If you aren't starting any modules with the fvwm command line
then it may be a bug.

Cheers,
Tim.
