I have started using fvwm2 (2.0.43) and was wondering if any of the
following features are possible:

1) Ability to resize a window during active placement, like other window
   managers (twm) can. Click (and hold) to place window, then drag window
   grid to desired size, and release.

2) Change a window's ClickToFocus/FocusFollowsMouse setting after the window
   has already been placed. Sometimes, I'll have an xterm that I open to
   tail a log file or something and I decide I don't want it autoraising.
   Setting it to ClickToFocus would alleviate that problem. A nice generic
   solution would be nice, but I would be happy with a solution specific to
   xterm if there is some way to do it by changing the window's title/icon
   name/class/etc. (i.e., have an "clickXterm" class defined in .fvwm2rc
   that is set to ClickToFocus and then somehow tell fvwm2 to use that class
   instead of "xterm".)

3) Override SmartPlacement/CleverPlacement/etc to ActivePlacement on-the-
   fly. For instance, I'd like to have a pop-up menu choice that says "xterm"
   (which does SmartPlacement) and a "Place xterm" option, that would have
   some option on that line to tell it to use ActivePlacement for the next
   window. This would be especially useful with my first question above,
   because then if you know you want a 100x60 xterm, you could easily do it
   all at once...

Also, I just got a second head for my Sparc, and when I start fvwm2, it makes
both screens look identical: it starts two button bars, two xloads, two
asclocks... even two FvwmAuto's (is that really necessary?!?!) I'd like to
have different configurations for each screen. Are there samples of how to do
this with m4? (I am already pre-processing my files with m4.)

Thanks for any suggestions,
Eddy
