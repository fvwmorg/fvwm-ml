I still can't reproduce the problem, it looks OK to me. Of course, I
can't really tell what you are looking for. I've included my complete
FvwmButtons config, and a "gif" file showing the resulting buttonbar.

I'm running this with the Solaris version of xclock:

*FvwmButtonsFore black
*FvwmButtonsBack gray
*FvwmButtonsFont -*-helvetica-medium-r-*-*-12-*
*FvwmButtonsGeometry -1-1
*FvwmButtonsRows 1
*FvwmButtons Start map.xpm Popup StartMenu
*FvwmButtons (2x1 Container())
*FvwmButtons Resize mini.resize.xpm Resize
*FvwmButtons Move mini.move.xpm Move
*FvwmButtons Lower mini.lower.xpm Lower
*FvwmButtons Raise mini.raise.xpm Raise
*FvwmButtons Delete mini.delete.xpm Delete
*FvwmButtons Destroy mini.destroy.xpm Destroy
*FvwmButtons "Stick" mini.stick.xpm Stick
*FvwmButtons Restart mini.excl.xpm ButtonRestart
*FvwmButtons (End)
*FvwmButtons(2x1, Swallow "FvwmPager" "FvwmPager 0 1")
*FvwmButtons(2x1, Size 64 64, Swallow "XSysStats" 'Exec xsysstats \
 -hidelabels -sample 10 -bg gray -fg black -type disk -color blue \
 -type packets -color darkgreen -type cpu -color red')
*FvwmButtons(Icon clock.xpm Padding 0 0 Swallow(Respawn) xclock \
  `Exec xclock -bg khaki4 -padding 0` )
*FvwmButtons(1x1, Padding 8 8, Container(Frame 1))
*FvwmButtons(Swallow "asclock" 'Exec asclock -shape -led green -noblink')
*FvwmButtons(End)
