Regarding Tuan Le's compilation problem on Solaris...

My guess is he didn't build the Makefiles on the Solaris box (that is,
he used the Makefiles from the Sunos build). That would explain the
missing `-lsocket -lnsl' from the link lines. The rest of the link
lines should be identical, especially if he uses gcc, so all the .c's
would compile properly. It does build "out of the box" for me on
Solaris, modulo an edit to Fvwm.tmpl I need to specify local paths.
