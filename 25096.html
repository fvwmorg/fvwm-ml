On Fri, Nov 19, 1999 at 12:16:44PM +0000, Aron Zeh wrote:
> I am running FVWM 2.2 on RedHat 6. Since I am relatively new to both, I am
> using RedHat's Another Level default configuration with only a few minor colour
> and font changes.
> This works fine usually, except that sometimes (I don't know what it is related
> to, unfortunately) the Buttons in the top window bar stop working. All the
> functions like resize, hide, etc. do however still work if invoked from the
> menu.
> Does any one know what is happening?

The usual reason for 'mouse/key bindings stop working' problems
is that you have pressed num-lock, caps-lock or any other of
the locking modifier keys. Pressing the offending key again
will solve the problem then. The 2.3.x beta releases have a
command 'IgnoreModifiers' that can prevent this from the start
(but which is very inefficient due to the way X handles key
and mouse bindings) as well as a FAQ entry explaining how to
disable those keys completely.

If this is not your problem, please have a close look at the
mouse pointer when you move it over one of the broken title
buttons. Usually you would get a small hand. If this is
not the case, what cursor do you get? The one you would get
ofer the title or perhaps one of the double arrows you
get with the mouse over the corners or border (which)?
In either case, please run the command

  # xwininfo -tree > xwininfo.out

click on the root window when asked for a target window.
Make sure you have a window with a unique name that has the
button problem and send me the name and the output of the
command. BTW, do the buttons on all windows stop working
at once or only for some or one window(s)?
