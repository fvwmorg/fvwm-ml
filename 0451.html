Previously from Andrey
>Eterm has a nice -x option, which causes it to have no borders, title or any
>other 'beautification' from the window manager. This option worked OK up until
>2.3.13 (which is the second to last of the devels I have compiled). I just
>compiled 2.3.16 and the option ceased to work.

Hmm, could you use FvwmIdent or xwininfo to determine if the window is
set Transient or Override Redirect when eterm runs with -x? I figure
that's likely to be the reason it can hint the window manager that it
doesn't want decorations.

>Also I tried using "Style <mytitle> NoHandles" shich did not work...
>"Style <mytitle> NoTitle" left the border intact, even though it removed the
>title (as it should)...

You could try setting "Style <mytitle> NakedTransient". If that
doesn't do it (which it wouldn't if the window isn't set transient),
you could try "Style <mytitle> NoTitle, NoHandles, BorderWidth 0".
