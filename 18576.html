Hi Thomas,

I believe I have found a bug in FvwmForm wrt selection colours (this
MAY be a general fvwm2 bug, so I have cc'ed this message to the fvwm
mail address..).

I use to following code to get a verify form when I quit fvwm2

Style QuitVerify NoTitle, NoHandles, BorderWidth 7
*QuitVerifyGrabServer
*QuitVerifyWarpPointer
*QuitVerifyFore Black
*QuitVerifyBack grey65
*QuitVerifyFont *helvetica*m*r*n*18*
*QuitVerifyButtonFont *helvetica*m*r*n*18*
*QuitVerifyItemFore Black
*QuitVerifyItemBack grey75
*QuitVerifyCommand Beep
*QuitVerifyLine center
*QuitVerifyText "Do you really want to logout?"
*QuitVerifyLine expand
*QuitVerifyButton quit "Yes" ^M
*QuitVerifyCommand Quit
*QuitVerifyButton quit "Cancel" ^[
*QuitVerifyCommand Nop

The colours work fine when running (fvwm2.0.42) on Solaris with 8 bit
displays. I run the same setup on my LinuxBox (S3 Trio64+ videocard)
and again it works correctly with 8 bit colour. But when I run at
16 bit (the usual) the popup form is only black and white. Quite
disappointing!

Any suggestions?
