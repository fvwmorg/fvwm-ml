patrick <patrick_at_mail.boxsoft.com> wrote:
>
> But my main concern is the Xpm lib. I believe I have a current version
> of the XPM lib. According to the ./docs/FAQ file the requirement is
> version 3.4f or better. Well this is what I show on my system:
>
> /usr/X11R6/lib/libXpm.a
> /usr/X11R6/lib/libXpm.so.4 -> libXpm.so.4.3
> /usr/X11R6/lib/libXpm.so.4.3
>
> Which shows that my system meets that requirement (unless the
> rules of math have changed recently: 4.3 > 3.4f).

        No, actually they changed a good while ago...where Xpm is
concerned. :-)

        Don't confuse the name of the library that the Xpm package builds
with the release nomenclature of the package itself - they're quite
different, and seemingly unreleated. For a reference point, xpm-3.4j
builds an xpm library called libXpm.so.4.10 (I believe that 3.4k is the
latest). So you need at least xpm-3.4f -> libXpm.so.4.6.

        Don't ask: I have no idea what historical accident caused this
discrepancy to come about. It's annoying and confusing, but there it is.
To compound the confusion, vendors/distributors often refer to the
package by the library version instead of the package version.
