I want to discover the "resource" and "class" of several windows,
in hopes that those from a particular application (which do not
share a common window or icon name) will have _something_ in common
that I can use to identify them.

It appears that the send_windowlist operation of FvwmCommand will tell
me what I need to know. However, I can't get FvwmCommand to work.

In FvwmConsole I typed "FvwmCommandS" and it created the FIFOs:

$ ls -l .FvwmCommand*
crw------- 1 phutchis phutchis 255, 255 Nov 16 19:03 .FvwmCommandC
crw------- 1 phutchis phutchis 255, 255 Nov 16 19:03 .FvwmCommandM

but FvwmCommand can't find them:

$ FvwmCommand -f .FvwmCommand send_windowlist
No such device FvwmCommand error in opening message fifo

The same thing happens with or without -f and/or -r.

The only clue I can find is that, when I type FvwmCommandS in
FvwmConsole, this gets appended to .xsession-errors:

    FvwmCommandS server error in opening command fifo, No such device

which doesn't make a lot of sense given that, as noted above, it
did in fact create the FIFOs. What am I doing wrong?
