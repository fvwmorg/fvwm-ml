> Neil,
> I think this is the second time you have asked this question. I
> don't think you have convinced anyone that there is a pressing need
> for this. In your original description of "follow-mode" you referred
> to the [X]Emacs "follow-mode" package. Since the functionality
> required is already in [X]Emacs, why aren't you using that?

Why? Because I want to do the same thing with vi and the various vi clones
and don't like the viper vi implementation that comes with Emacs that
much. Plus, with Emacs (I never used Xemacs but Emacs which I have used
can run in X mode or text mode) you can use follow-mode within a same
FVWM window but you can't use follow mode amongst separate windows.

You can have this:

+--------- Emacs ----------+
+--------------------------+
|1 |56 |
| | |
| | |
| | |
| | |
|55 |110 |
+--------------------------+

but you can't have this:

+---Emacs ---+ +---Emacs ---+
+------------+ +------------+
|1 | |56 |
| | | |
| | | |
| | | |
| | | |
|55 | |110 |
+------------+ +------------+

I will make another suggestion to the Emacs developers pertaining to this
feature then. Actually, here is where the problem comes in. In the latter
picture, both "Emacs" windos would have to have the same toolbar color.
If not, then scrolling down a highlighted window would move the cursor in
a nonhighlighted window. Really, this should be a window manager feature.

Even without follow-mode, having two windows display the same thing would
be nice. Just like 2 windows in Emacs without follow mode. Why should
everything appear in "one" rectangular box. Why not have things displayed
in more than just "one" rectangular box?

Neil
