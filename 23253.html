On Tue, Dec 22, 1998 at 09:19:22AM -0500, zbrown_at_lynx.neu.edu wrote:
> When I start X and FVWM, the pager displays a grid whose elements are the
> same size as the viewport. I would like this grid to represent things that
> are more meaningful for my desktop set-up, such as the distance the viewport
> jumps when I press certain hotkeys. Is the grid configurable?

Huh? We have a communications problem here. If the grid elements
were as large as the viewport the pager would cover the entire
scree. That is probably not what you want to say. So here is what
the pager (FvwmPager I assume) does:

For the specified range of desks it displays a mini-representation
of the configured pages. You can switch desks or pages via the pager
by clicking in it. The number of desks is given on the command line
for FvwmPager:

  Module FvwmPager 4 9

will give you six desks numbered from 4 to 9. Each desk has the
same grid size specified by

  DeskSize x y

Each grid rectangle represents a page, the dashed lines in between
are the page boundaries. If you want to change the size of the
grid you have to change the number of pages on your desktop.

Does this answer your question?
