Hello,
I noticed a smallish bug in fvwm:
When a new window with a private colormap is mapped (or there is a
colormap change on some toplevel window), I think that fvwm should
install that colormap and then come back to the current colormap, even
if the window doesn't have the focus.
This way, the window will have the right colors on multiple colormap
systems (the Indy here supports 4 colormaps and the HP has 2).
Mwm installs these colormaps properly, BTW.
