On Thu, 30 May 1996, Albrecht Kadlec wrote:

>
> well, HOSTDISPLAY is apparently set to `hostname`,
> which doesn't include the domain in our domain.
> (and I can't change that)
>
> Any way to tell fvwm the missing domain once and forever?

I had the same problem, way back with 1.2x on AIX. I fixed it by hacking
the source to put in the complete domain name using gethostbyname().

Note that setting and exporting HOSTDISPLAY in .xsession before launching
fvwm should work too, but it's not nearly as elegant and requires some
ugly code in .xsession to be general.

What are the chances of getting the official fvwm source to always use the
fully qualified hostname when setting HOSTDISPLAY? It should never hurt
anybody and will help those whose systems' hostname command returns the
short name instead of the fully qualified name.
