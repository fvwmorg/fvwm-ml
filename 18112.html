>>>>> "PK" == Peter Kabal <kabal_at_tsp.EE.McGill.CA> wrote:

PK> (2) As part of the standard installation, add a few extra Form
PK> modules (as links). For example, add FvwmForm1, FvwmForm2,
PK> FvwmForm3 as aliases for FvwmForm. This way, a user can set
PK> up different "forms" without having to have copies (or links)
PK> for the modules. For instance, I use one form for a
PK> "query/exit X" command, another for "rlogin to remote". My
PK> personal Modules directory contains only links to FvwmForm,
PK> with two different names. If the standard directory had a few
PK> extra generic links, I would not have to have maintain my own
PK> modules directory.

This not necessary since you can call FvwmForm with an arg. From the
man page:

INITIALIZATION
       FvwmForm uses the same configuration file (usually .fvwmrc
       in the user's home directory) as Fvwm. To use FvwmForm
       for multiple input forms, simply create symbolic links
       with different names, and they will be treated as differ­
       ent modules. Or, you can invoke FvwmForm with an optional
       parameter, which it will use as the name instead (e.g.
       'Module FvwmForm QuitVerify'). That way you don't even
       have to make a symbolic link for it!

Pretty nice, uh?
