Leo Deck writes:
> I realize that I am trying to do something that is probably out of
> the ordinary. Perhaps, it is not possible to do this. Any insight
> into the feasibility or possibility of doing something like this
> would be appreciated.

The program appres does part of what you ask for.

    appres App1

will print out the resource records that get loaded into the database
of app1.

Of course for this to work appres has to have the same environment as
app1. You'll need to take steps to set XENVIRONMENT,
XUSERFILESEARCHPATH, XAPPLRESDIR, etc. On some OSs such as AIX, flags
to ps can print out a process's environment. Which reminds me, appres
needs to execute on the same host as app1.
