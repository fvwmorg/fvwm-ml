I know you will probable receive several responses, and here's one.

I've had this problem with fvwm2 since converting over to version 2.0.38, and
its followed with each release. Any Sun application I started with a "Style"
of NoIconTitle and gave it an icon position via the "-WP X Y" would start in
the upper lefthand corner (position 0 0). This happens to be where I place my
contool. Consequently, that application would be hidden under my contool.

To get around this I removed the icon position and gave it a "Style" with an
IconBox.

While I think that this is not a nice solution, I've also found that this
problem isn't a real big concern (or should I say a high priority) for the
developers of fvwm. They have bigger fish to fry.

I think fvwm is the best window manager out there right now, so I can learn to
live with a little irritant like this. IMHO of course.
