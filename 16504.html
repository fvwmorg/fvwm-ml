(I have a Linux/ELF system). I'm having trouble with getting fvwm
2.0.4[12] (I upgraded my system just recently, so I haven't tried any
of the earlier betas with it) to work with pixmaps. I have defined
"HasXpm YES" in Fvwm.tmpl and get no errors while
building. The xpm library seems to be linked into the binary:

20:49:35 ~>ldd /usr/X11/bin/fvwm2
        libXpm.so.4 => /usr/X11R6/lib/libXpm.so.4.7
        libXext.so.6 => /usr/X11R6/lib/libXext.so.6.0
        libX11.so.6 => /usr/X11R6/lib/libX11.so.6.0
        libc.so.5 => /lib/libc.so.5.2.18

However, there are no icons on screen (well, no pixmaps; bitmaps work
fine), no pixmaps in FvwmButtons and no pixmaps in titlebar buttons. The
xpmroot program that is built at the same time works just fine, and so do
all other programs that use pixmaps. I have tried both relying on
PixmapPath and using full pathnames.

Oh, and when I put into the menu definitions a line like this:
+ "Xterm*xterm.xpm*" Exec /usr/bin/X11/xterm -r -aw -rw +sb
-sl 500 -j -ls -fn 7x14 &
The entry shows as "Xterm*xterm.xpm*" (without the quotation marks). I'm
not sure if this is related to the other problem.

So, what am I doing wrong?
