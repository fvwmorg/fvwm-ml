The application is a properitary telecomunication network umbrella
monitoring tool.

There is anything in the FVWM or X core that handles desktops differnet
to pages.
I see this on some slow applications (old UNIX servers over WAN).
Also, the X backing store option (/usr/X11R6/bin/X +bs -wm) has only
effect in an Desktop between pages.

May be, that KDE (I dont like KDE memory waste, but is true) uses only
pages, to improve
the performance during switching the virtuall workspaces.

I have no problem to change my config from desktops to pages.

(MoveToDesk --> MoveToPage)
(StartsOnDesk --> StartsOnPage and so on)

But we need good labels for the workspaces ......


###################

a pager without labels? I agree, but in "man FvwmPager" (2.4.18) is only
one option showed:

*FvwmPager: ShapeLabels
              Causes the pager to hide the labels of all but the current
desk.
              This turns off label hilighting.

I need only a flat, simply design, that means in this case 10 Pages in
one desktop.
This only one desktop is every time the current desk.

fvwm2rc:
DesktopSize 10x1
FvwmPager 0 0


+----------+----------+----------+----------+-----
| | | | |
| | | | |
+----------+----------+----------+----------+-----
|own label1|own label2| .. |.. |..


as substitude for this:

fvwm2rc:
DesktopSize 1x1
"I" FvwmPager 0 9
*FvwmPager: Label 0 "Label1"
*FvwmPager: Label 0 "Label2"

+----------+----------+----------+----------+-----
| | | | |
| | | | |
+----------+----------+----------+----------+-----
|Label1 |Label2 | .. |.. |..



shapelabels create this:

fvwm2rc:
DesktopSize 10x1
"I" FvwmPager 0 0
*FvwmPager: ShapeLabels

+----------+----------+----------+----------+-----
| | | | |
| | | | |
+----------+----------+----------+----------+-----
| Desk 0

(disables only highlighting for Desk 0)
