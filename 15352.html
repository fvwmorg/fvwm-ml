>>>>> "Mario" == Mario Dupuis <Mario.Dupuis_at_mfg.canadair.ca> writes:

Mario> Hi,
Mario> I'm having a problem with 2.034 on an IBM RS/6000 using AIX
Mario> 3.2.5. When selecting menus from different applications
Mario> (Framemaker,xemac,xterm ...) on occasions the menu will hide
Mario> behind the application itself , thus making menu selection
Mario> impossible. The only way out is to release the menu and try
Mario> again.

Mario> This was also happening with 1.23r ...

This is a known problem, that I think I need to put in the FAQ. This
should only be happening when you are clicking on the menubar and FVWM
needs to raise the window too. Motif menus are done via
OverrideRedirect windows, which are ignored by FVWM. They are coming
up before FVWM has a chance to raise the window, so then they get
obscured by the window. I'm not sure if there is a way to prevent
this or not, but I'll look into it again.

Also, 2.0.34 was a very buggy version. I'd suggest grabbing 2.0.40
from the new official site (ftp.hpc.uh.edu in /pub/fvwm/version-2/).
