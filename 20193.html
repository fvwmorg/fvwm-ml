>>>>> "Matias" == Matias Bjarland <Matias.Bjarland_at_emw.ericsson.se> writes:

Matias> I work at a large corporation and have tried to convert their
Matias> Huge olwm root-menus to fvwm (olwm sucks). Everything went well
Matias> until I started getting error messages like:

Matias> Popup/Function Matlab ignored, you have more than 50

Matias> I get lots of these every time I restart fvwm. My questions is:
Matias> 1. Is there a way to get around the max 50 limit without recompiling
Matias> fvwm from source?
Matias> 2. If not, what do I have to do to recompile fvwm.

This limit is in fvwm version 1.xx only (2.xx doesn't have any limit
like this).

You would have to recompile 1.xx to get around it (look in the
README.Install file for instructions). Look in menus.h for the
MAXPOPUPS define and make it bigger (say 200). Or switch to version
2.xx (it is still in beta but more than stable enough to use).

If you do switch to the latest 2.xx beta (2.0.45 right now but 2.0.46
will be out as soon as I actually get a chance to do some more work on
it), the syntax of the rc file has changed. There is a script in the
tar file that may help you do the conversions.
