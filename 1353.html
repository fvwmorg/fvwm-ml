On 28 Aug 2000 10:52:28 -0400, Raymond Toy wrote:
>
> >> Override Redirect State: yes
>
> Dan> This hint tells the window manager to ignore the window.
> Dan> Since that hint is set, I think all the behaviors you describe are
> Dan> correct.
>
> Hmm, this is weird. When I run it under dtwm, xwininfo says Override
> Redirect State: no. I wonder how rose knows how to set this? Can
> rose figure out what window manager is running? Note that I start
> dtwm and fvwm both via startx. The only difference is that I set
> USE_CDE=yes to choose dtwm instead of fvwm

Probably your application can know about dtwm running by USE_CDE or other
dtwm-documented methods (X Atoms?).

> Is there a way to tell FVWM to ignore this hint for a window anyway?

No, fvwm simply is not notified by X Server about such windows.
Some info in: http://www.hpc.uh.edu/fvwm/archive/9903/msg00958.html

> I don't want to use dtwm, so any hints on getting this to work with
> FVWM would be most appreciated. (I'm too lazy to switch, and I like
> fvwm.)

You can use "BugOpts RaiseOverUnmanaged on" in the betas to improve the
situation, but the real solution is to ask the app authors to fix it.
