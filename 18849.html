In my .fvwmrc, I run FvwmCpp on several files. A file for styles, one for
functions, one for menus, one for modules, one for keymaps, yadda yadda yadda.
However, if I add to the InitFunction in one of these files, the new
InitFunction never gets ran unless I open an FvwmTalk and execute myself or
bind it to a key or button. Even if I try and simply call the function as the
last thing in my .fvwmrc, it still does not get executed. What am I doing
wrong here?

Here is a copy of my .fvwmrc where I read in the separate files, if this helps.
--------
# Standard definitions
Read .fvwm/StdPelDef

# Styles
Read .fvwm/PelStyles

# Functions
FvwmCpp -Cppopt -I/home/ecelrc/staff/pel .fvwm/PelFunctions

# Menus
FvwmCpp -Cppopt -I/home/ecelrc/staff/pel .fvwm/PelMenus

# Keyboard and Mouse definitions
Read .fvwm/KeyMouseMap

# Modules
FvwmCpp -Cppopt -I/home/ecelrc/staff/pel .fvwm/PelModules
---------

I add to my InitFunction with the read from FvwmCpp of PelFunctions.

If anybody could give me some pointers as to how to get that thing going, I
