  Alex,

  depending on how you start the fvwm, it might not have the same PATH as
you have (e.g. when you start X by typing startx from command line, you
might not have X programs in your PATH). try to use the full path to xpmroot
when invoking it from menu as well as full path to bitmap.

  the other possibility is that fvwm thinks that it is a fvwm internal
command, you might need to use the exec or something like that.

        erik
