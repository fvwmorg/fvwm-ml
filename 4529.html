On Mon, Mar 11, 2002 at 08:43:03PM +0100, Uwe Pross wrote:
> Hi there again,
>
> Even I have upgraded to version 2.5.0. I realized a problem using the StickyIcon
> Style and the FvwmTaskBar option DeskOnly together.
> Normally if a window is iconified and the Style StickyIcon is applied it is
> sticky and it is always visible in FvwmTaskBar even when switching to another
> desk and the TaskBar option DeskOnly is applied. That's how it was in my
> previous versions of fvwm2, which was 2.4.3 and earlier.
> In version 2.5.0 an iconified window which is sticky caused by the StickyIcon
> Style disappears from the TaskBar if I switch to another desk and it is shown
> again if I switch back to desk where the window has been iconified.
> If I switch to another desk and apply the command
>
> All (CurrentPage !"Fvwm*B*") Iconify off
>
> the window deiconifies on the current page but it is still not shown in the
> TaskBar. After changing the desktop certain times, sometimes 3 or 4 times, the
> window is shown in the TaskBar.
>
> The problem does not occur if the window is put to the sticky state by applying
> the Sticky command to it. Nor if it is iconified after the Sticky command has
> been applied. Both cases work beautifully.
>
> Can anyone reproduce this behaviour and has a solution?

The problem is gone with a more recent snapshot. Try a snapshot
that is about two weeks old.
