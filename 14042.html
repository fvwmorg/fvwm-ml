On Fri, Apr 22, 2005 at 10:46:29PM -0400, Parv wrote:
> The exchange posted on lair.be is about the user configuration file
> for fvwm(1). Your use of "default config file..." and "tarball [2]
> contains the snapshot of 2.5.12", I got the impression that you were
> referring to the configure script (which creates the Makefile).

Not quite. The tarball contains a modified snapshot of fvwm-2.5.12
which contains the configuration files and some internal coding changes
to fvwm itself which reflects the config file. The reason why we're not
just releasing the config files is that fvwm has to be subtly changed -
to render a menu if no config file is found, etc. They're sublte, yet
important.
