> > display that you can use to start and debug another FVWM process; I
> > looked at ftp.x.org and found xpseudoroot-1.04 which looks
> > promising....
>
> X11R6 has a server called Xnest which is implemented using Xlib calls
> (wierd!!). This has the effect that if you run it you get a window which
> is itself another X server you can connect to and a start new sessions
> etc. without losing your main session.
>
Xnest is the more general solution, but under Linux, many people don't
realize that you can start additional X sessions on other virtual consoles.
You do need the proper command line switch to tell X that it's display :1
for example - I just tried it with:

startx -- :1

and sure enough, a second X server started up using another VC and a second
copy of fvwm, xclock, etc.
