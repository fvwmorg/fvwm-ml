Shared library usage poses yet another problem in trying to compare
memory usage since, at least under Solaris, a process' size contains
the sizes of all the libraries used---regardless of whether they are
shared among other processes.

I'm content to use fvwm based solely on its features and active
development---not on whether it's less of a hog than mwm (which I
wouldn't use in any event). If I really cared about memory usage, I'll
probably run the decorationless nawm. Having said that, I do
appreciate the efforts to keep fvwm itself reasonably lean.
