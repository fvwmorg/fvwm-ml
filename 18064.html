Hi,

I've recently compiled 2.0.43 whithout major problems under Solaris
2.4. In Fvwm.tmpl I set the FvwmIconPath variable to point to the
bitmaps and pixmaps - directory of our X11 - Distribution as well as
to the location of the Icons that come with Fvwm.
In my ".fvwmrc" (which is the proper file, I didn't rename fvwm) I
first set neither PixmapPath nor IconPath.

When I then started fvwm (starting the IconBox-module in the
InitFunction) it found the Icons it should - fine.

Then I set the PixmapPath variable in my .fvwmrc to point to
"/home/trampler/lib/Icons/", where I store some of my favourites,
logged off and back in again. Now FvwmButtons found all Icons (the
ones in the compiled-in directories as well as those in /home...)
while IconBox appeared to have forgotten about the compiled-in values,
i.e. it didn't show an icon for my xterms (xterm-sol.xpm from the
fvwm-Distribution) but knew about the icons in /home...

This means that to use IconBox I have to add the value of FvwmIconPath,
given at compile time, to the PixmapPath. This is no major problem,
but IconBox's behaviour is a bit strange.

Btw, is there a need for two variables to hold locations of Icons
(PixmapPath and IconPath) ? As I understand it, it was meant to
seperate xbms from xpms, but as far as I see they can be mixed in
directories and fvwm doesn't bother.
