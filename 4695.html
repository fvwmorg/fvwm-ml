Fixed. SOme hack in ewmh_events.c was broken by a rewrite of the
icon geometry code. I have cleaned up that ugly window/icon state
and icon placement code with all the global variables. All the
information is contained in a single structure that is passed
between the recapture/icon/add_window code. A side effect is that
the "start iconic" window flag is gone and can't be passed to the
modules anymore. The starticonic *style* is still there, though.

Anyway, it is possible that the rewrite broke something, so please
report any icon placement anomalies.
