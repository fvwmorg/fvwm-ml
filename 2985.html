> something not obvious is happening. it now works on one host which has a
> real three button mouse, but is not on another that has to emulate three
> button. but the pseudo-mouse-2 is working on that host, as i can get the
> appropriate menu in the root window.

Maybe X maps your middle button to number 4 not 2. xterm and emacs paste
using mouse button number 2 only. Start xev to see which (if at all)
ButtonPress/ButtonRelease events this button generates.

By default, all mouse buttons (say mouse button number 4) open menu
MenuFvwmRoot on the root. This binding and menu are built-in.
