Reading through the archives, it seems a bunch of years back there had
been an abortive discussion about making the Fvwm frames able to be
sized independently. Specifically, the discussion mentioned being able
to set the top, bottom, right and left borders to different sizes. I'm
curious how difficult these would be to implement with the current codebase.

Even just a new option for Border/HandleHeight would be interesting. It
could work the same as the Border/HandleWidth option, but obviously
would only set the height of the top and bottom borders. Alternately it
would be better, though probably harder, to make each border
independantly sizable. Any chance any of this is on the todo list?

Also a question. For a verticle 6 pixel border (drawn in craptastic
ascii) The colors draw like below.

   _shadow
 / _hilight
| / _ hilight
|| / _ background
||| / _shadow
|||| / _shadow
||||| /
||||||

Which when made transparent, makes only the background line transparent.
The hilight and Shadow colors are still opaque. If I set the border to
"flat" the entire thing is drawn as the background color and is
transparent, but lacks contrast between windows without the outer 1
pixel shadow color. Is there a way to make all but the outer and
innermost 1 pixel transparent?

Also, the hilight being drawn twice while the shadow is only drawn once
makes the bevel look really odd/bad. Pretty obvious with a setup like:

Style * Handlewidth 20, Colorset 1
Colorset 1 bg yellow, hi red, sh blue

In any case, thanks for your time all. I've been using Fvwm for years
and it just keeps getting better and better.
