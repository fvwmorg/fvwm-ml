Paul Fox <pgf_at_foxharp.boston.ma.us> writes:
> sometimes when i think an fvwm2 window has focus (i.e., it's turned to
> the focus colors) i still can't type at it, because the mouse pointer
> is over the scrollbar for the window. moving the mouse just a bit into
> the window text area fixes things. thinking about this, i've probably only
> seen this with xterms, but it might happen elsewhere -- i forget.
>
> is there a setting that gives focus no matter where the mouse pointer
> is?
>
> for the record, i get the undesired behavior on 2.2.4 on linux. i get
> the behavior i want with fvwm2 version 2.0.46 on solaris, and i think
> on a similar vintage version running on linux.

This is an issue with the Linux version of xterm.

It doesn't happen with the Solaris version of xterm,
or any version of rxvt.

If you want to get xterm fixed, you should contact the authors.

In the mean time, you might want to switch to one of the many
xterm replacements. rxvt suites me.
