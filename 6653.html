On Tue, 22 Oct 2002, Peter McAlpine wrote:

> I'm wondering about getting my terms to be transparent. I use
> gnome-terminal (but have no problem using xterm if it works better),
> and when I set the background to transparent, it's just black. I use
> qiv to set my background, and I heard that it sets the background by
> just making a window the size of the screen, which may be fooling
> gnome-terminal... anyway... what's the easiest (and I suppose most
> fvwm P.C.) way to set the background? And after I do that, what steps
> need to be taken to set up transparency on my terms?

Remember this part I sent you the other day?

AddToFunc InitFunction
+ I Exec Esetroot -f /usr/share/pixmaps/vpm_wp_01_1024.jpg &
+ I Exec Eterm -g 26x21+0+552 -F 5x7 -O -T XMessages -e tail -f $HOME/.X.err

That's what I use. Esetroot to set the background image and an Eterm with
the -O option (man Eterm gives an explanation).
It's really a pseudo-transparency, but it works well enough for this.
I think aterm also has a transparency option.
