>>>>> "John" == John Henders <jhenders_at_bogon.com> writes:

John> Charles Hines writes:
>> OpaqueResize is gone. It was an interesting feature, but needed a
>> really good X server & hardware to be used. It might resurface, who
>> knows? I can put it on the TO-DO list, if people really want to see
>> it back.

John> I sure miss it. It's easy to get used to on a standalone X box,
John> and was quite fast on my 486/80 with an old ATI gup.

Ok, I got more than enough notes from people who want it back. After
I fix the Style flags to allow more than 32, I'll make sure it gets
brought back.

John> On another track, has anyone using click-focus found that
John> occasionally when you click on any window border to move or
John> resize it, the window "sticks" to the mouse? I've had this
John> happen about once every 2 days, but I can't reproduce it
John> reliably.

I know about this, and see it quite often. I believe it has to do
with Motion being done with a timeout instead of a MotionThreshold
value. If a window takes a long time to refresh while raising, fvwm
thinks it was actually a Motion instead of a Click, and tries to move
or resize the window. I already plan to make Motion work off of a
MotionThreshold variable instead sometime in the near future (after
the first official release).
