On 19 Apr 2003 12:25:06 +0000, Peter Olsen wrote:
>
> I have just installed fvwm 2.5.6 on my RedHat 8 system, but I have this
> strange problem. The fonts are virtually unreadable. I can read the
> font on the buttonbar, but the menu which pops up when I press the
> "start" button is almost unreadable. The same goes for the fonts on the
> titlebar of windows and the fonts in the fvwm configuration form. How
> do I correct this problem?

There is no enough info to help you. And this is probably not an FVWM
problem.

By "unreadable" do you mean tiny or bad quality?

What is the font you use for menus (search for "MenuStyle * Font" line)?
What is the locale you use (run 'locale' from a shell)?

Do you have good fonts installed (at least XFree86-100dpi-fonts)?
Does your font server (xfs) function properly? How other apps show texts?

If you speak about the Setup95 configuration, for menus it uses font
"-*-times-medium-r-*-*-16-*-*-*-*-*-*-*", you may test it by running:

  xterm -g 40x10 -fn '-*-times-medium-r-*-*-16-*-*-*-*-*-*-*'

Of course it is not a monospace font, so xterm does not look ok with it.
But at least it should be pretty readable.
