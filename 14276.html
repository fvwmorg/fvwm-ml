>>>>> "Randall" == Randall S Winchester <rsw_at_eng.umd.edu> writes:

Randall> I had to make two changes to configure.h.
Randall> One is new for pl-35, the other I have always made. I think
Randall> both of these changes should be made for pl-36. (A patch
Randall> follows.)

Randall> This first one is that I had to put the following back;
Randall> /* Imake command needed to put modules in desired target location */
Randall> /* Use the second version if it causes grief */
Randall> #define TARGET_DIR BINDIR=FVWMDIR
Randall> /* #define TARGET_DIR */

Randall> If you just "#define TARGET_DIR", you will have your modules
Randall> installed in the X11 bin directory (/usr/local/X11R6/bin in
Randall> my case). The intent is to have them installed in FVWMDIR
Randall> (i.e /usr/local/X11R6/lib/X11/fvwm2). Otherwise you may not
Randall> be able to run fvwm and fvwm2 concurently.

Good point. Done for next patchlevel.

Randall> The second one is that I always comment out;
Randall> /*************************************************************************
Randall> *
Randall> * Really, no one but me should need this
Randall> *
Randall> ************************************************************************/
Randall> /*
Randall> #if defined __sun__ && !defined SYSV
Randall> #define BROKEN_SUN_HEADERS
Randall> #endif
Randall> */

Well, if Rob is really the only one who should need that, it makes
sense to comment it out by default, I guess. I put an #if 0 around it
for the next patchlevel.
