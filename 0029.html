"Leo deck" <Leo.Deck_at_trw.com> writes:
> I'm running fvwm on a Sun Solaris system. What is a good debugger for debugg
> ing potential problems with fvwm?

Whats a "potential problem"?

If you have a core file:

  If you compiled with gcc, use gdb.

  If you compiled with Sun's compiler, use dbx.

If you suspect memory corruption or leaking, use Purify (if you have it).

If you are getting incorrect behavior, find the area of fvwm you suspect.
See if there is any debugging code you can enable thru recompile.
Consider adding debugging statements or stepping thru execution with dbx
or gdb.
