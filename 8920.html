"Stephen Biggs" <yyyyy50_at_hotpop.com> writes:
> Hi all,
>
> I give up. I tried searching all of the "man" pages and search engines
> on the Internet for how to do this, but came up dry.
>
> Running the latest cygwin on Windows 98. I installed all of the Xserver
> and fvwm from cygwin setup, as well as the xpm-.
>
> It seems that fvwm doesn't want to install itself into the cygwin XFree86
> run system, so I added a line in startxwin.sh below the commented out
> line for twm that runs 'fvwm2 &'. I couldn't see anything else to do,
> and now, when I run startxwin.sh, I get the fvwm but a huge amount of
> errors. Also, when I try to shutdown the xserver, fvwm shuts down, but
> then it tries to continue running the Xserver until I shut that down
> explicitly with Alt-F4.

What version of fvwm are you running?

> Here is the output from a run of startxwin.sh. What am I missing?
> [FVWM][ReadDecorFace]: <<ERROR>> couldn't load pixmap mini.winXX-close.xpm

Fvwm can't seem to find any of your images.
Have you downloaded the icons package?
Have you set the ImagePath to point to the directory containing the icons?

> [FVWM][CMD_Style]: <<ERROR>> Bad style option: \
> [FVWM][execute_function]: <<ERROR>> No such command 'StaysOnTop,'

This looks like Fvwm might be having a problem understanding backslashes.
Do you have a style command that looks like:

Style * xxxx \
  StaysOnTop

Is there a space after the backslash (there shouldn't be one).

On your issue with shutdown, stopping Fvwm doesn't have to stop
X. If you want that to happen, the command to execute fvwm should
be the last commmand in your .xinitrc and there should be no ampersand
after the command.
