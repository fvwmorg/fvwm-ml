In short I disagree. Apart from already having over 100 "dot" files in my home
directory I think that breaking up the configuration file(s) is a bad idea.

I think it may be a better approach to allow the use of a common system config
file (eg /usr/local/lib/fvwm/fvwmrc). This file would be read in as a precursor
to any users' .fvwmrc file.

> entire .fvwmrc file. This would make it easier for novice users to debug
> faults in their configuration.

I dont think having more files and increasing the complexity will help
newbies. I think the biggest way to help them is to improve the documentation.
