Howdy, I've had this problem for ages now (ever since fvwm-2.0 was
released I think). Whenever I try to save something (ALT-s) within
netscape the "Save As..." window shows up with only 1 button
eventhough I have 2 buttonstyles specified and all my
windows/xterms/etc have these 2 buttons on them.

#.fvwmrc parts
ButtonStyle 1 17 20x20_at_1 30x20_at_1 50x40_at_1 70x20_at_1 80x20_at_1 80x30_at_0
                 60x50_at_0 80x70_at_1 80x80_at_0 70x80_at_0 50x60_at_0 30x80_at_0
                 20x80_at_0 20x70_at_0 40x50_at_1 20x30_at_0 20x20_at_1
ButtonStyle 2 14 0x0_at_1 0x20_at_1 20x20_at_0 20x0_at_0 0x0_at_1 0x25_at_1
                 25x25_at_1 25x0_at_1 55x0_at_1 55x55_at_0 0x55_at_0 0x60_at_1
                 60x60_at_1 60x0_at_1

# Button Context Modifi Function
Mouse 0 1 A Iconify
Mouse 0 2 A Resize

(please note that each buttonstyle is a single line)

So here's the weird thing. When I get this transient "Save As..."
window for the first time after starting Netscape its size is normal.
All subsequent times, its almost half the "normal" size and I have to
go into my menu setting and "resize" that window.

Is this a known bug ?

BTW-1: my ALT-f "Netscape: Find" also shows up without a second
button, but it doesn't have the half-sized problem.

BTW-2: my ALT-o "Netscape: Open Page" also shows up without a second
button and it also beeps whenever I try to resize it (that's fine,
just mentioning it to note that the lack of a second button makes
sense since that button is used for resizing, but is fvwm that
smart ?? :-)

Thanks for the help in advance.
