I have just finished writing a patch to FvwmIconBox that does much the
same as my last patch does; it makes the icons is the box the NextStep
shaded style. In addition it moves the shading functions into the /libs/
area so that other functions and modules can use them. If there is any
interest to this patch I will post it to my home page. In truth I will
post it no matter what, but interest will speed up the process.

Second, while looking at FvwmIconBox I observed that it was using a
structure that was similear to FvwmWindow, and it seems that if I changes
the structure to that I could move all of the icon code out of
/fvwm/icon.* and put it into the /lib/ area too, seeing as the code is
otherwise identical. Just a though. Is seems that FvwmGoodstuff and
FvwmButtons all use nearly identical code to the fvwm/icon.c code, so why
have three nearly idencial hunks of code when we can have one piece of
code. This is good for more than asthetics, becuase linux is sufficently
smart to share this code (if we make a shared lib that is).

Just my thoughts,
        Ian
