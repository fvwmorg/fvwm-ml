    Austin> At the moment, fvwm reports errors by calling
    Austin> fvwm_err(). This put the message out on stderr, where it
    Austin> will usually get lost lying in ~/.xsession-errors.

    Austin> How about sending the text of the error message to an
    Austin> 'FvwmErrorDelivery' module, which could pop up a window
    Austin> containing the text. Alternatively, if an FvwmConsole is
    Austin> open, then the errors should probably go there.

Chuck has already given fvwm_err a better interface as fvwm_msg (with
all of the old calls updated), and has already voiced intentions in
the future to allow one to send output to a logfile.
