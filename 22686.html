On Tue, 11 Aug 1998, Patrick Rogan wrote:

->It would be intersting to be able to selectively have titlebars on the
->bottom of windows -- either dynamically, or via the Style command.
->However, I think that this should be a seperate function that
->compliments the windowshade function you described.
->
->Most Style commands have a "style-option/oppositestyle-option" pair.
->Maybe all of these commands could have a 'toggle-option' varient as well
->(or add a toggle modifier like so:
->
-> Mouse 1 T A Toggle ClickToFocusDoesntRaise

I'd just like to bring up an issue from waaaay back that is related.
Years ago now a patch was posted that put title-bars on the sides of
the windows. I used to have a screenshot of this but I had to delete
it for space reasons. At the time it was only possible to specify all
your windows to have titlebars down the sides, and the drawback then
was that the text was not rotated and so looked a bit messy.

What I would like to see implemented (big wishlist item this) is to
have the option to have a title bar wherever I want it. I would find
this useful when, for example I am editing a bunch of files with long
names. Because my text editor is taller than it is wide on the screen
there's more room down the side of the window than across the top.
However, I don't want xterm windows to have side title bars as I found
that it was inconvenient having the title bar next to the scroll bar
(what can I say, my aim is very bad)...


Just a suggestion ;)
