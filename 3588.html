I am running 2.4.3 on a dual screen Sparc (Solaris 7) - not a Xinerama
setup, so I have two instances running.

I have configured FvwmEvent per FAQ 3.18 to look for new windows. My
"Cmd" just Echo's the window ID (for now). Problem is, if a new window
pops up asynchronously (simulated by "sleep 5 ; xterm &") and if the
pointer is not on the screen where the new window appears, the pointer
is warped to 0,0 of the new window's screen. Since I use SloppyFocus,
whatever I am doing is interrupted until I pull the pointer back to
where it was. (I've set "Style * GrabFocusOff".) The problem doesn't
occur if FvwmEvent isn't running.

Is there a way to suppress the pointer warping?

Less troubling (but possibly related?): This warping also happens on a
restart. If I restart on screen 1, then move the pointer to screen 0
right away, the pointer warps to 0,0 of screen 1 when the restart
completes.

Gregg Dameron
