Well, it turns out that indeed the version of FvwmIconMan I had was
incompatible with the version of fvwm I'm running. Looks like a lot
of the time the generic fvwm2 distribution ships with a version of
FvwmIconMan made for the version or two before. So I was running
.9 which was meant for fvwm-2.0.43 or so. I recompiled it from the
fvwm-2.0.46 distribution (which says it is for 2.0.45) and now the
commands work.

What I haven't done is install fvwm-2.0.46. Once I do so (I'm going to
add the startsonpage patch), can I expect FvwmIconMan to break again?
