On 18 Sep 2000 18:30:30 -0700, Michael Han wrote:
>
> > Is there any way of having taskbar-visible/invisible occurances of
> > the same Prog (Netscape) at the same time on the same desktop
> > (remember they are run by seperate users) ????
>
> After fiddling with your problem quite a bit, I've determined that
> your best bet is to do some stuff with the initialization builtin
> function in fvwm. I imagine something like:
>
> + "I" Exec exec netscape
> + "I" Wait Netscape
> + "I" Style Netscape WindowListHit
>
> will work if you throw it in your initialization function. N.B. I
> don't think fvwm95 has a Wait builtin. Basically, what you're doing is
> running the first netscape at fvwm's startup, waiting until it gets
> styled with normal settings, and then changing the styling back to
> normal.

Sorry, Michael, it is not a solution. Please don't even try this. :)
Even if some fvwm version allows here two Netscape windows to have
different styles, it is more than temporarily.

  1) On Recapture all your Netscape windows will do the same
  2) Since this is in InitFunction, this for sure will not work on Restart
  3) This can't work in 2.3/2.4, where the last line is evaluated for
  both existing and new windows, i.e. all Netscape windows do the same

The real solution is to try to have the second Netscape with a different
resource/class. I didn't find how to do this using -xrm or Netscape.ad,
but this does not essentially mean it can't be done. However I just edited
the netscape binary and successfully replaced Navigator by Myvigator...
