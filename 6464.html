I've recently started using .png images to get those smooth anti-aliased
image effects, and whilst it works perfectly for titlebar buttons, menu
icons etc., it does absolutely nothing concerning menu side pixmaps.

I'm using fvwm release 2.5.3.

I've tried both the "direct" approach (using "_at_picture.png@" in menu title
definitions), and the SidePic option of the MenuStyle command. Both of
them work identically - the image is shown, but it doesn't get
antialiased, and it seems that the side image is getting only a 1 bit
deep alpha channel. And all those nice .png images look as if they were
plain .xpm :(

So, am I doing something wrong? Or is it that png alpha-blending hasn't
yet been fully implemented in menu side pics?
