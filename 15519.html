David Kennedy wrote:
-> Where should I set an environment variable (say XAPPLRESDIR) in order for
-> FvwmButtons to pick up on it? Can it, will it? And ditto for paths.
-> Is there any way to get around this problem? I have some apps that run
-> from an xterm command line but not (properly) from FvwmButtons due

env XAPPLRESDIR=<something horrendous> xterm -args
See the man page on "env" for more info.
