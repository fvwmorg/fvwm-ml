%% Pete Forman <gsez020_at_compo.bedford.waii.com> writes:

  pf> FvwmIconMan can have multiple instances to manage sets of windows as
  pf> you request. Check out this list's archives for examples of
  pf> combinations of FvwmButtons and FvwmIconMans to construct a task bar.

This would be perfect... if I could get FvwmIconMan to "button-ify" some
of the managers.

What I'd like is to be able to specify that some of the managers just
display a button, then when the button is pressed they'd show all their
managed windows, in a popup window-type thing.

Since there can be only one "transient" manager definition, I don't see
how I can do that by starting up an FvwmIconMan module when an
FvwmButton is pressed (for example), since I'd want different managed
icons depending on which button I pressed.

Plus, I'd prefer not to have to start an icon manager "on-the-fly" as it
takes too long (I still use the builtin winlist bound to mouse-3 instead
of a separate module, for the same reason).

Thanks for the response, though...
