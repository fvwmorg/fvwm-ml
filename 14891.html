I don't know if it should be an optional behavior, a configurable option,
or the semantics of the implementation, I just know it would be very
useful if someone would allow a menu that currently looks like this:

 __________
|__________|
|__________|
|__________|_________
|__________|_________|
|__________|_________|
|__________|_________|
|__________|_________|
|__________|_________|
           |_________|
           |_________|
           |_________|
           |_________|
           |_________|
           |_________|
           |_________|
            ........
            (and forever into oblivion)
            ........
           |_________|
           |_________|

to look like this:

 __________
|__________|
|__________|
|__________|___________________
|__________|_________|_________|
|__________|_________|_________|
|__________|_________|_________|
|__________|_________|_________|
|__________|_________|_________|
           |_________|_________|

(you get the idea)

or maybe even 3 rows, depending on 'what is appropriate'. I don't have a
clue to suggest what is appropriate, it is just that the xlock menu of
possible modes is getting a bit too long :-) and so I thought maybe this
could be a 'To-Do' useful feature to be added SOMEHOW.

I leave it up to you guys to decide 'how' and 'what is good'.

Have fun!
