Is there any way to start separate icon managers on each page?

I previously reported a wrapraround problem with FvwmIconMan:
    http://www.hpc.uh.edu/fvwm/archive/0402/msg00179.html
I can reproduce this problem whenever there is a window on one page which
abuts an IconMan on another page; moving the mouse into the IconMan on the
other page warps to the IconMan on the original page. Adding space between
the windows helps, but is not reliable -- some mouse motion will still
occasionally cause this glitch, but I have been unable to determine exactly
what controls this.

I suspect what's going on is that IconMan doesn't refresh quickly enough, so
that IconMan continues to show the windows from the original page just long
enough to warp back to that page. A workaround would then be to start
separate icon managers for each page, rather than using a single Sticky
manager. But I can't figure out how to do this!

I've tried giving each instance a separate title, then using something like
        *FvwmIconMan*2*title "Page2"
        Style "Page2" StartsOnPage 1 0
but this seems to affect all instances of IconMan, not just this one. I would
happily write a command to start multiple instances of IconMan, using GotoPage
in between to set the page, but this fails since only one Module can be
running at once. The only other way I know to specify the startup page is to
use *managergeometry with pixel settings which push things onto another page;
this indeed works, but (I believe) causes those instances to think they're
still on the original page, so they don't show anything from the page they
appear to be on.

Any suggestions either regarding the original problem or possible workarounds?

Thanks.
