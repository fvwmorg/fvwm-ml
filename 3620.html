On 12 Nov 2001 12:30:58 +0100, Dominik Vogt wrote:
>
> On Mon, Nov 12, 2001 at 10:31:23AM +0200, Tom Alsberg wrote:
> > Hi there.
> > Is it possible to cause windows' movement to be shown on the screen
> > (not necessarily opaque, maybe an outline) when dragging them in the
> > FvwmPager? If so, how do I set it so?
>
> Well, you already answered your own question in the subject. If
> windows are moved as a wire frame, fvwm draws directly onto the
> screen. To prevent other applications from painting over this, it
> grabs the X server, i.e. forbids all other applications, including
> the pager, to do anything on the screen. If windows are moved
> opaque, this is not necessary, i.e. window motion is visible in
> the pager. Put this in your config file:
>
> OpaqueMoveSize 100
>
> (or if some windows move too slow, decrease the value).

Tom asked another question, he wants to drag a mini window in FvwmPager
and see the real window moved on the screen synchronously.

I don't think this is possible with the current code.
