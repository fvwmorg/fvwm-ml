>>>>> Andreas Schumann writes:
Andreas> Does anybody know how to configure the Emacs icon title so
Andreas> that the filename of the current buffer appears in the title
Andreas> instead of emacs_at_HOSTNAME ?

>>>>> Kai Grossjohann responds:

Kai> Modify icon-title-format and/or frame-title-format to get what
Kai> you want.

Kai> (setq frame-title-format "%b") ought to do something useful for
Kai> you, but is untested.

You'll actually want to set them both, most likely to be the same
thing :-) Normally if you set frame-title-format and leave
icon-title-format unset, the icon title will follow the frame title.
However, there is at least one situation where the icon title will not
take if icon-title-format is not explicitly set:

I normally start emacs iconic and run a couple of functions from the
command line to build a set of frames for Gnus, the calendar and w3.
As part of my startup I set frame-title-format to
("%b (" mode-name ")")
which lets me separate certain emacs modes into different FvwmIconMan
managers (yes, I'm a geek with too much time on my hands). Until I
explicitly set icon-title-format to the same thing, that initial
frame's icon kept the emacs_at_hostname title, even if I deiconified the
frame. The icon name would change appropriately when the frame was
open, but if I iconified it again, it went back to emacs_at_hostname.
