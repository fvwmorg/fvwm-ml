Hello, all -

This isn't an official announcement, but myself and Nick Fortune have
been discussing [1] changing the default config file which is currently
shipped with Fvwm. For many the styles and workings of it are too
archaic for today's more 'modern' tastes.

So we came up with our own proposal, and have been hacking on it since.
There has been some feedback, but it is limiting, and we could ideally
do with more, before we approach fvwm-workers with our proposal. The
tarball [2] contains the snapshot of 2.5.12 - you'll have to compile it
in the normal way. I suggest if you try this that you create a new user
and use Xnest to test it, or move your ~/.fvwm directory out of the way
beforehand, as some files may get overwritten.

The installation instructions do not differ. There is documentation
for the interface, and we hope that a lot of it is intuitive enough such
that it shouldn't require much in the way of explanation about how to
navigate things, etc.

The emphasis of the proposed configuration is that fvwm is all about
doing this Your Way (tm) - there will be people who hate the config file
produced. In a way, I'm hoping this reaction will spur them on to
change things for themselves. The config files are documented without
reproducing the manpage in so doing. Where applicable at the foot of
the document are references to other sources of material which might be
of benefit.

-- Thomas Adam
