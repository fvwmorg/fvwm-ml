Gail Mara Gurman wrote:
gg> Since you said that, in the future, there will be a higher number of
gg> possible styles, I'd like to make a suggestions. How about user-defined
gg> styles?
gg>
gg> That is, right now, the only way to group styles is for "*" or common Class
gg> or name strings. It would be nice if I could group them differently. For
gg> example:
gg>
gg> AddToStyle Info NoHandles
gg> + NoTitle
gg> + Sticky
gg> + WindowListSkip
gg> + BorderWidth 0
gg>
gg> I have this combination of styles on for several applications and it would
gg> be cleaner to just say something like:
gg>
gg> Style "Perfmeter" Info

This ist true, if editing the .fvwmrc remains the only way to configure
the behaviour. But if one is going to write a GUI to aid the
configuration of fvwm without resoting to use a text editor, the feature
you suggested will make this job harder. I don't know much about
'FvwmConfig', since it was not available in fvwm-1.24r, but I would
love to see a nice GUI Frontend for editing ~/.fvwmrc. This would
allow a much easier use for those casual users, which are unfamilar
with the editing of ~/.*rc-files.

reference in my clipboard for some weeks now, but since I've only
mail/news access to the net, I haven't got someone to fetch this software
