I have a strange problem with a few applications. When I iconize them,
the icon pixmap does not appear (although the text label does still).

I have the following line in my .fvwmrc:

Style "*" Icon unknown1.xpm

This works fine to catch all (or almost all!) unknown applications. After
that I have a number of "Style" statements for various applications. Most
work perfectly.

For a few applications, though, this causes the icon pixmap to not appear.
For example (for Andrew applications):

Style "atk" Icon andrew.xpm

This pixmap is definately available, readable, valid, in the path, etc.

Even if I change this to:

Style "atk" Icon unknown1.xpm

I get the same result!

Another problem application is Mosaic. I can't even get the default
"unknown" pixmap for this.

Has anyone else seen this, or got a fix? Or is it something I am doing
wrong?
