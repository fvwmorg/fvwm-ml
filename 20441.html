Rob Lowrie <lowrie_at_lanl.gov> writes:
> On Fri, 11 Jul 1997, Dan Espen wrote:
>
> > If you work up a patch, I suggest a command like "DistinctDesktops n",
> > where "n" would limit the "distinct desktops" to 0 thru n.
>
> In retrospect, I agree, such a patch would have to go into fvwm2 as
> opposed to trying to modify FvwmPager. "DesktopRange nStart nStop" might
> be a better syntax.

It might be, but the negative desktop numbers seems like a mis-feature
to me. The above syntax seems like it would invite the user to use
all kinds of strange ranges, and even more than 1 range. However, I
think the patch writer would have first choice, and Chuck would get
the last word.

> Originally it seemed a bit strange to have FvwmPager as a module, but I
> suppose there's someone out there who uses multiple desktops without
> FvwmPager.
>
> The reason I want this feature is so that the following keyboard short
> cuts (I have a 2x2 pager setup)
>
> Key Right A SC Desk 1
> Key Left A SC Desk -1
> Key Up A SC Desk -2
> Key Down A SC Desk 2
>
> behave with sanity.

I never run with the pager. (I don't use any modules at all.)

I use a 2x2 desktop and navigate the 4 desks with ctrl arrow:

Key Left A C Scroll -100 0
Key Right A C Scroll +100 +0
Key Up A C Scroll +0 -100
Key Down A C Scroll +0 +100

On the other hand, lots of people here copy my configuration,
and almost all of them find that they like to run the pager.

Go figure.
