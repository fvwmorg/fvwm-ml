Yes, if you do "cvs update -D '2002-07-06'" and keep the current Flocale.c
the initial drawing of non-first desks (grid and mini-windows) is
incorrect.

Your latest fixes (probably related to MapNotify) indirectly fixed this
problem, but it may appear again.

The problem is that until 2002-07-06 the very first HandleExpose calls
DrawGrid that calls XDrawLine, everything with correct parameters, but
only the first desk grid is shown, so I guess Desks[desk].w window itself
with non-zero desk is not visible. It becomes visible on any operation
(shade, move).
