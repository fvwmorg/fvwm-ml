How about another X server, kind of like Xest, which would eavesdrop on the
stream of requests to the main X server, and display the same thing except with
automatic subsampling?

Since this nested piggy-back X server would run only at the same time as the
main X server (only faster, due to less data to display), the user should only
perceive the added work as an X server slowdown. The slowdown would be a small
fraction of the normal latency of the main X server, ideally equal to the ratio
of the resolution of the piggy-back server to that of the main server.

Could Xnest be mangled to perform such a piggy-back service?
