Hi, I am new to this list, but I do like fvwm a lot because it is so light
and fast and yet powerful and I prefer to use it instead of the bloated
windows managers installed by newer versions of RedHat Linux (which we use as
the Linux distribution here)

Anyway, I was very pleased with a feature which was present in version 2.0.xx
of fvwm2 but it doesn't seem to be there for more recent versions ( I haven't
tried the v2.3.xx yet). With the newer versions I cannot make the taskbar to
display the application windows for the _current_ disjoint desktop only, in
other words it seems to me that there is no way to setup multiple disjoint
desktops each with a separate taskbar behaviour, as now the taskbar displays
all the active applications opened in all the independent desktops at once,
which is sometimes very incomfortable for me. As a result, I had no choice
but to uninstall the fvwm v2.2.xx which comes with RedHat 6.x and "downgrade"
to a 2.0.xx version that I took from a RedHat 5.2 distribution. I checked the
development history information at fvwm.org and I couldn't find a reason or a
mention of this feature being removed from fvwm2.

Am I missing something here? Was this feature (of FvwmTaskBar, I guess)
removed for good? Maybe I just don't know, for the new versions, how to make
the FvwmTaskBar to be desktop-specific and display only the applications
opened in the current disjoint desktop...

Thank you for any hints about this.
