Vikas Agnihotri <VikasA_at_cagw1.att.com> writes:
> Using RH5.1
>
> I have the following in my $HOME/.xsession (I launch X via XDM).
>
> #!/bin/sh
> ...
> xsetroot -cursor_name iron_cross
> ....
> ...
> exec fvwm2
>
>
> When fvwm2 starts up, the root cursor is reset back to its default value.
> When I remove the 'exec fvwm2' line to get a nekkid X session, my root
> cursor is indeed XC_iron_cross.
>
> Why is FVWM2 resetting the cursor? I didnt see anything in 'man fvwm2'
> about setting the root cursor?
>
> Help? How do I make fvwm2 leave my root cursor alone?

I don't see where fvwm2 is changing the root pointer, but adding this
to your .fvwm2rc should fix it:

AddToFunc InitFunction "I" Exec xsetroot -cursor_name iron_cross

(untested)
