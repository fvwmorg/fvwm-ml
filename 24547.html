FvwmAnimate is running. The save option just tells FvwmAnimate to save
the options to a file in the user's home directory named .FvwmAnimate
which is read by FvwmAnimate when it starts up. You hit the problem right
on the head, though, with your last comment. I am slowly modifying the
default "decor" of Fvwm95 on a RH5.2 system. As such the default is to
have both FvwmTaskbar running and Style * NoIcon! Commenting out this
style definition corrected the problem.
