%% Andre de Boer <AndreDeBoer_at_hot.A2000.nl> writes:

  adb> But your right: "checking for X" goes wrong.

  adb> Do someone has a suggestion?

  adb> checking for X... (cached) no

First, remove the file "config.cache" to punt the cached result.

Then, re-run configure exactly as you did before.

Then, send us (a) the configure command line, (b) the version of FVWM,
(c) the type of system you're using, and (d) go through the file
"config.log" and find the place where it starts looking for X and
fails. Cut that whole section and include it (or, maybe looking at it
will let you solve it yourself, but if not feel free to send it along).

Note that if it can't figure out X maybe you don't have the headers
installed. Particularly on Linux distros often every package of
libraries is split into a "normal" package and "dev" package, where, the
dev package has all the headers and static libraries. To build FVWM (or
any other X app) you need the dev package installed.
