Chuck = Charles Hines wrote:
>
> Or you can put the module config info in a separate file and use the
> 'Read <file>' command to load the changes without restarting fvwm.

I have observed that this is not true. If you "Read" a changed
config file, the previously set options aren't reset to the defaults.

In case of FvwmButtons this is a desaster, because after rereading the
config file you will get a button bar that is twice as big, because
every button from the new configuration is ADDED to the old configu-
ration. Nothing but a restart of Fvwm helps then.

I just tested it to be sure, and guess what I am gonna do next?
Yep, I have to restart fvwm, to get my original button bar back. :-(
