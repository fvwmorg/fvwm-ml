On Thu, 23 Jan 1997, Andrew Veliath wrote:

> On 23 Jan 1997, Mark Borges wrote:
>
> > However, starting with fvwm-2.0.44 on some of the FrameMaker5
> > windows I only get two rightmost buttons -- lower and minimize, while
> > on others I get all four. It's not a real big deal, but I would like
> > to have the right-most buttons on all my windows be bound to a
> > close/delete action.
>
> I think you are using MWM* stuff (MWMDecor, etc.) and if you do so you
> must setup your buttons in an MWM style (such as the standard fvwm2 look).
~
To clarify, if the application gives MWM hints to not show a Minimize or
Maximize button such the window in question, fvwm2 simply omits either one
or two of the rightmost buttons. Fvwm2 has done this WAY before 2.0.44
(i.e. it hasn't been touched). This is why you lose those two rightmost
buttons, leaving only lower and minimize (even though fvwm2 ideally should
have left out minimize and maximize alone).

Since people are having many new button combinations nowdays, it might be
worth it to have fvwm2 listen to button hints and remove buttons according
to binding -- but this could be iffy since you might have it bound to a
custom function, etc.

A possible option would be to have MWMMaximize and MWMMinimize, and
MWMTitleMenu button flags so the user can explicitly set which buttons
denote MWM functions (this would really be the only sure shot way to do
this...).

I can make a patch if you would like to try something like this. Come to
think of it, an MWMMaximize flag would nullify the need for the current
MWMButton flag as well.
