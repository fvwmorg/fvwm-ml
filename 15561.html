A bug I found (currently FTPing 2.0.41 so don't know if it's still
present there).

        * Trying to load either FvwmIconBox or FvwmWinList into
          FvwmButtons (SwallowModule) makes them (IB & WL) crash 

---
Also, I think that the addition of a Sleep built-in, would be a Good
Thing(tm).  The reason I'm asking for this is that I haven't been able
to find a way to do the following:
AddToMenu Quit-Verify   "Really Quit Fvwm?" Title
+                       "Yes, Really Quit"  Quit
+                       "Yes, Save Desktop" Quit-and-Save
AddToFunc Quit-and-Save    "I" Module FvwmSaveDesk
+                          "I" Quit
The reason it doesn't work is that before FvwmSaveDesk has the time to
create the new.xinitrc in $HOME, FVWM causes it to exit.  Unless I'm
doing something wrong, or there IS another way to do this, the
addition of a Sleep function, that would cause FVWM to pause for 'n'
seconds before executing the next command would fix this.
Any help / comments / suggestions should be directed at me
(adrenlin_at_ki.net) since I'm not subscribed to the list (get enough
mail as it is!) as well as posted for the benefit of others.
