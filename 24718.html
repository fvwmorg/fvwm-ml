Actually, this used to be the way it worked. I have three Icon
managers defined in my .fvwm2rc and have had it this way for a long
time over several different versions of fvwm2. With the newer
versions, the icon manager will display a blank placeholder if there
are no windows that correspond to that manager. Older versions simply
didn't appear in that case.

For example,

  *FvwmIconMan*nummanagers 3

  *FvwmIconMan*1*title "Cadence DFII Windows"
  *FvwmIconMan*1*buttongeometry 150x0
  *FvwmIconMan*1*managergeometry 1x0-0+500
  *FvwmIconMan*1*show resource=Cadence* \
                                resource=icfb resource=dockit resource=LSW \
                                resource=opus resource=cdsLibManager

If I'm running cadence, then I get several items in icon manager 1 as
expected. Now, however, when I'm not running Cadence, I get a single
entry in the icon manager with the title "Cadence DFII Windows" in it.

It seems to me that this is something that was a "fix" in the newer
revisions. However, I prefered it the old way.

Sorry, I can't remember the specific version numbers -- 1.4 maybe?
