> I have a programm "xisp" which I use for dialup. This programm can react
-snip-
> Also sometimes xisp doesn't get started. The cursor changes from X to O
> and back to X (you know what I mean :-), but xisp won't start...

I have a similar problem in trying to start ImageMagick from a menu in
2.0.46. The ImageMagick docs say to use: display logo:
to start from menu. The other way to start it is do run "display" from an
xterm.
What happens when executed from menu is the program dies as soon as the
programs logo is loaded. It works fine when executed from xterm.
A workaround I found was to create a shell script that executes "display"
and call that from the menu. It works. So perhaps a similar workaround MAY
work for you.
