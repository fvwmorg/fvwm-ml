On Fri, Mar 08, 2002 at 05:28:31PM +0100, Marcus Lundblad wrote:
> I just installed the latest snapshot, and was happily surprised there is
> now support for tear off menus. Thanks a lot! I have been missing this
> feature for a long time.
>
> Though when tearing off a menu and then clicking button 2 on the title the
> window jumps to a new location as if it opens a new tear off menu and
> closes the old

> (which I guess is what happens).

Exactly.

By default, menus are placed like any other window. If you don't
like that, put the style

  Style fvwm_menu UsePPosition

in your config file. But then, borders or even complete menu
entries are sometimes shifted off screen. Not very nice either.

> And also I think it would be nice if would be be possible to tear off a
> menu be dragging the title with button 1

I'd like to add that but I can't think of a way to continue
dragging once the old menu is unmapped and the menu window is
mapped.

> (I think this is more
> intuitive...), this is just a suggestion, I'm sure others have other
> opinions.

I'm not yet finished with the implementation yet. There are some
quirks that bug me. Any input is welcome.
