Tessa Lau <tlau_at_ofb.net> writes:
>
> > Does it matter wherher you start fvwm first or the application?
> > Do *any* key bindings work over this window?
>
> I always start fvwm first (from gnome-session). Do you think that
> starting Notes first and then fvwm later will change anything?
>
> Both the keybindings *work* as advertised. The problem is that in
> *addition*, Notes sees a Control or Alt keypress but not a corresponding
> keyrelease event.

I see it, but I don't know what Fvwm can do about it.

Notes is taking the Alt key as a keystroke.
Fvwm needs Alt plus another key before it thinks the keystroke
is for Fvwm.

If you do a binding like this:

Key Alt_L W N Move

Then Fvwm will get the keystoke, but I don't think thats
what you want. You want to use the Alt key as a modifier.

You can change the Alt key so that its a different modiifer
(like mod 1), but that will affect any program that wants Alt.
Most keyboards do have 2 alt keys, I see notes uses both of
them. You might want to remap one of them to a different
modifier.
