Hello.

The window-specific key bindings seem to work in a strange
way. Fvwm still catches the event, but, if the window does
not match the conditions, tries to send it to the application.
No wonder that it does not work sometimes, for example, for
xterm with "allowSendEvents: false". Application does not get
the event.

I think this should at least be noted in the documentation.
