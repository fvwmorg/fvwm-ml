On Mon, Oct 11, 2004 at 02:16:55PM +0200, Chris Rouch wrote:
> I have a sticky application, which has menu options for popping up
> non-sticky dialog windows. I have the following in my .fvwmrc
>
> + I ThisWindow ("Shortcuts") MoveToDesk
>
> With 2.5.10 this would mean that the pop window would always appear on
> the current desk, no matter which desk I'd viewed it on before. However
> with 2.5.11 it keeps popping up on the original desk.

Yes, you are right. MoveToDesk without an argument does not work
anymore. I'll fix it in 2.5.13. Meanwhile you can solve the
problem with

  + I ThisWindow ("Shortcuts") MoveToDesk 0 $[desk.n]

[snip]
