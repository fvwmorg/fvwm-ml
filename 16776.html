I think I have found a 'buglet', which although small is dead
annoying.

The situation:
        I have to use certain plotting windows, which are huge and awkward,
        along with an xterm (which must be of a fixed height should I
        wish to use certain commands and actually have scrolling *work*).

        There isn't quite room for both on screen, so I generally shove the
        plotting window up a bit, so that its titlebar is partly or
        mostly offscreen. This way I get to see the title of the plot and
        the x scale while still being able to type. Perfect.

The problem:
        Should I iconise this plotting window to get at something else
        for a moment when I uniconise it it appears on the page below the
        one I am on...so that part of the title bar is all that I get.

        It should of course be in that place, but one page up.

Any ideas? I have had this happen before but now that I am repeating the
same thing I realise why now and again certain windows appear on the
wrong page when uniconised. Their titlebars were off the top of that page.

Its minor but it definately does not seem right, this behaviour should
be fixed.
