Hiya Pete,

> I'm working on a Sun SPARCstation 5 using fvwm2 2.0.43 with one
> login/fvwm2rc and on an ULTRASparc 2 using 2.0.43 with a different login
> and resource file. On one login ( the first ) if I double click on
> (say) file.dat, only the
> "file" part is highlighted but on the second the whole "file.dat" is
> selected. I can't for the life of me work out which bit of the
> installation is causing this difference. I would like to have the
> "file.dat" behavior on both. Anyone point me in the right direction?

   I am not sure why the different behaviors but I'd guess it has
   something to do with the particular X server on each box.

   But as a work around you could get the "file.dat.and.some.more"
   highlighted if you:

      first: double cliek on 'file'
     second: as you do the above keep the mouse button depressed
               and continue dragging over the rest of the file name.

   You'll notice that i'll highlight each 'chunk' of the file.name.part.

   Personally I like this behavior since it allows me to easily
   rename a file using copy/paste over an xterm.

   Hope this helps,
