On Fri, 13 Mar 1998, Guido Bolognesi wrote:
> Patrick K. wrote:
> > . if this icon is ever moved out of that region
> > by the dragging it w/the mouse, it (the icon)
> > should be return to the IconBox area; like a
> > "sling-shot" effect.

If you have used a windows or mac computer, they have an "arrange icons"
function will take all the icons and place them in a nice little grid. This
is easier because these systems only allow icons on one size. I think what
Guido wants is something like this that happens every time an icon is moved,
and where the icons are moved as little as possible to snap to the grid.

It seems like this should be easy. Just find the code that gets run when an
icons is dropped and have it find the correct icon-grid position and move the
icon there.
