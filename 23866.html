1. I download FVWM 2.2 from http://fvwm.math.uh.edu/download.html. The file

2. Untar this file will create "fvwm-2.2" directory in "/usr/local/".

3. Execute "configure --prefix=/usr/local/fvwm > cfg.result" under directory
"/usr/local/fvwm-2.2".

4. Run "make > make.result" under directory "/usr/local/fvwm-2.2".

5. Run "make install > install.result" under directory
"/usr/local/fvwm-2.2".

6. Now there're three directories created in /usr/local/fvwm/: "bin",
"libexec" and "man".

7. Create my .xinitrc and .fvwm2rc according
/usr/local/fvwm-2.2/modules/FvwmSaveDesk/example/xinitrc and fvwm2rc.

8. Relog in with OpenWin session and screen turn dark with nothing on it. I
can only press left button of the mouse and select "Exit FVWM".

My workstation is ultra5, OS is SunOS 5.5.1. I don't know if I describe it

> First, the installation does not automatically install the system.fvwmrc
> file. You must do it yourself. Look at your Makefile or config.log to see
> where
> it should be placed (probably something like /usr/local/etc or
> /usr/local/lib/X11/fvwm or something like that), then copy it there.
>
> I also suggest you copy it over to your $HOME directory renaming it
> .fvwm2rc
> (but check your Makefile again to make sure you did not change the default
> name
> for it). This private copy enables you to modify your own fvwm
> configuration.
>
> As for xpm.h and rplay.h, configure simply check whether you have these
> insalled. If you do FVWM is built to use them if you don't then you won't
> have
> XPM support nor sound. These are not part of the FVWM package; you need to
> get
> them yourself. I think they're available from ftp.x.org but I'm not sure.
>
> Hope this help a little...
>
> Best Regards
