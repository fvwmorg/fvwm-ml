In addition to the StartsOnDesk style, use the SkipMapping style and,
if you've got a pager going you'll see the tiny window appear but you
dont' get switched to it.

Example from my .fvwm2rc:

Style "Netscape" StartsOnDesk 2, SkipMapping

Skipmapping (and it's inverse Showmapping) have been around a long
time, but I think it only made it to the man page recently but the man
page doesn't really describe it.

May I suggest an addition to the man page (under the Style section):

  SkipMapping causes new windows that do not start on the current desk
  (by way of a StartsOnDesk or "-xrm *Desk: foo" directive) to not be
  mapped until that desk is visited. This prevents switching to that
  desk when the window is opened. If SkipMapping is set as a default
  then ShowMapping can turn it off for specific windows.
