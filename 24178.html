Dominik Vogt wrote:
> Well, as a last resort you can override the icon with your own.
> Other window managers draw a background behind each icon. We have this as
> an enhancement request already, but didn't find the time to do anything.
> I could imagine this:
>
> Style * IconBackground <colour>
>
> and
>
> Style * IconBackground pixmap foo.xpm
>
> as the syntax.
>
Don't forget that since fvwm can now use non-default visuals that you
can't simply combine a client supplied icon pixmap with fvwm's graphics.
Fvwm will be using whatever visual the use asked for and the client
pixmap will use the default visual.

To draw anything with client supplied icon pixmaps you would have to
create a window with the fvwm visuals that fvwm could draw into and then
create a window with the default visuals that is a child of the previous
window to copy the icon pixmap into (and shape).
