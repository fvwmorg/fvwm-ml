I've tried the transient fvwm buttons solution, here is what I've found:

Because the button is pushed from within the context of GridButtons,
the MoveToDesk (and presumably similar commands) still require the user
select a window.

Also, the buttons start at +0+0, which I do not know how to fix :(

Either way, here is an example .fvwm2rc. If anyone can get it to work
better, I too would like to see it :)

# I use 8 instead of 16 desks... you may want to change it...
DestroyModuleConfig GridButtons
*GridButtons: geometry 128x64
*GridButtons: columns 4
*GridButtons: rows 2
*GridButtons: (Title "0", Action (Mouse 1) "MoveToDesk 0 0")
*GridButtons: (Title "1", Action (Mouse 1) "MoveToDesk 0 1")
*GridButtons: (Title "2", Action (Mouse 1) "MoveToDesk 0 2")
*GridButtons: (Title "3", Action (Mouse 1) "MoveToDesk 0 3")
*GridButtons: (Title "4", Action (Mouse 1) "MoveToDesk 0 4")
*GridButtons: (Title "5", Action (Mouse 1) "MoveToDesk 0 5")
*GridButtons: (Title "6", Action (Mouse 1) "MoveToDesk 0 6")
*GridButtons: (Title "7", Action (Mouse 1) "MoveToDesk 0 7")

AddToMenu Window-Ops2 "&move" Move
+ "to &page" Module FvwmButtons -transient \
GridButtons
+ "&resize" Resize
+ "r&aise" Raise
+ "&lower" Lower
+ "" Nop
+ "identi&fy" Module FvwmIdent
+ "" Nop
+ "(Un)s&hade" WindowShade
+ "(Un)&stick" Stick
+ "(Un)ma&ximize" Maximize
+ "" Nop
+ "&close" close
