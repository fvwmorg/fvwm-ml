>> I have several windows on one desk, some of them are stiky. I move one

>> non-stiky window still on the top of all. On the pager I see the stiky
>> windows on top of non-stiky ones... Here are the pictures describing what I

>> On the screen:

>> +----------------------------------------------+

>> Is it a bug or an intended feature and if it's intended how can I make

MG> It is not a feature, pager should show the real world, so it is a bug.
MG> Please always tell us the fvwm version when reporting problems (you should
MG> test over the latest stable or unstable one). When reporting bugs, it is
MG> very helpful if you show how it is possible to reproduce them. Thank you.

I upgraded to
[FVWM][main]: Fvwm Version 2.3.16 compiled on Apr 20 2000 at 13:11:17
[FVWM][main]: with support for: ReadLine RPlay Stroke XPM IMLIB Session
and the bug is still here. As for how to reproduce it, I think I described
it all well enough. In short:

1. On a desk where you have several overlapping sticky and not stiky
windows bring the not sticky window to the front.
2. Switch to another desk.
3. Return back to the desk where you started.
4. On the screen you will see the not sticky window on top of others, on
the pager you will see sticky windows on top of it.

Hope it helps.
