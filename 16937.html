>>>>> "Carlos" == Carlos Carreras <carreras_at_die.upm.es> writes:

Carlos> I am the manager of a few workstations and I would like to
Carlos> provide a basic fvwm configuration file for all the users in
Carlos> my network. My interest is in maintaining a menu structure
Carlos> which is updated everytime a new application is installed so
Carlos> it can be accessed from the workspace menu by
Carlos> everyone. Besides, each user should be able to configure his
Carlos> fvwm with his $HOME/.fvwm2rc.

Carlos> Is it possible to do this with fvwm2? How?

It should certainly be possible, and I'll be addressing some of these
problems in the next release.

Carlos> I have tried many things, but I have not been successful. Here
Carlos> is what is happennig:

Carlos> - If there is a $HOME/.fvwm2rc, no other configuration file
Carlos> seems to be read during configuration.

This is true, currently.

Carlos> - If there is no $HOME/.fvwm2rc, I can't find where the
Carlos> default configuration should reside to be read by fvwm2. It
Carlos> is not even clear how the file should be called: .fvwm2rc
Carlos> or system.fvwmrc. I have tried both names in several places
Carlos> but none of them worked:

Carlos> /usr/lib/X11/fvwm/
Carlos> /usr1/lib/X11/fvwm/ (I have X11 in /usr1)
Carlos> /usr1/lib/X11/twm/
Carlos> /usr8/lib/fvwm/ (fvwm is installed in /usr8)

It doesn't currently look for any system.fvwm2rc file. However, it
should be looking in 'FvwmDir' for .fvwm2rc if it can't find it in
$HOME.

For the next beta release I plan to make it look explicitly for
system.fvwm2rc in the 'FvwmDir' directory before it tries to read
on by default (haven't decided yet).

Carlos> - I also tried to call fvwm2 with the option -f "Read <file>"
Carlos> but that didn't work either. (What am I doing wrong?)

Hm. Now that should have worked...

Carlos> I have version 2.0.42. Is there anything else I can do? Thanks
Carlos> in advance!

Well, another way is you could have all of your users read a file at
the top of thier .fvwm2rc files that has all of your explicit menu
definitions and such:

  # top of ~/.fvwm2rc
  Read /path/to/your/system.fvwm2rc
  # rest of user's rc file follows, which may override these things...
