On Fri, 25 Oct 1996, Charles Hines wrote:

> I'm going to give it a try, and if it looks good I'll merge it (or a
> minor variant) into the next beta as the new SmartPlacement behavior.

As you will see from the code, the new CleverPlacement can be made to
behave the same as the old SmartPlacement with a flag, so it is possible
to have both behaviors without much difference in code size. It is up to
you whether you think is is desirable to keep the old behavior. If not,
then just remove the related code from CleverPlacement().
