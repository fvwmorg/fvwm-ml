> > > Anyway, I would like to have several Desks each with a 2x2
> > > Desktop (I think I have the terminology right), similar to
> > > what is at the top of the www.fvwm.org web page. I've
> > > searched the faq and mail archives, but am not sure what I was
> > > supposed to be looking for, hence this mail.
> > >
> > > Versions of stuff:
> > >
> > > fvwm-2.4.15-1
> > > fvwm-themes-0.6.2-1
> > > fvwm2-2.2.4-9
> > > fvwm2-icons-2.2.4-9
> > > Red Hat Linux release 7.1 (Seawolf)
> >
> > You are using either fvwm 2.4.15 OR fvwm 2.2.4.
> > You can't be using both.
> >
> > Fvwm 2.4.x reads $HOME/.fvwm/.fvwm2rc
> > Fvwm 2.2.x reads $HOME/.fvwm2rc
>
> Ok, I have
>
> FVWM version 2.4.15 compiled on Feb 20 2003 at 13:08:43
> with support for: ReadLine, Stroke, XPM, GNOME WM hints,
> Shape, SM, Xinerama
>
>
> and a file named ~/.fvwm/.fvwm2rc that contains
>
> Module FvwmPager 0 3
>
> DeskTopSize 2 2
>
> *FvwmPager: Label 0 Main
> *FvwmPager: Label 1 Web & Mail
> *FvwmPager: Label 2 Devel
> *FvwmPager: Label 3 Games
>
> But still only have the 3x3 Desker/Desktop that is labelled
> 'Main'.
>
> Also, when I use the mouse to choose Theme Management --> Current
> --> modules -> Choose Pager --> Desker apparently FVWM restarts,
> but nothing is different (except that the window close/open
> animation seems to occur three times).
>
> Again, Any hints?

Respond to fvwm_at_fvwm.org, not to me.

The stuff above will create a race condition between the
pager starting and the rest of the config being read.

Start the pager with:

AddToFunc StartFunction I Module FvwmPager
