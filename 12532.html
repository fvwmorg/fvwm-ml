Hi,

I was missing a feature to be able to create named tabbers instead of
numbered tabs. This is useful for auto swallow so you don't have to
depend on the ordering of opening the tabbers. Attached is a patch that
implements that. Unamed new tabbers still get number names as before
(the change was quite minimal thanks to using a hash in the original
code ;-)

The syntax is
SendToModule FvwmTabs createNewTabber <name>
*FvwmTabs: autoSwallow(Class|Name|Resource) <appname> <tabber name>

so for example to create a tabber named vim and auto swallow gvim into
it:

SendToModule FvwmTabs createNewTabber vim
*FvwmTabs: autoSwallowClass Gvim vim

If anyone else was missing that also, enjoy
