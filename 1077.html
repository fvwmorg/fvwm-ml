You are right, PassId option of FvwmEvent was introduced only in 2.3.
And in FvwmEvent 2.2.4 there is no alias support, only symlinks, so you
normally can only have one event module if you use this version.

It is possible to invent an example working with 2.2.4, but it may be
less relable. Replace "My Window" with your real window name.

# ----------------
DestroyModuleConfig FvwmEvent*
*FvwmEvent add_window SetGeometryForMyWindow

AddToFunc SetGeometryForMyWindow
+ I Next ("My Window") Move 200p 100p

# choose one of 2 invocations (the second is preferable)
Module FvwmEvent
#AddToFunc StartFunction I Module FvwmEvent
# ================

This works with 2.3.20 and should work with 2.2.4, I hope.
