    Thomas> Well: After finally happily running and properly stopping
    Thomas> fvwm-2.0.43 under Digital Unix 4.0 there is only one (well
    Thomas> actually minor) problem:

    Thomas> The title of the 'dxterm' (terminal-emulator - DEC
    Thomas> counterpart of xterm) is messed. When I set the title to,
    Thomas> say, "DECTerm" it appears as

    Thomas> 1]DECTerm

    Thomas> on the title bar! All other window titles appear correct!

    Thomas> So before I start digging around in fvwm2: Has anyone
    Thomas> stumbled accross the same thing or has anyone a clue what
    Thomas> could cause the problem?


Yep - at least I think so. I had a very similiar problem, but on a
much larger scale - because I use a Hebrew version of Motif on my
Digital Unix machine (3.2c). I had to work pretty hard, but I finally
discovered that the Motif I have uses something called compound
text. Apparently, the text is preceeded by some additional binary
information - which font to use, which direction, etc. Digital's Motif
knows what to do with this stuff, fvwm doesn't. Apparently, someone in
Japan developed a version of fvwm which handles this stuff, but all
the documentation is in Japanese, and I don't know how up to date it
is, so I rejected this solution. What I did in the end, was to hack
some of the fvwm code to look for certain values (the binary stuff) at
the beginning of strings, and to rip them out. It turns out that I had
to make only a few minor changes in only a few places. I'm not a C
programmer, so if I was able to do it, it couldn't have been too
tough. If you want, I can forward to you some of the messages I
received on this topic. Also, if you don't mind lousy C code, I could
make up some diff files, and send you those too.


Good Luck.
