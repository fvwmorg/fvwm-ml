I've just noticed some lossage using XEmacs-19.14b3 and fvwm-2pl35. If
anyone has a similar combination, can you please verify or discount
the following?

After selecting the Restart option of fvwm (Restart fvwm), XEmacs
redisplay of minibuffers is lost. \C-x\C-f doesn't echo in the
minibuffer, and cursor movement keys do not appear to
work. Fortunately, the (blind) input is still recognized so that I can
\C-x 5 2 to get a new frame which works.

Actually, the above happens with XEmacs-19.13 as well, so I'm inclined
to think it's a problem with fvwm-2pl35.

Does anyone else see this behavior after an fvwm Restart?
