Scott Raney wrote:
>
> > For me, I use pointer focus. I remember when I was first
> > introduced to the X-window system, I immediately adopted
> > the pointer-focus model. I have been using it since.
>
> You were probably not given any choice and accepted the default. This
> is a common event, and one we should recognize means that the default
> setting is important.
>

Well, actually as I remember it, click to focus was the default.
I guess because the window manager was motif based, and for motif
click to focus was the default. I just didnt like it. So I figured
out how to change this. XmNkeyboardFocusPolicy was one of the first
resources
I learnt.

> > I remember that, when I was a novice with this, I found it counfusing
> > having the pointer in one window and typing in an other...
>
> Actually, early research at Apple clearly showed that the opposite was
> more likely to be a problem, usually because a dialog opened. For
> example, the naive user's first reaction when a file selection dialog
> opens should be to attempt to type in a file name. This only works if
> the dialog gets the focus when it's opened. Unless you want to head
> down the slippery slope of pointer warping, this means explicit focus.

Well, I agree that its convenient when a dialog pops up, and the field
you want gets immediately the focus. But usually there is more than one
(text) field in this dialog. And the field you want is not always the
field
that has the explicit focus. For me, I found it always annoying
discovering
that I actually typed things into the wrong fields because I just forgot
to 'click'.
I know it works much better if you use the return and tab keys but
somehow I
always liked pointer focus better. Especially if the application has the
'highlight on enter' resource set. It makes a much more lively
impression.
But, anyway, thats what I did.. I'll look around at my work, see what
other
people are using.
