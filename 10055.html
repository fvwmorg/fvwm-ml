On Sat, Sep 20, 2003 at 07:22:02PM +0200, Johan Svedberg wrote:
> Hi!
>
> Just wondering if anyone knows what can cause applications not to listen
> to Style-commands? I'm having difficulties with the Debian package of
> MozillaFirebird. However, when I tried the binary on
> http://www.mozilla.org/ it works just fine so it must be something with
> the Debian build. I recall something that it has something to do with
> the fact that they built it against GTK 2.0?

Several possiblilities

 - Its windows are initially created without a title, class or
   resource and are renamed only after styles have been applied.
   Try to find our the original settings of the windows.
 - The application overrides the style settings. This is only
   possible for a few styles. Try disabling gnome and ewmh hints
   for the application.
