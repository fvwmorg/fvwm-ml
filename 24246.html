Quoting Paul Hurley (hurleyp_at_aston.ac.uk):
> Okay, here we go. fvwm-2.2.1.tar.gz downloaded and unzipped, wasn't sure
> where
> to put it (is this important ??) so put it in /usr/X11R6 .

It isn't important where you put it, as when you do the "make install" step
all files are put in their correct locations. If you don't want to keep the
source around (I don't), /tmp would be a sensible place to unpack it, otherwise
/usr/src is the usual place to put sources.

> <quote>
> checking for strchr... yes
> checking for sysconf... yes
> checking for uname... yes
> checking for div... yes
> checking for sigaction... yes
> checking for siginterrupt... yes
> checking for sig_atomic_t... yes
> checking whether setvbuf arguments are reversed... configure: error: can
> not
> run test program while cross compiling
> </quote>
>
> I presume this error stops configure from generating the makefile needed
> for
> make install.

Yes. The error means that the ./configure script couldn't execute a small
test program that it compiled (and assumed that it meant the compiler was
compiling for a different architecture). The ./configure script should have
created a config.log file with more information about what went wrong. If
you post it (or the relevant portions), I/we may be able to provide more
help.
