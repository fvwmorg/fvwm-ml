FVWM uses Xlib but not Xt, so it ought to be restricted to using Bool,
False, and True for binary logic. Boolean, FALSE and TRUE are defined
for use in the Intrinsics toolkit. As cks points out, Bool and
Boolean are different underlying types, int and char (or long)
respectively.
