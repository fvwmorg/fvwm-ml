R.Tanner said:
> 1. In my buttonbar I have a swallowed xterm which is very useful, but I have
> to click on it before I can enter text, whereas my normal xterms focus as soon
> as my cursor moves onto them. Is there a way to make a swallowed xterm focus
> like a window does?

As far as I know you'd have to allow the whole button bar to have focus just
like a window. I keep track of commands I'm always typing to start programs
etc and add a button instead. A swallowed xconsole takes care of error messages
and the like instead of a xterm then.

> 2. Is there a way for a swallowed object in my buttonbar to still act as a
> button. In particular I want buttons that make remote logins to other servers
> which have the load graph as their face. I can make each part work seperately,
> buttons for logins and others with swallowed xmeters but is there a way to
> combine them?
I honestly don't know about this one, I know you can do different things
with different mouse clicks...anyone else know? Its a nice idea.

  
> 3. Netscape doesn't seem to obey my configuration. I have smartplacement
> selected in my .fvwm2rc file but rather than find an empty area netscape
> allways appears in the top left corner of the screen. Is there a way to
> fix this?
No fix that I know of, it does this in my setup as well. Annoying isn't it?
Just start it with a given geometry perhaps, again quite simple if you have it on
a button.
 
> 4. I have four separate 'pages' setup in FVWM2 and want to be able to start
> a program in a different page to the one I am using. eg I am working in page
> 1 but want to be able to start a program in page 2. At the moment I am forced
> to start the program in my current page, then make it sticky, goto the page I
> want it, unstick the window, then go back to the first page again. Is there
> a better way?
This is why I'm always pressing for 'StartsOnPage'! :-)
You can write a function to go to a desk, start up program X, wait for X to appear,
and then return to page N. But thats irritating. I usually just start things iconic
now and slide them around in the pager using the middle mouse button whenever I have
time to open them and sort out what goes where.

I'd like to use StartsOnPage and SkipMapping to just have, say, Netscape start on another
page (it takes ages to start on my machine) and I'll move there when I've finished typing
some more email for example.

> 5. Are OpaqueMove and OpaqueResize implemented in FVWM2 or have the command
> names merely changed?
Erm...I've forgotten. Read the 'man' page, its in there.
