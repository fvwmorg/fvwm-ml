On Sun, Dec 09, 2001 at 03:25:47PM +0100, Dominik Vogt wrote:
> On Tue, Dec 04, 2001 at 11:50:01AM +0100, Paolo Perego wrote:
> > Hi guys. A stupid question about buttons for you. I've got a button on which I
> > swallowed xclock. As action I decided to run xmessage displaying the output of
> > "cal" command. If the mouse is over the button and I pressed "enter" xmessage
> > is displayed, double-clicking on it results in nothing, why?
> >
> > Fvwm:2.4.3
>
> Can you provide your FvwmButtons configuration lines?

Ah, I see. You are using a 'feature' that works only by chance
and is not properly implemented. Although the syntax allows to
bind actions to buttons with a swallowed window. For mouse clicks
this only works on the relief of the button, not on the
application window itself (because the application gets the click,
not FvwmButtons). With key presses it only works if the
application window does not want keyboard input itself. If you
had an xterm instead of the xclock, the key press would go to the
xterm (I think).
