I don't know if this is a bug or something that was intentionally done,
but I noticed a difference in behavior between the WindowList in FVWM-1
and in FVWM-2. Basically, in FVWM-1, the windows in the windowlist
would be sequenced in the order in which they were last accessed. This
was very convenient because I had bound some keys to bring up the
WindowList and then I could use keystrokes (j or k) to easily go up or
down to switch between menus. So, if I was working between two windows
and wanted to toggle between them, all I had to do is bring up the
WindowList (i.e. Alt+F1), and then press j-j to switch to the previous
window, then enter to select it.

This "convenient" reordering of the windows in the WindowList is not
present in FVWM-2, and maybe that was by design. I just think that it
would be more convenient for the windows to reorder in the order in
which they were last accessed, especially when there are many windows
open.
