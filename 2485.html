On Wed, May 09, 2001 at 09:40:46AM -0400, Norman Walsh wrote:
> Apologies if this is a FAQ; I didn't see it in the FAQ, but I might
> have missed it.
>
> I notice the following behavior in fvwm menus: if a popup menu is
> shifted "up" because you're too close to the bottom of the screen;
> clicking on the button that pops-up the menu has the effect of
> immediately selecting the item that appears where the click
> occurred. (Because the shifted menu has placed the menu title bar
> "above" the location of the pointer.)
>
> Is it possible to change the timings or menu behavior so that a second
> click is *always* required to select a menu item?
>
> Be seeing you,
> norm
>
> P.S. Why, you may ask, doesn't he just move the window up so this
> problem doesn't occur? Well, it's because I'm running fvwm2 on my iPAQ
> and it has a really tiny screen. :-) It also has a touch-sensitive display
> which makes it really easy to click but considerably more tedious to
> click-and-hold or click-and-hold-and-drag.

Although this is not a problem with the current 2.3.x betas you
will probably want to replace the 'Popup' command with the 'Menu'
command in your config file. If you use Menu to open menus they
do not pop down when you release the button.
