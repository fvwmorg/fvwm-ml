On Tue, 18 Mar 1997 jmknoble_at_pobox.com wrote:

> i've got a `patch' for gradient-ized icons that i've been using since
> fvwm-1.24r ... it's called the 3d-pixmaps collection. not that hard to
> use, they're all 56x46, and they're all decent-looking. it's easy
> enough to use a tool like ImageMagick to make mini-icons and to get the
> number of colors used to <=64 (in case you're still using an antiquated
> video card that only gives you 8bpp). why do we need another patch for
> gradient icons when these things are available? that kind of stuff just
> adds bloat to fvwm.

I don't think so. I also wrote my own patch and it only added a few dozens
of lines of code to borders.c. That's not much in the way of bloat.

And yes, I tried the CD pixmaps collection: and it doesn't meet my needs.
Why? because some applications provide their own icon windows, and I don't
want to replace these. And I was icons-in-a-box (never mind the gradients)
because this is the behavior provided by DTWM (and 4DWm, and..) I aim to
make my CDE emulator (FvwmCDE) as CDE-like as possible, and this is just
one of those things..
