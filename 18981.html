        Hi all, bussy people

I just observed for the tenth or twelfth time a strange (and annoying)
comportment of XEmacs occuring after the 2.0.44 install. I hadn't the
time to pay attention to it though not more the time to try to trace
it down (or even isolate it). My message for now is to try to see if
somebody else pinned-down some similar bug:

If I have an (or multiple) XEmacs window(s) opened and I do a Restart
of fvwm2, I get stuck with those windows because they don't get
anymore input. I mean they can get focus, but mouse click/commands and
keyboard input seems not active. STRANGE. The only work around (really
VERY annoying) is to close all XEmacs windows and to start a new
session.

With 2.0.43 this dosen't exist. There was a similar (but in some sort
smaller) problem with GNUS, which did not display sometimes if it was
on another desktop than the visible one during GNUS start. I think
this problem was largely discussed in fvwm and XEmacs groups but I
missed the solution.

Howevewr, I would like to know if anybody else noticed this
comportment.

Thanks a lot

                                Friendly yours, Cristian
