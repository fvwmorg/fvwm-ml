Hello,

I use 2 Xinerama screens (with NVidia's Twinview option). I have a
FvwmButtons panel that slides on to one screen if I click on an icon.
The icon is supposed to be positioned in the top right of my second
monitor, using the line: '*BarButtons: Geometry 30x30-5+0_at_0'

This works provided the mouse pointer is in the first screen when FVWM
(re)starts, but if the pointer is in the second screen then the icon
disappears off somewhere else never to be seen again.

As far as I can understand the '_at_0' in the geometry should put the icon
in the first screen, and I can't see how that has anything to do with
the position of the pointer. Can someone explain what is happening?

Thanks,

Andy
