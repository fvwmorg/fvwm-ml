On Thu, Nov 25, 1999 at 05:17:14PM +0100, josvanr_at_xs4all.nl wrote:
>
>
> Hi
>
> I have a few questions concerning the use of separate files
> for configuring fvwm2.3.10.
>
> 1) Is the scope of functions (AddToFunc) in files global, or
> local to the file

Global. Fvwm doesn't care where the commands come from.

> 2) And if I use M4 preprocessing, is this local a file, or
> global: ie if I import a file using
>
> FvwmM4 TheFile
>
> are the M4 definitions in the root file also applicable
> to TheFile, and can definitions made in TheFile be used
> in the root file?

They are local. To get then in the main file you have to
process the whole config file with FvwmM4 and 'include' the
sub file with the proper M4 preprocessing directive (I don't
know how it's called).
