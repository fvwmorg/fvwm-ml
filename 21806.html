On 24 Feb 1998, Kai Grossjohann wrote:

: >>>>> On Mon, 23 Feb 1998, James J Stewart said:
:
: James> i'm using fvwm1, and i'd like to have it perform
: James> SmartPlacement on my ICQ, but if i try to define it as a
: James> style using the window title (my icq#), it tells me that
: James> SmartPlacement is a bad style command.
:
: AFAIK, fvwm-1.X doesn't grok SmartPlacement; it was something added to
: fvwm-2.0.40+.

Zzzzzzzzzzzttttttt! Thanks for playing. fvwm-1.2x groks
SmartPlacement as a top-level config directive, but not as a style, e.g.:

    # (in .fvwmrc)
    # This is all-or-nothing; you can't have SmartPlacement for
    # some windows and `active placement' for others.
    SmartPlacement
    RandomPlacement

while fvwm2 groks this:

    # (in .fvwm2rc, or whatever you've chosen to call it)
    # You can restrict this to only the windows you want in fvwm2,
    # unlike in fvwm-1.2x.
    Style * SmartPlacement
    Style * RandomPlacement
    Style

--jim
