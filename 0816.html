greg shebert <gshebert_at_efs-us.com> writes:
> hi there...
>
> i know that fvwm already manages two environment variables...
> DISPLAY
> DISPLAY_HOST

Fvwm may read those variables. I don't think "manage" is the
right word.

> i would like to be able to set an environment variable via fvwm menu
> functions and have this variable passed on to fvwm's children as well
> anyone know how i would go about doing this?

"Normal" UNIX behavior is for a parent process to pass all environment
variables to its children. You can export variables before you invoke
fvwm to get the variables to the process fvwm starts.

You haven't mentioned the release of fvwm you use, but
fvwm has had the "SetEnv" command for a long time,
check the man page for your version.
