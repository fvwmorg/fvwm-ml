There should be a libXpm.so somewhere in this picture.

Normally the symbolic link would look like:

MyMachine> ls -l /opt/public/lib/libXpm*
lrwxrwxrwx /opt/public/lib/libXpm.so_at_ -> libXpm.so.4.10
-rw-r--r-- /opt/public/lib/libXpm.so.4.10

If there is no file, "libXpm.so", I think the rest of the bug
report falls into place.
