I downloaded the fvwm-snap-20010510.tar.gz snapshot, rebuilt, and indeed it
is fixed :-)

A couple of minor comments on the snapshot - if it's of any help:-

1) The build complained about missing libs/fvwmlib.h. I replaced libs/ with
../libs/ in Makefile.in to fix it.

2) I noticed the new comment in INSTALL.fvwm:-
     Cygwin users: try "make -i -k" and "make -i -k install" to work
     around the Cygwin problem, not supporting POSIX's mkfifo;
     FvwmCommand will be skipped.
In fact this is not necessary (of course it does no harm) in the latest
version of Cygwin, which includes a dummy version of mkfifo (just returns
-1, and errno=88=ENOSYS="Function not implemented"). Of course FvwmCommand
doesn't work, but it now builds OK.
