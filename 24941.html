Hello,

I'm wondering why SaveUnders was removed from FVWM. It's there in 1.24r,
but not in the current version. This is a step backwards, IMHO, because
SaveUnders can sometimes be quite useful, particularly in applications
which have 3D graphics windows which are expensive to redraw. Because of
the lack of SaveUnders in FVWM, there are two expensive redraw operations
(~1/sec) apeice in my application everytime a dialog is popped up.
neither AfterStep, nor twm suffers from this problem, but FVWM is my WM of
choice.

Please add the SaveUnders code back into FVWM. It's a simple code change,
and well worth having the ability to do so.

Configuration: RH Linux 5.2, fvwm 2.0.46, XF86 3.3.5.

P.S. Please CC any replies to me, since I'm not on this list.
