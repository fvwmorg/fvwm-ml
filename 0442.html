  what's going on:

  X window system: a server that can display graphics (primitives like
windows, lines etc.), it does not care about desktop (window
borders/icons/menus etc...)

  widget libraries: buttons, menus, icons, scroll bars and other parts
of user interface

  window manager: program that manages windows (provides borders, means
for user to move/resize windows, ...)

  desktop environment: higher level, usually provides combination of:
basic programs, widgets with common look/feel, window manager etc...

  gnome is desktop environment, it can use one or another window manager

  fvwm is window manger. it requires X server running.

  check the gnome docs on how to change window manager.

  to try if fvwm is working:

  from virtual console type:

setenv DISPLAY :0.0 # csh, tcsh, ...
export DISPLAY=:0.0 # sh, ksh, bash, ...

X&

  this runs an X server (plain)

  switch back to virtual console and run

fvwm&

  and you should get the fvwm running (switch back to X server window to
see it)... try to click in the root window, you should get some menu.

  depending on how your X is set you might have to be root to run X,
although it might run Xwrapper by default...
