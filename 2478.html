On Wed, May 09, 2001 at 10:47:57PM +1200, Cameron Kerr wrote:
> Hello, I just downloaded, comp'd, and installed fvwm 2.3.32, so
> I could use my scroll wheel on FvwmButtons.
>
> However, I've now got this annoying problem of a default builtin
> menu when I scroll on the root window. (I'm using my old config
> file).
>
> I've tried removing that pesky thing with
>
> Mouse 0 R A -
> Mouse 1 R A -
> Mouse 2 R A -
> Mouse 3 R A -
> Mouse 4 R A -
> Mouse 5 R A -
> DestroyFunc FuncFvwmRaiseLowerX
>
> which is similar to what the man page for fvwm2 suggests,
> but no such luck. How can this be done?

Try

  Mouse 0 R N -

The default binding is

  Mouse 0 R N Menu MenuFvwmRoot

To remove a binding the context and modifiers have to match the
original binding command. What part of the man page do you refer
to? Is there any outdated or incorrect information in the man
page?
