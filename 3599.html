Which version of fvwm do you have? If its any other version but
2.4.x, please upgrade and see if the problem persists. The were
some problems in earlier versions when restarting from any page
other than (0, 0), but these should be fixed in 2.4.x. Usually,
fvwm should start on exactly the same page as the old fvwm was
on when you restarted. If you'd like to always start on page
(0, 0), try restarting with

  Restart --dont-preserve-state

Or you may add

  AddToFunc ExitFunction
  + I GotoPage 0 0

to your config file (if the function already exists, just append
the second line to it).

Just to point out the terminology:

  +------+------+------+
  | | | |
  | | | |
  | | | |
  +------+------+------+
  | | | |
  | | | |
  | | | |
  +------+------+------+
  | | | |
  | | | |
  | | | |
  +------+------+------+

The whole work area is called a "Desktop", and is divided into
"Pages" (3x3 in your case). The visible portion of the desktop is
called "Current Viewport". This usually - but not necessarily -
matches one of the pages.
