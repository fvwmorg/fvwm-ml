%% Pete Forman <gsez020_at_compo.bedford.waii.com> writes:

  pf> You need to set a "Reply-To:" field if you want replies to go to
  pf> the list rather than yourself. This is documented in RFC 822
  pf> section 4.4.3. I see that you have a "Mail-Followup-To:" but this
  pf> is non- standard.

No, you need to use the "followup" command in your mailer to send to all
recipients, instead of the "reply" command to send only to the author.
"Followup" will send to fvwm-workers, since it's on the CC line.

Certainly if Dominik never wanted to receive private replies he could
set his reply-to, but people sending patches, etc. should reply to the
list regardless.

  pf> Perhaps the majordomo could be configured to add a reply-to if none
  pf> has been set by the original poster.

No. Or rather, majordomo can be configured that way, of course, but it
is The Wrong Thing To Do.

  http://garcon.unicom.com/FAQ/reply-to-harmful.html
