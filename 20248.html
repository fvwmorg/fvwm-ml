Hi,

I want to ask whether there are any plans to make fvwm compliant to
X11R6's session management. I think that there is a great
perspective for new functionality when fvwm was extended. New modules
could take advantage of the additional information xmsp provides (unique
identification of windows across sessions, grouping of windows according
to application instances,...). Even the session manager itself could be a
module which might use the module interface to restore geometry of windows
when sessions are restarted.

We are currently developing a similar module for a research project which
will enable users to manage task-related groups of windows instead of
single windows. Also, we have patched or will patch our local copy of fvwm
to inegrate multi window operations like "move group of windows", "resize
group of windows" etc. So I'd like to ask the members of this list what
they think about such kinds of modifications and what feature requests for
task-related window manipulations they might have.

Due to our local patches to fvwm (which are for version 2.0.43), I cannot
promise that we are able to provide a module for the baseline
version of fvwm but would appreciate any comments.


Thanks,


Stefan.
