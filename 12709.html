Now I found a bug.
Say you have a normal FvwmButtons as following:
####
Style FvwmDockUt* StaysOnBottom, NeverFocus
DestroyModuleConfig FvwmDockUt: *
*FvwmDockUt: Geometry 128x28+56-25
*FvwmDockUt: Font none
*FvwmDockUt: BoxSize fixed
*FvwmDockUt: Frame 0
*FvwmDockUt: Padding 0 0
*FvwmDockUt: Colorset 12
*FvwmDockUt: Rows 1
*FvwmDockUt: Columns 12
*FvwmDockUt: (1x1, Title "1")
*FvwmDockUt: (2x1, Title "2", Icon Ut/configure.png, Action (Mouse 1) 'Menu
MenuFvwmConfig Rectangle +$left+$top 0 -100m')
*FvwmDockUt: (2x1, Title "3", Icon Ut/gphoto.png, Action (Mouse 1) Exec exec
gvidcap)
*FvwmDockUt: (2x1, Title "4", Icon Ut/aim.png, Action (Mouse 1) Exec exec
LumaQQ/lumaqq)
*FvwmDockUt: (2x1, Title "5", Icon Ut/Calculator.png, Action (Mouse 1) Exec exec
xcalc)
*FvwmDockUt: (2x1, Title "6", Icon Ut/office.png, Action (Mouse 1) Exec exec
texmaker)
*FvwmDockUt: (1x1, Title "7")
Module FvwmButtons FvwmDockUt
########################

When you init fvwm, FvwmButtons won't display the icons.
A Restart will turn on the icons.

The above setting works fine without the hovericon patch.
So I guess something is wrong with the hovericon patch.
