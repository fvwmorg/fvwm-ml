The AIX xmperf program will not run under fvwm 2.0.45 & AIX 4.2.
<snip>
> It runs under motif, and reportedly other window managers.
> Is there any info I can provide that would help resolve this?

Welcome to fun with xmperf and fvwm. Let me start by saying I'm running
a version of AIX 4.1 and have not tried this on 4.2. (#include
<std_disclaimer.h>) There are a couple of things that work on 4.1 that
you might try:

1) Run with the -u option. This is pretty painless and seems to be the
only way I can get xmperf to work on 4.1. This option switches to popup
menus instead of the pulldown ones.

2) You can't run a version of fvwm compiled on AIX 3.2 on 4.1 and have
xmperf work. I'm guessing there is some incompatibility in the X11
libs. What's weird is that xmperf is the only program I know of that
has this problem. So if this is the case, you'll need to rebuild fvwm
on your 4.2 system.

Good luck,
-Bill
