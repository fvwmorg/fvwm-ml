On Fri, 23 May 1997, Morsy Cheikhrouhou wrote:

> > On Fri, 23 May 1997, Morsy Cheikhrouhou wrote:
> >
> > > 1. Our system administrator is maintaining a .fvwmrc for all the users who
> > > have links to that file. Thus, everyone has the same fvwm configuration which
> > > is automaticly apdated whenever needed. The problem is that I would like to
> > > have my own fvwm configuration, so I tried to create a .fvwmrc+ file in my
> > > home dir. But that doesn't work as it does for .bashrc+ and .emacsrc+ for
> > > example. Any tip to make fvwm take into account my .fvwmrc+ file ?
> >
> > assuming the system RC file is named system.fvwmrc, try naming your file
> > .fvwmrc ... If not, you can try naming it .fvwm2rc ...
>
> Thanks for everyone who tried to help, and sorry for being not clear. What I
> wanted is to keep the original .fvwmrc file, which is a link to the system rc
> file, and to define a .fvwmrc+ file, in which I can put my own preferences.
>
> >From Martin's suggestion, i had a hint: I will try to call fvwm in this way :
> fvwm -f "cat ~/.fvwmrc ~/.fvwmrc+"
> I didn't try that yet, and I don't know what happens if there is conflicts
> within the rc and the rc+ files.

Well, this probably won't work unless you copy the system default to your
home directory. Your best bet is to talk to the system administrator about
adding a line to the system fvwmrc that reads a users .fvwmrc+ file ...
Otherwise, just copy the system default file to your directory and modify
it to your liking.
