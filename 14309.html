I've installed 2.0.35 and it looks good so far. Other than getting it to
install in a $LIB/fvwm2 directory, which I ended up doing by hand, and
the conversion of my old rc file to the new format, I had no problems
with Linux and XFree 3.12.

I've got one problem though that I'm still stuck on, and a couple of
ideas for the 2.0 suggestion box.

First, the problem, mapping to a key that doesn't have a name. A long
time ago on this list, someone mentioned they'd mapped the cntrl-grey
insert-home-pg.up block of keys to GotoPage commands for a 2x6 pager. I
liked this idea, because those keys are easy to reach with your thumb on
the control key for fast window changes. However, I remap the delete key
to ^D with Xmodmap keycode 107 = 0x04. Unfortunately, this seems to
leave me with no name to map the delete key with, and fvwem won't map to
the keycode number. Any suggestions?

As far as suggestions go, two things I'd like to see in fvwm would be:

First. a per page strategy for window placement. Currently , I use
SmartPlacement and RandomPlacement. This works fine with the first few
windows I open, which are tiled to the NW gravity point. Then, if I goto
a different page, the first window is placed in the open NW corner, but
subsequent windows are opened at random, as fvwm seems to think it's
exausted the tiling space, even though the new page in the pager only
has one window open. Turning off RandomPlacement just means I have to
hand place every window after the first one open, which I'd like to
avoid. Maybe a TiledPlacement keyword would be appropriate. As well, it
would be nice to be able to define the space left between windows when
they tile. Currently, I find the windows are tiled a bit tightly, and I
end up adjusting them by hand.

Second, an alternate drawing strategy for FvwmPager. Maybe this is a
bug, but I discovered I can swallow FvwmPager 0 0 and have a 6x2 pager
showing in FvwmButtons, but I can use Desk 0 1, Desk 0 2, etc to jump to
new desktops and open windows. But, they don't show up in the pager
window, which continues to show Desk 0 0. I think it would be quite
useful to exploit this, to avoid the pager window gobbling extra space
on my desk, by having FvwmPager be able to map the active desk into the
same window.

I see fvwm 2 now allows xlock -mode swirl to work properly, which was a
pleasant surprise.
