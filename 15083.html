James LewisMoss <moss_at_usceast.cs.sc.edu> writes:
> > Hi,
> >
> >
> > Is it possible to force positioning of iconified windows in a manner
> > similar to HP VUE? I'd like to force icons to be in specific areas of
> > the screen, in neatly arranged columns. For this, I would have to use
> > specific icons for all windows i.e. all icons are the same size. If a
> > user tries to move that ic on to an "invalid" position on the grid,
> > the icon would be placed on the closest "valid" position. A
> > snap-into-place type thing if you will.

>
> Yes. use a line like
> Style "*" IconBox +foo +bar +blah +yackety
>
> (look in fvwm man page for parameters).
>
> Along these same lines I've got a question (and I'm betting on a no answer).
>
> I like having the styles for each window in a central place for each
> window. But if I use FvwmIconBox I have to comment out all the global
> style "blah" Icon some.xpm
>
> and add the the same thing below
> *FvwmIconBox "blah" some.xpm
>
> Is there any way to get the FvwmIconBox module to grab the icon setting
> from the fvwm style lines? And is there any way to tell fvwm to not
> display icons even when there are Icon style parameters?
>
> I've looked but haven't found.

I think I can help with the second part. I use this to turn on ICONBOX
on the fly, (without removing the "sytle icon" specifications:

AddToFunc ConfigIconBox "I" module FvwmIconBox
+ "I" Style "*" NoIcon
+ "I" Recapture

AddToMenu "Module-Popup" "Modules" Title
+ "IconBox" ConfigIconBox
...

(It is the order of execution of the style commands that is important.)
