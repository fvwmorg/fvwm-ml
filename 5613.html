Hi,
I'm running FVWM 2.5.2 on a RedHat 6.2. with fvwm-themes-base-0.6.0-1

When starting fvwm via fvmw-themes
(entry in .xintrc:
exec /usr/bin/fvwm-themes-start
) I see this error message on the console:

"[FVWM-Themes]: Starting FVWM under the main fvwm-themes session
FvwmTheme is obsolete, see the Colorset section of the fvwm(1) man
page [FVWM][executeModule]: <<ERROR>> No such module 'FvwmNetHints' in
ModulePath '/u sr/libexec/fvwm/2.5.2'"

1:
How do I get rid of this error?

2:
Is there a chance that there will be a new RPM for
fvwm-themes-base to get rid of the error?

To help myself for the first error ("FvwmTheme is obsolete") I changed
.fvwm/themes-rc and .fvwm/themes/personal/colors according to man fvwm
(Colorsets):

#######################################################


In the 2.4.x versions a special module FvwmTheme was
       introduced to manage colorsets. The FvwmTheme functional-
       ity was moved to the core fvwm, so this module became
       obsolete. Some additional information about colorsets may
       be found in the man page of FvwmTheme(1), although it is
       obsolete now.

       Old syntax:

         DestroyModuleConfig FvwmTheme: *
         *FvwmTheme: Colorset 0 fg black, bg rgb:b4/aa/94
         *FvwmTheme: Colorset 1 fg black, bg rgb:a1/b2/c8

       corresponds to the new syntax:

         CleanupColorsets
         Colorset 0 fg black, bg rgb:b4/aa/94
         Colorset 1 fg black, bg rgb:a1/b2/c8

#########################################################
to no avail so far ...

The FVWM-Beta is running very well here (i.e., the errors I got so far
were probably due to my own nonsense and I was able to fix them ..)

Help :) ... and thanks in anticipation
