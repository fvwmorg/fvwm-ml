Olivier Chapuis wrote:
>
> Dominik Vogt wrote:
> > Then you should use FvwmButtons, not FvwmPager. You can easily define
> > buttons looking the way you like there. E.g.:
> >
> > *FvwmButtons(Title Desk 1, Action 'GotoDesk 0')
> >
>
> This a very good idea that I tried at once !
> (Desk 0 x seems better than GotoDesk x :o).

        Ah, right. This depends on which version of FVWM you're using. The Desk
command has been renamed to GotoDesk in the current development sources
(for consistency with the GotoPage command); so if you use one of the
newer development snapshots, or the next major release, you should use
GotoDesk. Otherwise (for 2.2 and earlier), the command you need is Desk.
So, you're both right.
