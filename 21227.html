Mike Stutz wrote:-

> The problem: Whenever I run X in any depth higher than the default 8pp (for
> instance, "startx -- -bpp 16"), the window outlines that are normally drawn
> when you move a window disappear. Is there a way to get them at these higher
> depths?
>
> System details: I'm running Debian Linux 1.3x, 2.0.30 kernel, XFree86 3.3,
> S3 server (Diamond Stealth Video Vram 4mb card), FVWM2 2.0.45-BETA as
> installed by Debian's dselect program.
>

I have a 2Mb S3 and Xfree3.2, with fvwm 2.0.46.
Put the following into .fvwm2rc :-

# Need this to get window outlines with bpp > 8.
XorValue -1

This will cause reverse-video outlines to appear when you drag windows larger
than
the OpaqueMove size defined in your .fvwm2rc.

Hope this helps,
