On Mon, 21 Oct 1996, Rob Whapham wrote:

> I work for a systems integration software company, and we have one of
> practically every Unix box in house. I consider Fvwm to be far superior to
> most of their window managers and would like to install it wherever possible.
> I've been able to get it up and running on all of our Solaris machines
> (Sparc 2.3, 2.4, and 2.5; and x86 2.4, 2.5), but all of the other OS's have
> session managers which automatically start their window managers (i.e., you
> don't have to type xinit or openwin from a command prompt). I know this
> doesn't pertain directly to Fvwm, but can anyone give me any help in setting
> up the following OS's to use Fvwm:
>
> Irix 5.3
> HPUX 9.04 & 10.01
> AIX 3.2 & 4.1
> OSF (Digital Unix) 3.2

For Digital Unix use the following:

1.) Click on "Options->Window" in the session manager menu.
2.) Under the Window Manager Title, choose "Other" instead of "Default"
    and enter the path to the fvwm executable into the entry field.

Log-out and log-in again.

This setting just causes the following Resource to be set in the
.Xdefaults of the User's Home directory:

      DXsession.windowManagerName: /usr/lib/X11/fvwm/fvwm

If you want to make it the default for all users, just put it in

          /usr/lib/X11/app-defaults/DXsession

Hope that helps -- Tom

P.S: Beware, though, that this setup requires a patch to FVWM 2.0.43.
     Without the patch, you can have dead FVWM processes lying around
     after logging out. These processes eat up all CPU time they can
     get.
