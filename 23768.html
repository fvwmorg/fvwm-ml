On Mon, Mar 08, 1999 at 10:07:57AM -0500, chabe04_at_cai.com wrote:
> Hi all,
>
> I just upgraded my X11 server to R6.4. I also just upgraded my fvwm to 2.2. I
> also run on Solaris 2.5.1 (if it ever has an effect).
>
> My question is this:
>
> When I click the root window (to get the "Utilities" menu say), it pops up
> ok. Note that I click and release the button x. Now if I click and release on a
> submenu entry, it pops up ok BUT the parent menu disappear. If I change my mind
> I cannot go back to the parent and must re-click the root menu to get the menu
> again. It's annoying.
>
> Does anybody have any idea why that is? I checked with fvwm2.0.45 and it does
> the same thing when I know that running it above X11R5 (the stock X11 server
> with Solaris 2.5.1) worked fine. I know it may be an X11R6 question but Im
> asking just in case somebody could help me.

That's because it is coded this way inside fvwm. I can't image why this should
depend on the X11 revision, but if it works differently with X11R5 then it's
a bug (with X11R5, not X11R6). Usually submenus are popped up automatically so
you don't have to click to open it (assuming that you use the 'Popup' builtin
for the submenus, not 'Menu'). I see that this is a problem when you have
turned off automatic popup explicitly (which I will probably fix with another
option when I've go the time).

Could you please post your MenuStyle lines so I can judge what exactly
you are trying to achieve?
