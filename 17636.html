>>>>> "Ron" == rgo <rgo_at_drmail.dr.lucent.com> writes:

Ron> I recently upgraded to fvwm2.0.43 from fvwm2.0.42,
Ron> and I'm having trouble getting the same behavior with
Ron> the FvwmIconMan module as I had before.

Ron> I want to have mouse button 1, when clicked on
Ron> an entry in the FvwmIconMan table, deiconify the window
Ron> if necessary, switch focus to
Ron> the selected window, and warp the cursor to that window.

Ron> In fvwm2.0.42, with a patch to the focus code,
Ron> I could do this via the following line in my .fvwm2rc file:

Ron> *FvwmIconMan*action click1 Iconify -1, Focus 1

Ron> Is there a way to get this same behavior in fvwm2.0.43?

Ron> Is a patch needed?

Read the ChangeLog and you'll see that the Focus behavior has changed
to no longer warp the cursor (coincidentally, it was at the request of
FvwmIconMan's author, among many other people) and that was part of
what his patch covered.

Basically, you should be able to get the same behavior by calling a
function instead of the builtin Focus, which could be defined like so:

AddToFunc FocusAndWarp "I" Iconify -1
+ "I" Focus
+ "I" WarpToWindow 2p 2p
