Is it possible to tweak the placement algorithms in 2.2.4 short of
hacking the code? Using

    Style "*" RandomPlacement, SmartPlacement

I'm getting results that seem slightly odd. For example, creating
six 80x30 xterms (FvwmIdentify reports Width: 499, Height: 394,
Boundary Width: 6) starting on a blank page, they appear at:

    +2+1 +517+1 +1032+1
    +1+428 +516+428 +1031+428

leaving two-pixel gaps between them. I would prefer, instead:

    +0+0 +513+0 +1026+0
    +0+425 +513+425 +1026+425

to leave no visible gaps.

After I manually tighten up the arrangement to my preferred layout,
if I create an 80x24 xterm (Width: 499, Height: 316, Boundary Width:
6) it lands at +1+855, which causes its bottom border to spill onto
the next page down by two pixels. The decision to align at the
bottom instead of to the windows above is fine, but to be flush with
the bottom the Y placement would need to be +853 instead of +855.

In case it matters, this is on Red Hat 6.2.
