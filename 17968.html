You could split you .fvwmrc into separate file and use FvwmM4/Cpp to
reassemble them at startup time.

An idea for the Fvwm hackers would be for fvwm to look for an icon in the
PixmapPath that matches the window name (or icon name) and use that by default.
That way you would only have to copy the icon you want to the appropriate file
name and fvwm would use it.
