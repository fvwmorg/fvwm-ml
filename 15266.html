On Wed, 24 Jan 1996, David Kennedy wrote:
>
> Does anyone out there still use Sun's Mailtool with Fvwm (preferably Fvwm2)?

Nope, but ....

> If so could they drop me an email regarding fixing its behaviour where double clicking on
> a new message drops an icon instead of an open view message pane?
>

I still have Mailtool on my system and it does not exhibit this
behaviour. Just a curious question, do you have a line like this in your
.fvwmrc?

  Style "Mailtool" StartIconic

If so, this would not only start the initial application iconic, but it would
also open any new windows spawned from the application in the iconic state.
This is because they all share the "mailtool" resource and class name.

If not, look for any Style definitions with the StartIconic option and
make sure they are not getting matched up with Mailtool somehow.

Also, mailtool does start iconic on my system by default, even without
specifying it as a Style.
