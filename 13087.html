Did you read the Dan's reply? It answers this. It's very simple, really.
I am not sure why this unproductive thread should be continued still.

If you are an author, you have a say on your part of the code in case the
license ever changes. No, the fvwm license is not going to change any
time soon, because the developers agree about the current license.

If you see anyone violating the GPL, write to gnu.org and fvwm-workers.
FSF should be informed, so their lawyers help to deal with this, since
the fvwm authors don't have free software lawyers on their own.

If you disagree with the Free Software philosophy and GPL specifically,
don't develop fvwm, noone forces you to. You may still run it freely.

If you want to simply rant about GNU GPL, these lists are not the place.

Here, "you" means "someone", this is a generic, not a personal answer.
