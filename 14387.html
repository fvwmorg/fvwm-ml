>>>>> "mb" == Mark Borges <mdb_at_cdc.noaa.gov> writes:

mb> I built pl36 today, and had two SEGV's in the last couple
mb> hours. So I've gone back to pl35 for now.

Bummer.

mb> I'm not exactly sure what is triggering the problem. Here's the
mb> contents of my xsessionlog:

mb> ------------------------------------------------------------------------------
mb> 14975 Segmentation Fault - core dumped
mb> X connection to hpx06:0.0 broken (explicit kill or server shutdown).
mb> ------------------------------------------------------------------------------

mb> and the X terminal reports a fatal I/O error (broken pipe) or
mb> somesuch.

mb> The only modules I use are FvwmPager and FvwmButtons. Were any
mb> changes made in these w.r.t. pl35?

I made a change to FvwmButtons to prevent it from doing nasty things
to the X server if you killed it and it had swallowed modules in
there, but that's about it.

mb> Is it possible for a module to cause fvwm to crash, or is the
mb> problem necessarily in fvwm itself?

I don't think that a module can really crash fvwm, since they only do
socket communication and are actually to separate processes. But
module communication could trigger a bug in fvwm.

mb> At the time of the crash, a new window was attempting to be
mb> created. I compiled with SUNWspro cc and -O under Solaris-2.3.

What type of window? Was it for one of the modules (i.e. being
swallowed) or was it something else?

mb> I know, not much help. I'll try to provide more info when I have
mb> some free time over the weekend.

If you can consistently reproduce it, let me know. Also, try
compiling w/ debug and getting a traceback off of the core file.
