>
> What is ModuleConfig tracker anyway, and what is the difference between
> this and EnableOptions? Are they two different ways of doing the same
> thing, in that case what's best?

These are pretty independent things, ModuleConfig tracker (see perllib
documentation) automatically gives you all "*YourModule: name value"
configuration, statical and dynamical.

EnableOptions gives you command line options, like --verbose --desk 2.
See FvwmPager or other modules that use both, options and module config.

Usually, the user preferences are defined using module configuration
and the things that may be different for different module instances are
passed as command line options.
