I would really like to see a new option added to "*FvwmIconMan*sort".
It would be nice to have the icons sorted using the windows' horizontal
and/or vertical distances from the top left corner of the virtual
desktop.
The window centers could be used for the calculation (as is done with
the
builtin Direction command).

I often have about 10 xterms open and I never know which button
corresponds
to which window.

The sorting should not be affected the state of the window (iconized or
non-iconized).
The icons would have to be redrawn every time a window is moved but I
would hope this wouldn't be too difficult.
