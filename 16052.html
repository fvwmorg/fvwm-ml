Tired of "Read Xpm Error: Failed to parse or alloc some color"?
Tired of having your netscape resort to use 4 colors because all of
your fancy set of icons have a different shade of blue in it, thus
using up all colormap entries? Now there is an answer!
Romano Giannetti and Martin Kraemer are proud to announce the
release of pixy in his latest version 3.2!!

What is pixy?

Pixy is a little program to reduce the number of colors of a set of pixmap
files to a fixed predefined set of colors.

Often, especially with the recent xpm-icon capable window managers like
fvwm, people have the problem that their color map is almost full after
starting the window manager, thus preventing other applications from
using a
sensible number of still available colors. Of course, this is mostly a
problem with 8-bit deep displays; they have only 256 available colors in
their color palette.

Where I find it?

Primary site is ftp://sirio.iet.unipi.it/pub/romano/pixy3.2.tar.gz, but this
is a little PC and anonymous ftp throughput is severely limited in it.
For some time it will be mirrored by Martin on
ftp://ftp.mch.sni.de/incoming/Martin.Kraemer/pixy-3.1.tar.gz

I will upload it to sunsite.unc.edu and some other wide-band server as
soon as possible. (My network connection with USA is not working
now). Everyone that can offer a better site is welcome.
