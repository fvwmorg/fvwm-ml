On 24 Mar, Rob Urban wrote:
> Hello,
>
> I've just built and installed 2.0.45 on Digital UNIX V4.0 and I've
> noticed something a bit weird:
>
> All windows titles work properly with one exception:
> the DECterm titles have "1]" prefixed to them, so if the title in
> the DECterm config says "DECterm", I get "1]DECterm" and so on.
> If I change it to "huhu" using an escape sequece, it changes to
> "1]huhu".

That's an artifact from the DECWindows applications that you are
running: I see it when I run dxcalendar, but not when running "normal"
X applications like tcl/tk, xterm, Netscape, etc. Since the calendar is
the only program that I currently run that exhibits this behavior, I
tend to simply ignore the problem.

Of course, this isn't a SOLUTION, just another person that noticed this
problem. I recall somebody mentioning it on this list a few months
ago... but searching the FVWM archives at http://hpc.uh.edu/fvwm/archive/
didn't seem to locate the message (I searched for 'title' in all
messages in 1996 and 1997 and surprisingly, it told me that there were
no matches).
