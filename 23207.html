On Thu, 17 Dec 1998 09:34:25 +0100, "Philippe BOURDEU d'AGUERRE" <bda_at_aime.insa-tlse.fr> wrote:

> Hello, With a former release (2.0.45), I had :

> EdgeResistance 250 50 EdgeScroll 0 0

> This allows me : to switch from one page to another by moving a
> window across border of screen to NOT have automatic scrolling when
> mouse reaches border of screen.

> With fvwn 2.1.45, this behaviour seems to be impossible despite the
> man page:

> Note that, with "EdgeScroll 0 0", it is still possible to
> move or resize windows across the edge of the current screen. By
> making the first parameter to EdgeResistance 10000 this type of
> motion is impossible. With EdgeResistance less than 10000 but
> greater than 0 moving over pages becomes difficult but not
> impossible.

> This behaviour was very useful for new users. I use Solaris 2.5.1

I have a similar problem (see bug report message id 65) with edge
scroll.

On my system, the edge scroll does work, but only going up or to the
left. Since my setup automatically starts in the upper left "page",
it appears that edge scroll doesn't work at all, but when in a page
that has another page above or to the left of it, I can edgescroll in
that direction. Moving windows across page boundaries also works in
only two of the four directions. Please try all directions to see if
you are able to use some, but not all directions like I can.

This does NOT occur with our UltraSPARC machines running Solaris.
This also did NOT occur with fvwm 2.0.46.
