(I hope this is the right mailing list -- apologies if it is not...)

I retrieved the 2.0.44 on Friday of last week and have it compiled and running on both
Sparc and Intel Solaris-2.5.1. On the Sparc side, no problems (yet, anyway); however,
on the Intel platform (same .fvwm2rc file on both, same locations, etc.), the "rubber
bands" which used to show up under 2.0.43 and earlier versions are gone. This includes both
the outlines which appear when moving and the outline+"grid" which appears when resizing a
window such as an xterm.

On the x86 machine, I first compiled with the SunPro compiler (-xO3 for optimization) and then
discovered the problem. So I went back and compiled without optimization to see if I was getting
an optimizer bug, and the results were identical. Finally, I used gcc-2.7.2 with no optimization
and the same thing still occurs. On the solaris-x86 platform the results are the same whether
I am running the Openlook server or XFree86 (x11r6.3).

I have not been able to track this one down yet -- has anyone seen this one before, and/or does
anybody have any suggestions?
