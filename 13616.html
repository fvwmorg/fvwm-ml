On 28 Dec 2004 11:50:23 -0600, jpkotta wrote:
>
> I've been having trouble with the Layer and StaysOn* styles. With an
> empty setup file, I startup fvwm and start an FvwmConsole and a few
> windows to play with. I type into the console:
>
> Pick WindowStyle Layer 6
> Pick WindowStyle StaysOnTop
> UpdateStyles
> Pick FvwmIdent
>
> and select a window. FvwmIdent says the layer is still 4, and it behaves
> as if the layer is still 4. 'Pick Layer 0 6' works fine.

Yes, you should use Layer command after the window is mapped. Even
Recapture does not reset the style layer in 2.5.x. This is intended.
BTW, UpdateStyles is only useful in the middle of some function.

> Am I missing something or is this a bug?

You are not the first "missing" it. What is actually important for us is
that Dominik does not think it is a bug. If you really want, some well
constructed Google query may bring previous discussions on the topic.
