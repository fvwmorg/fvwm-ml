>
> --- Stefan Kamphausen <mail_at_skamphausen.de> wrote:
>
> > > > 2. and in my .fvwm2rc:
> > > > Style "hidden" NoTitle, WindowListSkip, NoHandles, Sticky,
> > \
> > > > BorderWidth 0, Layer 0, ClickToFocus,
> > StaysPut
> > >
> > ^^^^^^^
> > > Try replacing that with: StaysOnBottom
> >
> > Of course I already did that but it didn't work. Funny enough: today
> > it does :-)
> > Don't ask me what my error was, probably I am guilty of some plain
> > stupidity.
>
> Probably :) But you have to remember that the commands *have* to be
> consistent in terms of spelling:
>
> startsonpage
> StartsOnPage
> STARTSONPAGE
>
> are all equivilent and should work, yet:
>
> Startsonpage
> STartsonpage
>
> will not.

Tomas, this is incorrect. All these spellings should work.

> > > > (And (just as a side note): is there any way to start a programm at
> > a
> > > > specific position besides the -geometry option? Like, having my
> > > > XEmacs always live at +0+0?)
> > >
> > > Yes, add them to your ~/.Xdefaults file:
> > >
> > > XEmacs*Geometry: 0+0
> >
> > OK, now I have an alias for "xemacs" giving -geometry option.
>
> No, the whole point of that file is so that you do not need the -geometry
> option to Xemacs, since the resource hint for geometry is already defined
> in the file.

This is a good solution, but there may be very good reasons to create a
shortcut wrapper rather that to put such option into ~/.Xdefaults. For
example, to preserve the default behaviour of smart or manual placement.

> > I must admit, though, that I liked the matched-window mechanism in
> > Sawfish.
> > BTW (and may now this email gets some public interest at all?): is
> > there any plan to make the matching in regexp style?
> > For example:
> > REStyle "^mozilla" StartsOnPage ....
>
> At the moment, nonse, since matching is done, either by Name, Class or
> Resource.

Tomas, this is not quite what was asked.

FVWM already supports globs, these are less powerful then regular
exressions, but in practice they are more than enough. So, your
REStyle line may be expressed like:

  Style "mozilla*" StartsOnPage ...
