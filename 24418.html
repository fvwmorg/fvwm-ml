Hey Bruce

> I just got fvwm2 running on my Sun/Solaris and CDE 1.2. However,
> I can't find a pixmaps directory. So, apparently, I can't use toys
> like xclock, xbiff, and who knows what else.

Huh? The absense of a pixmaps shouldn't affect the applications you
mention.


> Should this installation of Solaris have a pixmaps directory? (No,
> I don't control the source, else I would look!) I do have XPM 3.4k
> for Solaris installed, if it matters.

I'm running Solaris 2.5 and can only find some bitmap directories:

  /usr/openwin/share/include/X11/bitmaps
  /usr/openwin/share/include/Xol/bitmaps
  /usr/dt/share/include/bitmaps

You should be OK just using the icons/ directory that came with fvwm.


> Is there some way to simply get rid of dtlogin/CDE, and simply
> use a nice "startx" program? No, I really don't think CDE is a "good
> idea".

On my machines I rename /etc/rc2.d/S99dtlogin to something
like _S99dtlogin (anything not starting with an 'S' or 'K').
This prevents dtlogin from starting at boot time.

You can base a startx program on /usr/openwin/bin/openwin (ugh),
or if you like I've got one I wrote which I could send you.
