MK> I think (especially for 640x480 notebooks) this is a very useful
MK> addition. If I had one, I would certainly use it, and I'm going
MK> to suggest this feature to a friend (who happens to have

MK> Chuck - do you really want to keep this out of the official
MK> distribution?

Yup. It kinda goes along with my not feeling a need to support a
bunch of different window manager looks. I just don't see a real
pressing need for it right now, although I do understand why you would
like it.

If you want to save the vertical space, you could just as easily use a
really small title font or shut off the titlebar completely. If you
needed to find a window based on it's title, the builtin window list,
special Next/Prev keybindings, FvwmWinList, FvwmPager, or FvwmIconMan
could all do this nicely for you. And you wouldn't be wasting any
horizontal space from a vertical titlebar either.

MK> Once it is stable, it could be kept in #ifdef's but wouldn't deviate from
MK> the main distribution.

I'd rather not do that either, because people who compiled it in would
be expecing 'official' support. And what if some site didn't compile
it in, but one of their users wanted it because they heard about it
from somewhere, or saw a reference in the code or docs?

For now I think it should just remain a patch on Jarl's web page.
