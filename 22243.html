gensch_at_zk3.dec.com writes:

> I think it would be a useful thing if fvwm could support installable
> packages. A package would be a set of related files needed to
> configure fvwm in some special way.

One final thing. One goal I have is to be explore writing actual modules which
can change the look+feel of fvwm, probably implemented as dynamically loaded
libraries. So, for example, somebody could decide at fvwm run-time whether
they want the texuture code loaded or not, depending on their display depth. I
suspect this could make fvwm more streamlined and organized. Anyhow, it seems
that this package idea would map pretty easily to a way to specify which
modules are loaded.
