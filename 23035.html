Moody Ahmad <moody_at_mwt.net> wrote:
>
> I can start xterm in any desk I want with
> xterm -xrm "*Desk:n".
>
> Question:
> How do I select which page (desk grid #) to start
> a program in?

        Specifying the page to start on is not currently supported by
FVWM, but it's coming soon.

        You have two options. If you're using 2.0.46, and are feeling
adventurous, you can grab the StartsOnPage patch from

        http://www.pcnet.com/~proteus/Fvwm/FvwmPAtches.html

Or, you can wait a bit (not too long, I hope) till the next release is
announced. The StartsOnPage style will be in that release.

        If you apply the patch, there are a couple of things you must
do. You must also apply the DontFocusOffscreen patch (available from
the same location, and also included in the next release), and you must
add the following command to your .fvwm2rc:

        StartsOnPageModifiesUSPosition

(This command will not be needed in the formal release; in fact, it
won't be recognized, and FVWM will whine about it.)

        The StartsOnPage style is like a superset of StartsOnDesk. Its
syntax is

        Style "SomeApp*" StartsOnPage d x y

where d = the desk, and x and y are the x.y positions in the desktop
grid.

        For ad hoc page assignment, the -xrm technique is also recognized.
Following your example,

        xterm -xrm "*Page:2 2 1"

would start your xterm on the page in the third column, second row of
the third desktop grid.
