Just to be sure this problem with FvwmTaskBar core dumping when
TrueColor is used isn't just with fvwm-2.5.5, I also loaded fvwm-2.4.13
and fvwm-2.5.4 with '-visual TrueColor' and the same problem occurs.

Here's the PrintInfo Colors when run with '-C TrueColor'

FVWM info on colors
  Visual ID: 0x28, Default?: No, Class: TrueColor
  Depth: 24, Number of colors: 16777216, No Pallet (static colors)
  red: 256, green: 256, blue 256

So all looks good as far as picking the right visual ID, with TrueColor.
