Hmm, autoconf generated configure is intended to test for features.
For example the absence of getwd, if the system supports shared
libraries, if the system has an ansi c compiler and so on.

This requires that the developer adapts to it ie he/she needs
to use a number of #ifdefs for example.

This would not be necessry if libfvwm provides all modules
with an homogen interface or proxies for functions missing
in the system for example getwd.

The main stumble is ususally signals and its different interfaces.
SunOS4 supports for example both systemV and BSD.
Solaris 2.4 is POSIX and so on.
