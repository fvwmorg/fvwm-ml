%% "Dave (Grizz) Glaser" <dsglaser_at_mtu.edu> writes:

  dg> The old system.fvwm2rc file. Located in /usr/local/etc.

Hmm.

Did a new one get installed on top of that?

  dg> No ~/.fvwm2rc file. I log on through CDE. and choose openwin for
  dg> my manager (which looks at the ~/.xinitrc file to find what
  dg> manager to run.

OK. Modify your .xinitrc so that it redirects stdout and stderr to a
file (maybe ~/.fvwm-errors or something) so you can see any messages
FVWM is generating.

  dg> I fixed my profile path and that seems to be working fine. Now it
  dg> seems that the configuration file that was used in 2.0.42 does not
  dg> exactly work in 2.2.4.

No doubt.

  dg> I had a logout button that you could click on and it would ask for
  dg> confirmation, if you clicked on yes, it logged you out (exited
  dg> fvwm2).

This was probably implemented using the FvwmGoodStuff module; the name
of this module changed to FvwmButtons.

  dg> It also had a little map of the desks that you could click
  dg> on and it would change virtual desktops.

This is called a "pager", and is implemented by the FvwmPager module.
It's possible (it's been so long I can't remember anymore) that 2.0.42
still had the builtin pager, which may be what the config is using.
That won't work anymore I think; you must use the FvwmPager module.
