joe mama <fr0sbyte_at_yahoo.com> writes:
> I'm trying to install fvwm2.2.4 on a solaris 2.6 ultra
> 10 machine. But when I run ./configure it runs
> through most of the checks until it gets to setvbuf.
> I get an error msg and it stops there. Here's part of
> the log:
>
> checking for sig_atomic_t... yes
> checking whether setvbuf arguments are reversed...
> configure: error: can not run test program while cross
> compiling

It shouldn't think its "cross compiling". Earlier it evaluates your
compiler.

You can compile with Sun's cc (if you've paid for it, and its in your
path), or gcc. Heres what I get on Solaris with gcc:

checking for gcc... (cached) gcc
checking whether the C compiler (gcc ) works... yes
checking whether the C compiler (gcc ) is a cross-compiler... no
checking whether we are using GNU C... (cached) yes
checking whether gcc accepts -g... (cached) yes
checking how to run the C preprocessor... (cached) gcc -E

If it doesn't find gcc, it goes on to test cc.
What do you get?
