>>>>> "Fredrik" == Fredrik Vhrstrvm <d92-foh_at_nada.kth.se> writes:

Fredrik> I have activated the module autoraise with a delay of 2
Fredrik> seconds or so. When I enter any motif program and after 1.9
Fredrik> seconds press the Filemenu choice, the filemenu is nicely
Fredrik> popped. But then, since now it has passed 0.1 second and
Fredrik> since i am not in the newly popped up filemenu window.
Fredrik> Autoraise decides to raise the main window above this popped
Fredrik> window! Oups where did the filemenu go! Is there some way to
Fredrik> detect if a menuwindow is a child of a window and should not
Fredrik> be put behind it? Is there any way to prevent this? I am
Fredrik> using version 2.0.38.

No way to prevent this except to wait until the window is raised
before popping up the menu. The reason is that override redirect
windows are used for the popups, and are ignored by the window
manager. I don't believe that this can really be helped.
