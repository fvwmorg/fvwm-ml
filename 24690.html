Of course it's autogenerated. Robert did mention he's using FvwmSaveDesk.
Still editing the file seems to be the only solution. There isn't an
automatic way to do this because the xterm obviously changed its name
at some time. Since the information about the initial title is lost,
the only way to find it out would to look at the command's arguments
which would require fvwm to know about the syntax of applications.
Another way is to not change the window title manually. From the title
in the example I guess Robert uses it to display information about the
pwd and the terminal connection.
