On 03 December, 2001 - Mark Allan sent me these 1.8K bytes:

> We're in the process of upgrading to RedHat7.2 here, and I'm having an
> odd problem running the gnome panel with fvwm2.
>
> As soon as I start panel, GUI updates become extremely slow. After
> resizing a window, there is a pause of about a second before the window
> border and application redraw. If I move a window that was covering
> another window out of the way, it takes about a second for the uncovered
> portion of the window border to be redrawn. If I kill panel, everything
> acts normally. This behavior was present when starting panel from an
> FVWM session, as well as using FVWM2 as the window manager in a gnome
> session.
>
> In addition, I get these occasional messages printed to the X log:
> Message: resetting focus according to XGetInputFocus() for 25166128
> I don't get these messages when running Sawfish under GNOME, or running
> FVWM2 without panel running.
>
> Does anyone have any suggestions? I've been using the same FVWM2/panel
> setup for ages now, and it worked perfectly on RH6.2 and Mdk7.1. I don't
> want to give it up!

Compile your Fvwm with Gnome Hints support. If it doesn't, the tasklist
in the panel will traverse the X windows tree looking where all the
windows are every now and then, with the X server grabbed to avoid
infinite loops. It should pop up a message complaining that you're not
running a Gnome compliant WM and that it will be slow.. doesn't it for
you?
