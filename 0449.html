Julian Bradfield wrote:
>
> I'm using fvwm-2.2.4.
>
> I have SloppyFocus set by default, and an FvwmIconMan running which is
> marked ClickToFocus.
> I have GlobalOpts MouseFocusClickDoesntRaise
> set.
> In the icon manager, button1 is bound thus:
> *FvwmIconMan*Action Mouse 1 N sendcommand "Iconify -1",
> sendcommand Raise, sendcommand Focus
>
> The thing that infuriates me is that whenever I click in the icon
> manager to raise a window, the icon manager window itself raises as
> well.
>
> I haven't yet managed to track down why this happens. Anybody
> know it off the top of their head?

        Try adding

GlobalOpts ClickToFocusDoesntRaise

and see if that doesn't help.

        Note that in the recent betas, all those GlobalOpts have been replaced
by corollary Style options, so that in future you'll be able to specify
this behavior for only FvwmIconMan, if you like.
