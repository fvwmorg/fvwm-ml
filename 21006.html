On 22 Sep 1997, Uncle Bulgaria wrote:

->*FvwmButtonsFore Black
->*FvwmButtonsBack lightgrey
->**FvwmButtonsFont -adobe-helvetica-bold-r-*-*-10-*-*-*-*-*-*-*
->*FvwmButtonsGeometry 720x76-2-2
->*FvwmButtonsRows 1
->
->*FvwmButtons(Swallow coolmail 'Exec coolmail -e "" -f /var/mail/lwillis -geometry 76x76-100-100 -int 20 -fr 60 &')
->*FvwmButtons(Swallow xclock 'Exec xclock -fg black -bg lightgrey -hd red -hl red -geometry -1000-1000 &')
->*FvwmButtons(3x1, Swallow perfmeter 'Exec perfmeter -g -S -H -geometry +1048+0 -bg lightgrey -fg #660066 &')
->*FvwmButtons(3x1, Swallow xdaliclock 'Exec xdaliclock -12 -nocycle -noseconds -bg lightgrey -fg black -geometry 280x80+0+0 &')
->
->Everything works fine except that when I first start up the performance
->meter doesn't appear and the button is pressed in (All the others are
->up!)
->It's not a problem with the path as I've tried including the absolute
->path name in the Exec bit, and when I copy the perfmeter -g -S -H
->etc. to an xterm and run it from there it runs, and Fvwm swallows it
->fine, but I can't seem to get it to start when I start (or restart) my
->Fvwm
->

I suspect it may be a problem with the way you specified the
foreground colour in the perfmeter the -fg option really only likes a
name for the colour. Try using the alternative option

-Wf red green blue

where red/green/blue are _decimal_ not hex values from 0-255.
