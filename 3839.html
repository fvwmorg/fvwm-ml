> 2) I get an error message when I start fvwm2. It says,
> "[FVWM][ParseBinding]: <<ERROR>> No such key: Help"
> I echo its sentiment. Help? Must be some old junk somewhere in my dot
> file, but I am having no luck finding it.

Sun workstations have help keys. Why all workstations don't is
a mystery. The binding is built into FVWM. As Dominik said, ignore
it for now, it will go away with the next release.

> 3) What's this mysterious ~/.fvwm/ directory that the new version created
> for? Anything in particular that the creators had in mind?

FVWM can create these files:

.fvwm2rc
.FvwmAnimate
.FvwmForm
.FvwmForm-Rlogin
(and more)

having them all in $HOME looked like too much of a mess.
The ~/.fvwm directory is meant to hold those files.
