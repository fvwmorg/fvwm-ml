Ray> I work primarily in fvwm, it's my WM of choice.

As it should be. :)

Ray> But, occasionally I have to switch to olwm for GUI testing.
Ray> Unfortunately, I have not discovered a way to gracefully switch
Ray> back to fvwm. As it stands now, when I exit olwm, I get logged
Ray> out, and thereby loose all of the work I had running in other
Ray> windows. I've tried killing olwmslave -- this has no apparent
Ray> affect. Neither did I see anything in the pulldown menu that
Ray> looked useful.

Ray> Is it possible to switch back to fvwm from olwm? If so, how?

If you Restart olwm (and it uses argv[0] like mwm does) instead of
exitting, it should go back to fvwm (since that should be argv[0]).

Or, you can install XSession and use that to start fvwm and switch
between various window managers. It's also handy for capturing the
output of fvwm, and for keeping X from going down when the beta
version of fvwm that you downloaded mysteriously coredumps, which
sometimes happens, unfortunately. :)

Hope this helps.
