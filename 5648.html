At any rate: after having installed now fvwm-themes-base-0.6.1-1 and a
fresh rebuilt of FVWM 2.5.2 the only error message that I can see now
at fvwm start-time is this line on the console from where I start
fvwm-themes:

FvwmTheme is obsolete, see the Colorset section of the fvwm(1) man
page

Depending on the FVWM configuration trials here I sometimes get other
error messages (including the "old" one concerning "FvwmNetHints"),
but I think (not being sure of it, still trying) these errors come
only if i make FVWM read the old (??) config files (i.e. those from
previously installed FVWM versions) in
.fvwm/themes/personal
instead of those in
/usr/share/fvwm/themes/*

After having installed fvwm-themes-base-0.6.1-1, I rebuilt the current
stable and unstable FVWM versions: I did this cos I didn't know
exactly what impact the current fvwm-themes-base version here on my
machine has on a FVWM RPM build process on the same machine. Results
(for a RedHat 6.2) are here:
