think so, too.
can't imagine how this could be happening.

add this to your .fvwm2rc:

*FvwmAudioPlayCmd echo

*FvwmAudio new_page "new page"
*FvwmAudio new_desk "new desk"
*FvwmAudio add_window "add window"
*FvwmAudio raise_window 'raise window'
*FvwmAudio lower_window 'lower window'
*FvwmAudio configure_window "configure window"
*FvwmAudio focus_change "focus change"
*FvwmAudio destroy_window "destroy window"
*FvwmAudio iconify iconify
*FvwmAudio deiconify deiconify
*FvwmAudio window_name "window name"
*FvwmAudio icon_name 'icon name'
*FvwmAudio res_class `res class`
*FvwmAudio res_name 'res name'
*FvwmAudio end_windowlist end_windowlist
*FvwmAudio icon_location icon_location
*FvwmAudio map map
*FvwmAudio error error
*FvwmAudio config_info "config info"
*FvwmAudio end_config_info "end config info"
*FvwmAudio icon_file "icon file"
*FvwmAudio default_icon "default icon"
*FvwmAudio string string
*FvwmAudio beep beep
*FvwmAudio mini_icon mini_icon
*FvwmAudio startup "Welcome to the Machine"
*FvwmAudio shutdown "I am rebooting now"
*FvwmAudio unknown "goofed up"


Start FvwmAudio.

this will show you all events in fvwm2 stdout (written).
start fvwm2 from an xterm in foreground and watch the events, as you do the
above mouse movements again on an rxvt.

Also try with an xterm.

What's the difference?
Does rxvt generate additional focus events ?
