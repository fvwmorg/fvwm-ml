> I'm using Rox filer as file manager. http://rox.sourceforge.net
>
> But I'm start rox with a terminal, because it doesn't work in fvwm !
>
> I simply put an entry "Exec exec rox" in my menu. I try again with "rox
> -n" and "xterm -e rox"(really bad but ...).
> No one works ! ! ! Nothing happens !
>

It looks like 'rox' is just a script that calls another script that
that handles development related switches (starting rox from the
debugger etc.) and then calls the right binary.
Using just the binary seems to work perfectly.
I added the following line to the menu

+ "Rox" Exec exec \ /usr/local/apps/ROX-Filer/FreeBSD-ix86/ROX-Filer
