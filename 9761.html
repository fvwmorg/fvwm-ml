On 21 Aug 2003 at 12:49:33 -0400, Dan Espen wrote:

> I think it would be possible to write functions to print
> out some of the internal structures in close to their
> original form, but I don't think its necessary or even
> desirable.
>
> If you have a config tool and you feed its output into
> fvwm, why not edit that output file and feed it to fvwm
> again?

I think it might lead to consistence problems if a
config tool does not get feedback from fvwm. Having fvwm
writing its config ensures a config file which represents
the actual state. Further the config tool could be
initialized by fvwm and does not need to read the config
file which ensures that both fvwm and the config tool are in
the same state.
