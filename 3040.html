On 26 Aug 2001 12:18:25 -0700, Erik Steffl wrote:
>
> I see that new fvwm 2.4 is already packaged in debian, can we expect
> themese to be packaged as .deb (external or as part of debian)?

I think, Alexander Kotelnikov is a more adequate man to respond, but he
seems to only reply in threads about RedHat. :-) [joke]

The author of this packaging proposal did not respond to me as well:
  http://bugs.debian.org/cgi-bin/bugreport.cgi?archive=no&bug=79428

I think, the same man (or group) should package both fvwm and fvwm-themes,
otherwise small inconsistences or non-synchronizations are possible.

I would prefer not to deal with deb packages since I can't test them,
but if there is a demand and the fvwm packager can't package fvwm-themes,
we may do the same automatical deb packaging as we have with rpm in
both projects. This will lead to 2 fvwm deb packages, one is a clean
package built by "make deb-dist" and another is the existing one.
Similar to what was done here: http://dove.net.au/~awol/fvwm/debian/ .
I would wait for other solutions before entering this situation.

BTW, the current fvwm-themes from cvs supports the Debian / Mandrake
update_menus (if detected) with several user choices.
