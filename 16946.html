file by default. It is less of a change from the current situation.)

> A system.fvwm2rc that reads files from the user's directory would be
> better, yes, but since fvwm doesn't currently look for a
> system.fvwm2rc file, that's a minor problem...

The Debian Linux setup has a symlink from /usr/lib/X11/fvwm2/.fvwm2rc to
/etc/X11/fvwm2/system.fvwmrc. I've probably been confusing everyone else. :)

BTW, reading both the global and local config files is not quite enough for
full extensibility, since the user will need a way to add entries to the
middle of the main menu (we don't want to have "Exit" in the middle of the
menu, do we?).
