On Fri, Jan 22, 1999 at 11:19:42PM -0500, David J. Pfaltzgraff wrote:
+----------
| I have a requirement to automatically exit fvwm when an
| application is closed and have been unable to determine how this
| could be accomplished. If someone coould provide me with some
| pointers it would be greatly appreciated.
|
| The situation is that I use xdm to provide the login and then,
| through the m4 series of scripts provided with the Red Hat
| distribution and heavily modified, kick off the execution of an
| expect script which provides a GUI front end to a process
| control. The reason for the heavy modifications is that the user
| is limited to this application and cannot get into anything else.
| With all this, everything works fine except that when the expect
| script is done, the user is required to specifically select
| 'Start' and 'Exit FVWM'.
|
| Again, thanks for any pointers...
+----------

Why not make the .xsession like this

fvwm2 &
yourapp

Thus you application would control the session; when it was done
the session would end and a new xdm login would start.
