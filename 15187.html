porter> 4Dwm (SGI IRIX window manager) correctly displays the icon
porter> titles for xwsh, etc. But fvwm's icon titles for these
porter> clients are blank!

This is because that fvwm doesn't know how to treat CTEXT string. We
"fvwm users' mailing list in japan" already made a patch for handling
CTEXT string, to display Japanese character in fvwm/modules. This
patch uses X11 I18N feature, so fvwm will display whatever languages
locale can handle (if client sets collect value :-p).
