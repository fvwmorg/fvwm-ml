On Wed, 31 May 2000, Dave (Grizz) Glaser wrote:

> When I put /usr/local/bin/fvwm into my .xinitrc file, fvwm will start
> up, but the system defaults do not show up (a logout button and the
> map with the desktops on it) nor is there anyway to log out. If I
> choose restart FVWM from the menu, then everything comes up ok and I
> can use it normally.
>
> I did just a plain ./configure on my system with no arguments.

Does `which fvwm` return '/usr/local/bin/fvwm' ?

If not, the restart might be starting a different fvwm than the xinitrc,
which in turn might be reading different config files.
