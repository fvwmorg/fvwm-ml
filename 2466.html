> On 06 May 2001 14:46:31 +0200, Dominik Vogt wrote:
> >
> > > > > 2) Deiconising or switching to a window (by menu or taskbar) does
an
> > > > > annoying full screen refresh. This doesn't happen when the window
is
> > > > > selected by just clicking on it.
> > > >
> > > > I don't see this. Are you using some kind of function to
> > > > deiconify that includes a refresh?
>
> Maybe the problem is iconifying animation not working well under Exceed?
> You may turn it off in "Settings/Animation".

No, I have FvwmAnimate disabled.

Dan Espen <dane_at_mk.telcordia.com> wrote:
>
> I've used Exceed quite a bit and I don't see any full screen
> refreshes.
>
> There are some settings that might affect this.
> [...]

I have mostly the same settings. I tried changing the rest (except for
"Enable System Pixels", which doesn't seem to be in Exceed 6.2), but that
didn't fix the problem.

> I use FvwmAnimate with the builtin IconBox with no problem.

I tried the IconBox with the same result.

I did find one setting that would "solve" the problem: I turned off
FvwmBacker ("Settings | Background | Disable Background"). In fact all the
configurations that I've tried (eg. initial sample-95 settings, even with
MouseFocus) give this spurious screen refresh behaviour with FvwmBacker, and
the correct behaviour when FvwmBacker isn't running.

Of course one would expect the screen refresh when switching desktops, but
I'm getting it when I just select another window on the same desktop (or
even if I have just one desktop).

So, it looks like the bug is in FvwmBacker. Does anyone else see it, or is
it just me?

Tim.
