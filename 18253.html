> Any change to the default would break every .fvwm2rc file around. I
> would hope that we would worry more about the existing fvwm2 user base
> than some new users. I think this viewpoint is more significant than
> which policy is the better default. I, for one, would feel compelled
> to change the source code before installation, if this change was
> actually made.

I think you misunderstand my proposal. There is no need to change the
fvwm source code, just include a sample system.fvwm2rc file with the
following line (uncommented) in it:
Style "*" ClickToFocus

The fact that fvwm2 doesn't even install a system.fvwm2rc file seems
to me to be a significant problem, since the tool is nearly useless
without one (the fact that there isn't even a sample fvwm2rc
configuration file in the distribution is completely baffling to me).
My proposal is just that the fvwm2 "make install" should install a
sample system.fvwm2rc file, and that file should have ClickToFocus
enabled by default. If you wanted to make the install process a
little smarter, you could even prevent it from overwriting any
system.fvwm2rc file you may have already installed.
