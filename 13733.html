Andrew Dolgov <cthulhoo_at_gmail.com> writes:
> Hi,
>
> I have a feature-request regarding the Style command. :-)
>
> It would be really nice to have an ability to specifically request
> what window property to match on,
> e.g. Style ClassName BlahBlah ... or Style Resource BlahBlah, maybe
> even with boolean expressions.
>
> Reason: some applications use common classes like "Window" or
> "program" and/or use the same class on different windows (for example,
> IM client main window and conversation windows) which could cause
> wrong styles being applied to wrong windows when using current
> fuzzy-logic matching.

That's number 16 in docs/todo-3.0:

 16) Expand Style command to support name=pattern syntax.
     Style (Name "File *", Class XPaint, Resource *browser) NoIcon

There's no telling when it will get done.
