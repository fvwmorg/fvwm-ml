Has there ever been any discussion about automating the selection of a
MiniIcon value based on the Icon setting?

What I'm suggesting is something like a global "MiniIconPrefix"
setting that would calculate a MiniIcon file name by appending the set
prefix to whatever the Icon value is for a given Style.

This would ease the use of KDE-style icon directories, where there is
a "/mini" subdirectory within the pixmap directory that contains small
versions of each image. It also would work for those, like me, who
keep their mini-icons in the same directory as their pixmaps, but with
"mini." tagged to the begining of the filename.

Of course, this can also be done by preprocessing the .fvwm2rc (and
MiniIcon accepts values like "mini/myicon.xpm", interestingly
enough).

I'm willing to try my (somewhat shakey) hand at a patch, if there's
enough interest.
