> Currently, FvwmPager uses about three colors: One for the background,
> one for the foreground, and one for the highlighting. Is this
> correct?

  I think it uses a few more colors. The window with focus gets colorized
  by HilightColor, i.e. background is the color of the active window
  border, label is the color of the text of the active window
  title. Windows without focus get colorized with a ForeColor/BackColor
  scheme.


> In a non highlighted desk/page, the little windows can be seen only
> because they have a border round them. Is there any way of making
> them have a different color than the little root windows?

  It must be that your default BackColor is very similar to FvwmPagerBack
  color. Try making it different, or use a per class color, with something
  like:

    Style "XTerm" BackColor Pink
