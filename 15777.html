Here's what you said yesterday:

....
Jack> I then copied the example xinitrc and fvwmrc files included in
Jack> the archive to /root/.xinitrc and /root/.fvwmrc and ran startx.
Jack> fvwm 1.23 came up and ran fairly successfully but with some
Jack> error messages. So I editted /root/.xinitrc so the line that
Jack> mentions fvwm now has fvwm2. When I run startx X seems to
Jack> start, giving me the familiar blue screen background and x
Jack> curser but nothing else.

So, it looks like you are trying to use a .fvwmrc from fvwm 1.23 to
run 2.0.41, which won't work at all, or (more likely) fvwm 2.0.41 is
looking for .fvwm2rc, which doesn't exist, and it is currently useless
in that case (there is the RTFM - it says this in the fvwm2.man file).
I'm planning on making fvwm2 have more defaults built in for the next
release, so it will at least work if no rc file is found.

So, take the system.fvwmrc included in 2.0.41, and copy it to your
home directory as .fvwm2rc, and try it then.
