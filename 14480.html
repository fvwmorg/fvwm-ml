chitturi> How can I trap debug messages of fvwm? Should I redirect the
chitturi> output from fvwm > to_a_file in my xinit or use FvwmDebug? I
chitturi> will be compiling the fvwm2.1.37 tonight, so I want to trap

I recommend using XSession to spawn fvwm if you want to capture it's
debug output. It's very useful for that.

Also, keep in mind that FvwmDebug is not 100% updated for version 2.xx
and it's only showing messages to the modules (basically) anyways.
