On 21 Feb 2004 22:33:46 +0300, Andrew Dolgov wrote:
>
> I use a few programs which can create notification area icons (both KDE and
> freedesktop protocol), is there any way to have full-featured native
> notification area under FVWM?

Noone implemented such protocol support AFAIK.

> I have tried several approaches already:
>
> 1. FvwmButtons & swallow specific icon window: does not work at all with
> programs built with libegg, not flexible, loses icon if the window was
> closed.

Besides not supporting the protocol, FvwmButtons works with statical
fixed size buttons only. If I understand you, you want new icons to
appear/disapper dynamically. Currently, FvwmButtons is mostly statical
except for small possible changes to initially created buttons.

I am not familar with the protocol, maybe it is easy to create a new
module implementing a tray. If you are a programmer, you may try.

> 2. swallow applet like peksystray/docker - works a little bit better, but
> is generally ugly and unstable.

This is the only option currently. If swallowing of some fixed size tray
application into FvwmButtons does not work well, then maybe you may use
it as a separate non-swallowed window, then it may grow freely.
