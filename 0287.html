My platform:
Solaris 2.6, Sparc 5, head 1 is S24 (24 bit), head 2 is CG3 (8 bit). Soon
there will be head 3, CG3, 8 bit greyscale. I'm using FVWM 2.2.4, built
from scratch.

I'd like to know the method for specifying configuration settings based on
the display. For example, I can do HGradient title bars on my 24-bit head
with no problems, but this (obviously) nukes the colormap on the 8-bit
head. Furthermore, I may need to specify a 'shades of green' palette for
the greyscale head when it goes in.

So, in other words, I'd like to be able to use different configs for the
different heads.

This would also help me with another issue. Currently, everything
specified in FVWMButtons comes up on each display. Umm, I don't have a
need for three load monitors, or biffs. I'd like to display the main
FVWMButtons (with all the goodies) on display 0.0, but only show the
window buttons on displays 0.1 and 0.2.

I've got FVWM integrated with CDE, with '*wmStartupCommand:
/usr/local/bin/fvwm2' specified in ~/.Xdefaults. Login is handled by
dtlogin, logout is handled by a MenuItem calling 'exec dtaction
ExitSession'.

One message in the mailing list archives mentions using M4 for
preprocessing. How exactly is this done to parse for separate displays?
Or is there a way to specify display-specific config options in .fvwm2rc?

Thanks in advance for any advice you may have.
