Quoth Stephen Wray:
>
> Openlook windows don't seem to get focus properly.
> Is this a problem with fvwm or openlook?
>
> Specifically, workman...
>
> I'm running linux and fvwm 42 or so.

Actually, it's a feature.

By defaults, fvwm will not give input focus to a window which has set
InputHint to "false" in WM_HINTS (eg., via XSetWMProperties). I believe
the OpenLook stuff does so unless the window has a reason to get input
focus...

You might notice, for instance, that if you click on 'CD Info' in
workman, the window that pops up gets the focus just fine.

Which begs the question, "why do you *want* workman to get input focus?"

If you *really* want all windows to get input focus, add the line
    Style "*" Lenience
to the top of your .fvwm2rc file.

See the section on "ICCCM Compliance" in the fvwm man page for the gory
details....

Hope this helps,
