Hi all,

I've been using fvwm2 (and fvwm1 before) for some years now and I've
just subscribed to this here mailing list. I thought I'd take the
opportunity to contribute some questions/comments.

Why are the Color and HilightColor commands so different? The syntax is
different ("color1 color2" vs. "color1/color2"), the places where they
can be used (in a Style vs. at the top-level), the one doesn't accept
an "rgb:rr/gg/bb" color spec and the other does, the one can be split
into a BackColor and a ForeColor and the other can't. I end up having
to specify normal and hilighted colors in different places and in
different ways, which looks really awkward.

I want my multiple icon boxes back! Part of my desktop used to look like
this:

+-------+-------+ First IconBox 1 would fill up, and then it would
| | | "spill over" into IconBox 2. With 2.0 that's no
| Icon- | | longer possible (other than defining different
| Box 1 | Icon- | icon boxes for different programs, which I don't
| | Box 2 | want). I also don't want to use the IconBox
+-------+ | module because it takes up desktop space with
|Xfaces | | its own window.
+-------+-------+

I would like to be able to define shortcuts for menu entries at
the same time as I define the menu entries themselves, with the
AddToMenu function. Now they are defined in different places,
which doesn't help in keeping them consistent.

Regards, - Jacco
