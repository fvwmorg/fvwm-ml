>When I try to restart my fvwm2 session, for the changes to take effect, why
>is my fvwm session terminating?

I can only offer personal experience. Usually I make incremental changes
to my configuration files. Sometimes (i.e. usually) I make an error in
what I type in or I would just guess at something which might work and
find that it doesn't (like what I was trying a while ago with switching
decors). Occasionally this causes my login session to terminate and
return me to xdm or simply hang (for at least long enough for me to know
that I did something wrong). Are you redirecting errors to a file? Have
you tried loging into a virtual terminal i.e. not starting fvwm and
checking out what the errors tell you? Have you tried using a simple
.fvwm2rc file to see if the problem still persists or if it might be
something in your present configuration file which is causing the problem?
All I can suggest is that you play around a little to see what the
behavior of the problem is.

Sorry I couldn't help more,
