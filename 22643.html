Vikas Agnihotri <VikasA_at_cagw1.att.com> writes:
> On Fri, Jul 31, 1998 at 04:14:47PM -0400, Vikas Agnihotri wrote:
>
> > > # Button 2 with Meta (diamond), moves window: dje
> > > Mouse 2 W M Move
>
> > Cool. Works like a charm.
>
> Small problem with this recipe.
>
> In a xterm, when I do this with my mouse anywhere in the window, the move
> works fine, but when I release the mouse buttons (1+3 since I am using a 2
> button mouse to simulate Button 2), the current primary selection gets
> pasted into the xterm!
>
> I understand that it is happening because Mouse 2 in a xterm is mapped to
> paste the current selection, but this is certainly a unexpected side
> effect and could be dangerous if I am running a curses-based application
> that interprets the pasted characters as commands!
>
> Comments?

Get a 3 button mouse, or use button 1 or 3.

"Meta Button 2" should never be seen by the xterm. fvwm2 will not pass
the "Meta-button 2" onto the client. Something is wrong. From this
distance, I can't tell if its your fingers, your X-window emulator, or
something else.

I just tested this using "xev". Sure enough, xev, sees Meta button 1
and 3, but never sees button 2. It does see the "meta" key being
pressed and released, but button 2 is completely invisible to it.

Try the same (xev) with your 2 button mouse. Notice if the underlying
application sees both the press and release events.
