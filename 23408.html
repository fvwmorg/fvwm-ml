  ak> I am trying to build fvwm on solaris 2.7 and somehow it appears to

Please address issues with the beta tests to the fvwm-workers list,

  ak> The "make install" does not copy sample.fvwmrc and other modules
  ak> to proper directories.

Define "proper directories"? Note that modules install into a
completely different directory structure starting with the 2.1.x series,
which uses GNU autoconf with all that implies about directory structure,
from the 2.0.x series.

You should unset the ModulePath setting in your config file and allow it
to use the default. If you need to reset ModulePath (add stuff to it,
for example), use the environment variable $FVWM_MODULEDIR to insert the
default path into the ModulePath setting where you want it.

If you think the modules didn't install at all, we'll need to see a
transcript of the "make install" command you invoked and what it
printed.

  ak> I tried to go to sample.fvwmrc directory and looked at Makefile
  ak> and found that

  ak> and hence make install does not do anything.
