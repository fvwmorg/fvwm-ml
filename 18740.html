Hi!

When I log in using the departmental X terminals, I get an xterm
running, and then the window manager is started from the .login file.
I use fvwm version 2.0.43 with the CleverPlacement patch. I would
like to iconify the initial startup window as soon as fvwm has
started, so I use this InitFunction:

AddToFunc "InitFunction"
+ "I" Next [!iconic CurrentScreen xterm] Iconify 1
+ "I" Module FvwmButtons
+ "I" CursorMove 30 40

When fvwm2 starts up, the xterm is iconified, but immediately
deiconifies again. The deiconified window is not updated, so only the
frame shows, the xterm window shows what was below it when it
deiconified, but the contents move along with the window. Choosing
refresh screen from the menu causes the xterm window to become black.
It becomes normal if I use the mouse to iconify and deiconify it
again. Adding the same command to the RestartFunction causes the same
weird behavior.

I use SloppyFocus, and the only modules I start are the pager and
FvwmButtons.

Any suggestions?

Jakob
