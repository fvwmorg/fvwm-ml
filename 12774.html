On Tue, Jul 13, 2004 at 02:12:55AM -0600, Matthew J Kettlewell wrote:
> Hi all,
>
> It's late and I'm sure I'm missing something easy. I'm trying to change
> the color of the clock that floats on the right hand side of the taskbar,
> but can't seem to find the correct setting. It only wants to be black.
> I've seem some screen shots where it is white, but I can't find a .fvwm2rc
> file to explain what it does.

The clock uses the standard FvwmTaskBar colour set:

  Colorset 99 fg white, bg black
  *FvwmTaskBar: Colorset 99

The obsolete Fore and Back options work too, but they must be set
before the task bar is started, while colour sets can be changed
at any time.

It looks like the clock string is not updated automatically when
the colour set changes, though. If you start with a black
foreground, start FvwmTaskBar and change the colour set's fg to
white it may take a while before the clock is drawn in white too.
I have committed a fix to CVS. If it still does not work with the
fix, please post your colour set and FvwmTaskBar configs.

> I'm using 2.5.10 (unstable release) on Linux 2.4.18 kernel and XFree86
> 4.4.0
>
> I've played with colorsets and the Fore and Back settings to no avail....

> Thoughts ?? ( Help would be even better, but I'll even take an insult at
> this point )
