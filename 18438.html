I did a couple of greps thru the source code for 2.0.42, and I don't
find any uses of the word "undefined". Also I don't find any
occurrences of the string "_NoResource".

I do find a couple of uses of the string "NoResource" but not in the
context of printing a message. (It seems to be used when something
doesn't have an X resource name.)

Moving to 2.0.43 might be helpful.

I always compile with the xpm package, and I suspect that most others
do this also, so I would recommend taking the time to do that.

You mention a crash, if there is a core file, a stack trace would be
helpful.

If none of these work, I would attempt to start fvwm2 under "trace"
and if that didn't help, I would start adding printfs to fvwm2.

> Also i would be glad if i can get a precompiled binary for SUNOS4.1.3.
>
> Please dont disappoint me THIS TIME.

I think that sometimes, there is no answer posted because no one has
an answer. There is no one on this list that is designated as the
last resort question answerer. (I am not suggesting that there should
be.)

I would suggest that if you don't get an answer the first time, try to
get more information and post again. Eventually given enough data
someone will become intrigued and jump in.
