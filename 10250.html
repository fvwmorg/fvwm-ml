But you do not want to do this for all windows. Use FvwmIdent to
find out a better pattern for all windows of this application.

> Based on what xwininfo is showing, why
> won't fvwm simply honor the user specified size that I've asked for?

That's hard to say without analysing the situation. The
application does something that causes fvwm to limit the window
size. It may request that size on its own, request a maximum
window size smaller than the minimum size, or ask for a size
increment step width/height that is so big that fvwm rounds down
to the minimum window size. Do you get any warnings from fvwm on
the X console related to this problem?
