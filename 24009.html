Hi everyone,

I'm new to this list. Sorry if this problem has been coverd before, I
didn't find anything about it in the archive...

So, here is what happens when I try to toggle between windows on different
desktops, for example :

- windows "a" and "b" on desktop 0
- window "c" on desktop 1
- focus on "a"
- windowlist "a,b,c" (result of the built-in command "WindowList
NoDeskSort")

if I do the following actions :

1) Prev [c] FlipFocus
  
   the windowlist becomes "c,a,b" (this is the normal behaviour)

2) Prev [*] FlipFocus

  the focus gets on window "a" and the window list becomes "a,b,c"
(instead of "a,c,b" which is, if I understood well the documentation, the
expected behaviour)

3) Prev [*] FlipFocus

  At that time, I would expect the focus to get back to c, but since the
previous windowlist is wrong, it gets to b.

NB : when all windows are on the same desktop, FlipFocus works fine.

Has anyone any idea to fix this problem ?

I'm running fvwm 2.2 on redhat 5.1.

Thanks in advance.

Denis
