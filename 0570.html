On 03/26/00, Anthony Heading said:
> So finally my query: is it possible/sensible to run fvwm
>inside another window manager instance, controlling not the
>display and the root window but instead a single large window,
>swallowing and then managing all the top level windows of
>my application?

As far as my understanding is, the WM is a special X client that deals in
special ways with the server and root window. This uniqueness allows for
only 1 of such types of apps at once. (if I'm wrong, someone correct me)

You might want to look into Xnest (part of XFree86), which runs a whole new
X server inside an X window. You can run FVWM or whatever else you want
inside this window.
