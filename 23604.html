> Hello,

> After i installed xfree86 3.3.3.1 i can't get any pixmaps

> I get the following error:
>
> FVWM [setPixmap Path]: <<error>> XPM support has not been
> included in this version of fvwm2

It sounds to me like the xfree86 installation has given you another version
of fvwm2 that is compiled differently. Try typing the following in an xterm:

whereis fvwm2

On my machine this gives the answer:

fvwm2: /usr/local/bin/fvwm2

This is the default install location for the newer versions of fvwm2. If
when you do it, it gives two locations for fvwm2, something like this:

fvwm2: /usr/X11R6/bin/fvwm2 /usr/local/bin/fvwm2

then that would imply that an extra version of fvwm2 has been installed, and
you can get the old one just by calling it with the full path, ie:

/usr/local/bin/fvwm2

instead of just

fvwm2

(probably in your .xinitrc). You may wish to delete the version of fvwm2 you
are not using, since it is just wasting disk space and risking confusion.

If whereis only shows one version of fvwm2 installed, with "X11" in the path
somewhere, something like:

fvwm2: /usr/X11R6/bin/fvwm2

then that probably implies that your old version of fvwm2 has been
completely overwritten by the xfree86 installation and you will need to
install a new version.

The whereis command can also give the path for man pages (although in my
case it doesn't seem to), so disregard anything with a filename like fvwm2.1
or fvwm2.1.gz.
