Well... thanks for all the nice advices, but really I did not like
to install some giant image elaboration package for this... so, I made a
little program that seem to work. This is the README:

******************
This is a little stupid program to reduce the number of colors
of a pixmap set.
Use:

   pixy [-v] base.xpm icon.xpm > newicon.xpm

after this magic, newicon.xpm will have its color changed to the
nearer one of base.xpm. Next, if you want to get rid of duplicate
colormap entries, do a

   sxpm -nod appl2.xpm -o appl3.xpm

and you have it.

Enjoy!
**********************

... well, it is really an hack and it control almost nothing on
correctness of parameters etc etc... so keep attention. I have used
it one hour without problems.

find it on:
                ftp://sirio.iet.unipi.it/pub/romano/pixy.tar.gz

No version number because I really do not know if I'll like to support
and/or improve it.

Bye,
        Romano.
