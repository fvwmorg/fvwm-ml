First off, thanks to Rasmus Tamstorf and Chuck for letting me know that
my posts are making it to the list.

Here's some more data points on the none.xpm/Iconify anomaly.

fvwm 2.0.42 on Irix 6.2.

I'm using the following sort of thing:

Style "*" Icon none.xpm, SmartPlacement, NoPPosition
Mouse 2 I A Iconify

On the mouse 2 event in an iconified window it is de-iconified but
not raised, i.e. it comes up *under* all other windows on the screen. If I
use real pixmaps instead it is always raised as well.

I have got around it by using the following function instead:

Mouse 2 I A Iconify-and-Raise

AddToFunc Iconify-and-Raise "I" Iconify
+ "I" Raise

This works fine.
