In fvwm, I get 5 areas for each button. In fvwm2, I get Desk 0 only.
Selecting another button removes highlight from Desk 0. Clicking on
any of the 5 areas for Desk 0 returns to button zero.

In FvwmPager, if I use 0 9 rather than * *, then the pager becomes 9
wide with 10 desktops always shown.

I have tried all combinations that I can think of to circumvent. Is
this the expected behavior of fvwm2? Can the "Desk 0, Desk 1, etx."
be removed?

Here are applicable parts of my .fvwm2rc

DeskTopSize 5x1

*FvwmButtons Work foobar.xpm Desk 0 0
*FvwmButtons Whom foobar.xpm Desk 0 1
*FvwmButtons Gen foobar.xpm Desk 0 2
*FvwmButtons CCT foobar.xpm Desk 0 3
*FvwmButtons Docs foobar.xpm Desk 0 4
*FvwmButtons Test1 foobar.xpm Desk 0 5
*FvwmButtons Test2 foobar.xpm Desk 0 6
*FvwmButtons Test3 foobar.xpm Desk 0 7
*FvwmButtons Test4 foobar.xpm Desk 0 8
*FvwmButtons Test5 foobar.xpm Desk 0 9


*FvwmPagerGeometry +0+965

*FvwmPagerDeskTopScale 32

Style "FvwmPager" NoTitle

Style FvwmPager WindowListSkip

Style "FvwmPager" Sticky

AddToFunc InitFunction "I" Module FvwmButtons
+ "I" exec xsetroot -grey
+ "I" Module FvwmPager * *
AddToFunc RestartFunction "I" Module FvwmButtons
+ "I" exec xsetroot -grey
+ "I" Module FvwmPager * *
