> > when I move the icons, I can still

> >
> > Style "*" IconBox +900+8+927+800, IconGrid 90 90

> > I would have had the icons dropped at least at 90 pixel from each

>
> I am not too sure (I possibly could be wrong here), but I
> think what he is trying to get the icons to do is:
>
> . after minimizing a window the icon should end up
> in the IconBox area defined above.
You're right here.
> . if this icon is ever moved out of that region
> by the dragging it w/the mouse, it (the icon)
> should be return to the IconBox area; like a
> "sling-shot" effect.
No...
Let me do an example.
Let's say I defined an icon grid of 30x30 pixels. This grid should cover
the entire desktop space.
        ----------------------------------------------
        | <- 30 pix -> | <- 30 pix -> | <- 30 pix -> | etc.
        | ^ | | |
        | | | | |
        | 30 | | |
        | pixels | | |
        | | | | |
        -------------------------------------------------

Ok?
Now: I have an icon in the icon box, and I want to move it somewhere
else
on the desktop. If I decide to drop it (let's use the upper left corner)
in +30+30, it should be dropped.
If I decide to drop it in +40+40, the icon should be dropped with the
upper left corner
in +30+30, while trying to drop it, for example, in +55+35, would drop
it
effectively in +60+30...

...hope it helped to understand...
