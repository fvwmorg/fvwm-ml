> I' m currently working with a workstation SGI indigo 2 running Irix 5.3
> using the fvwm. I have two monitors connected.
I did not think that you could double head an SGI. Oh well.

> I just want to be able to tranfer a window on my machine from the display
> falco:0.0 to falco:0.1.
You cannot. There is no general way to move windows from one display to
another. This is almost equivalent to moving a window from host1:0 to
host2:0, it can't be done. (I wish that it could, my Ultra has 2 heads.)

You would have to change the Display pointer in the X program, reconnect
to the display, re-load all the fonts, possibly change visuals and depths,
and so forth.

It is possible to write a client that can do this, e.g. xemacs. The
make-frame-other-display function can do this. I have also seen patches
for Xnest that allow Xnest to connect to 2 displays, giving you 1 large
virtual desktop across 2 monitors. There is a huge speed and memory
penalty, and I don't even remember where the patches are.

If it were to be done, it would have to be done on the server level, not
the window manager level.
