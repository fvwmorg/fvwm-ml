FVWM *DOES* support program 'Name' & 'Icon Name' changes to change the
titlebar and icon names respectively (see HandlePropertyNotify in
events.c), which is all XTerm does when it sees those esc sequences
(it changes the XA_WM_NAME and XA_WM_ICON_NAME property values for the
window that it creates), which FVWM should then get notified of the
change and update the titlebar. XEmacs also updates it's 'Name' &
'Icon Name' information to up

I (and many others as you've noticed) have had no problems with this,
so I suspect something else going on in your system that's
interferring here. Not sure what to tell you to help figure it out
though - sorry.
