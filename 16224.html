Well....

punk$ ll f*
-rw-r--r-- 1 windsor src 576551 Apr 15 13:35 fvwm-2.0.42.tar.gz
-rw-r--r-- 1 windsor users 23447 Nov 24 18:31 fwtk.patch
punk$ cp fvwm-2.0.42.tar.gz /tmp/windsor
punk$ cd /tmp/windsor
punk$ ll
total 581
-rw-r--r-- 1 windsor wheel 576551 Apr 18 01:10 fvwm-2.0.42.tar.gz
punk$ gunzip fvwm-2.0.42.tar.gz
punk$ ll
total 2637
-rw-r--r-- 1 windsor wheel 2682880 Apr 18 01:10 fvwm-2.0.42.tar

But, that's not the 'size that counts'. Anything that is near as portable
as fvwm will have it's share of #ifdef'd code that isn't usable for
a particular arch/os.

This is what is important:

punk$ cd /usr/local/install
punk$ ll fvwm-2.0.{39,40,41}/bin/fvwm
-rwxr-xr-x 1 windsor src 126976 Dec 10 02:46 fvwm-2.0.39/bin/fvwm
-rwxr-xr-x 1 windsor src 135168 Feb 27 06:02 fvwm-2.0.40/bin/fvwm
-rwxr-xr-x 1 windsor src 135168 Feb 27 06:07 fvwm-2.0.41/bin/fvwm
punk$ file fvwm-2.0.{39,40,41}/bin/fvwm
fvwm-2.0.39/bin/fvwm: NetBSD/i386 demand paged dynamically linked executable
fvwm-2.0.40/bin/fvwm: NetBSD/i386 demand paged dynamically linked executable
fvwm-2.0.41/bin/fvwm: NetBSD/i386 demand paged dynamically linked executable
punk$ ll `which twm`
-rwxr-xr-x 1 root bin 155648 Jan 6 23:34 /usr/X11R6/bin/twm
punk$ file `which twm`
/usr/X11R6/bin/twm: NetBSD/i386 demand paged dynamically linked executable

So..

Ain't nuttin' wrong here. It's still smaller than twm, which I fully
appreciate.
