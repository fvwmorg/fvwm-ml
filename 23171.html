%% Paul D. Smith <psmith_at_BayNetworks.COM> writes:
%% Pete Forman <gsez020_at_compo.bedford.waii.com> writes:

  pf> You need to set a "Reply-To:" field if you want replies to go to
  pf> the list rather than yourself. This is documented in RFC 822
  pf> section 4.4.3. I see that you have a "Mail-Followup-To:" but
  pf> this is non-standard.

  ps> No, you need to use the "followup" command in your mailer to
  ps> send to all recipients, instead of the "reply" command to send
  ps> only to the author. "Followup" will send to fvwm-workers, since
  ps> it's on the CC line.

Followup sends two copies to the originator, one directly and one via
the list. Most of my list activity is in USENET, so I'm used to
sending followup to the newsgroup only. The originator should get a
direct copy only if they explicitly request it.

I suppose that the solution is for the mailer to recognize mailing
lists and remove the author from the followup recipients, unless
followup & reply has been explicitly called for.

  ps> Certainly if Dominik never wanted to receive private replies he
  ps> could set his reply-to, but people sending patches, etc. should
  ps> reply to the list regardless.

  pf> Perhaps the majordomo could be configured to add a reply-to if
  pf> none has been set by the original poster.

  ps> No. Or rather, majordomo can be configured that way, of course,
  ps> but it is The Wrong Thing To Do.

  ps> http://garcon.unicom.com/FAQ/reply-to-harmful.html

Point taken.
