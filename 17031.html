>>>>> "JWR" == James Wheelis Riely <jamesri_at_cogs.susx.ac.uk> writes:

    JWR> I'm using 2.0.41 with ClickToFocus set. I have a problem
    JWR> when using applications like netscape where windows pop up
    JWR> and then disappear (say for searching). The problem is that
    JWR> when a window dies, the focus does not necessarily go back to
    JWR> the parent window (which had the focus beforehand).

I have seen this with many applications, including Tk-based apps,
XEmacs 19.13, and Netscape. I have seen this problem exhibited on
most (all?) twm derived window managers, including ctwm, tvtwm, and
fvwm. I've seen it on olwm and dtwm too which I don't *think* are twm
derivatives. The only wm I haven't seen it on is my colleague's SGI
box. I don't know what the standard IRIX wm is called, but that one
handled these focus problems okay.

This is the most annoying buglet about fvwm. I'd love it if fvwm was
fixed!
