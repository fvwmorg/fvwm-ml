Hm. Unfortunately, I'm not sure what we can do about this situation.
The fatal error is occurring -within- the autoconf macro that tests for
a valid C++ compiler, and I see no way to avoid it and still run that
test (which is important, IMO).

We do everything we can: for example, if you have _NO_ C++ compiler, it
should work OK because we look around first and only try to test for a
valid compiler if we can find something that looks like it aught to be
one.

I think the only error case is if you have a C++ compiler, but it
doesn't work. In that case you have to provide --without-cxx or
--with-cxx=no or configure will fail.

I'll ask the autoconf folks about this, but I'm not sure there's much we
can do (short of writing our own version of AC_PROG_CXX that doesn't
generate a fatal error).
