%% "Thomas A. Gardiner" <gardiner_at_pas.rochester.edu> writes:

  tag> If I may chime in, I think it's worth mentioning that the choice
  tag> of "where" to install the compiled code is worth thinking about,
  tag> since they can not be moved from there without recompiling...

Well, FVWM can be moved without recompiling, but the builtin ModulePath
will be wrong so you'll have to reset that in ~/.fvwm2rc.

I guess some of the newer self-config features in 2.3.x may also not
work quite correctly.

But, in general, FVWM is fairly portable. There do exist some GNU tools
(Emacs and GCC spring immediately to mind) which are a pain in the rear
to move--easier to recompile them.

  tag> Therefore, I think a different directory would be preferable, say
  tag> without the dash...

OK... but what's wrong with the dash?
