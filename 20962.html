I think FvwmCommand has to run from Fvwm, like any other module?

For a non-module process to send commands to Fvwm, for now you would have to
run FvwmPipe, or if the command is dedicated to just returning data for Fvwm,
you couls use PipeRead in a function. I've tried using FvwmPipe, but the wait
loop it goes into instantly boosts my server's load by a full point.
