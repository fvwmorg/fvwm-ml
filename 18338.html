Hello,

I think I've found a bug in all recent fvwm2 versions (this includes fvwm95
and Bowman up to 2.0.42, too). When a toplevel window gets resparented by
another widget, fvwm2 doesn't remove the decorations of the reparented
toplevel. fvwm1, mwm and tvwm behave correctly in this case.

I've remarked this error when I've started tgdb 2.6 (the new version of
our graphical gdb interface) under fvwm95. tgdb has a "debugee window"
which provides I/O for the program to debug. This window is a Xterm
that is reparented by a Tk frame widget and packed into another toplevel.
Unfortunately, the Xterm decorations won't disappear and fill the whole
background of the current screen. You can reproduce this even with the
old tgdb version 1.4 (you can find it on sunsite.unc.edu in the
/pub/Linux/devel/debuggers directory).

Does anyone know about this bug? Is there possibly a workaround? We'd like
to release tgdb 2.6 very soon, but only if we can be sure that it also
works on fvwm2 and other WMs based on this version. Please let me know
what develops (I'm not subscribed to the mailing list, so I'd apprectiate
sending me email).


Thanks very much,
mike
