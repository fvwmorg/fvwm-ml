David Koo <koo_at_ishoni.com> writes:
> Hi All,
>
> I've got a strange problem regarding iconising netscape. The scenario
> is as follows:
>
> * In my .fvwm2rc file I have (among other things):
> Style "Netscap*" Icon netscape.xpm, IconBox 32x900-1+50
>
> * I start netscape.
>
> * If I iconise this window, the icon I get is NOT netscape.xpm - its
> the netscape icon that comes with the application.
>
> * Now I open a new browser window and iconise this new window. The
> icon shown IS netscape.xpm. In fact, for all subsequent windows,
> the correct icon is shown. The first window, however, still does
> not iconise to netscape.xpm.
>
> * I Restart Fvwm and iconise the original (first) netscape window.
>
> * The icon shown now is the correct netscape.xpm icon.
>
> * The same thing does NOT happen with, say, ghostview.
>
> Am I missing something?! Is it a bug with fvwm2? BTW,
>
> fvwm2 -version:
> FVWM version 2.3.28 compiled on Feb 26 2001 at 14:04:22
> with support for: ReadLine, XPM, GNOME WM hints, SM

There were a whole series of changes to icon handling starting before 2.3.28
and ending afer 2.3.28. Please update to 2.3.29 and let us know if you
still have a problem.
