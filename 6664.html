Victor,

I figured this out.

The fvwm project is in the process of converting the main name of their
executable from fvwm2 to fvwm, and there are some other name changes,
including a change from xpmroot to fvwm-root.

I have not followed this change in the fink packages, because when I first set
the fink packages up, I followed the example from the Debian project and
constructed a setup which allows the original fvwm (fvwm1) and fvwm2 to be
installed simultaneously. This recent conversion process at the fvwm project
would break my setup, so I have been disabling it by using a flag they provide
(NEW_NAME_SYMLINKS=false). This is why you are not finding fvwm-root: it is
actually called xpmroot, it's old name.
