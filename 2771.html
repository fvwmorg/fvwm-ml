On Wed, 27 Jun 2001, n6tadam wrote:

> Dear List,
>
> The only thing I have changed, is the variable "ModulePath", in the per user
> file ".fvwm2rc.m4" (I am still using AnotherLevel -- Lestiff style). I
> changed the variable from the following:
>
> ModulePath /usr/X11/lib/X11/fvwm2
>
> to the following:
>
> ModulePath /usr/local/libexec/fvwm/2.3.28
>
> (I assume that is what I am supposed to do??)

I think you want to do something else.

At the top of fvwm2rc.m4 there is a section which looks like

changecom(%%%)

... snip ...

#`FVWMDIR'=FVWM_MODULEDIR
#
#`XRELEASE'=RELEASE
changecom(#)

... snip ...

ModulePath FVWM_MODULEDIR


This is the default for AnotherLevel-1.0.1-1

I think what Dominik is telling you is that even this is overkill, as long
as you don't relocate the modules, since these paths are compiled in.

To keep things simple, I would just suggest you leave the default
AnotherLevel config in place and it should work just fine.

Tom

Ps. You might want to note the changes with ImagePath and PixmapPath to
modify AnotherLevel though...
