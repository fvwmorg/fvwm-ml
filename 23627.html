>>>>> "J" == John <jrsmith_at_eng.utah.edu> writes:

J> Forgive my mailing to this list (I sent to the fvwm95 list last night,
J> but it doesn't seem to very active) ...

I believe that Fvwm95 is, in fact, a dead project. Fvwm is undergoing
active development and is rapidly approaching the stable 2.2 release. (The
current testing release is 2.1.11.)

I believe that all of the functionality of what was Fvwm95 is available in
Fvwm (although if the look is all you're after, you may have to do a bit of
configuration). It may be worthwhile for you to try the current Fvwm
version and see if it shows the same problem; given the work that went into
colormap issues, I believe it should perform much better.

J> I'd like to make this the default window manager in a lab of about 120
J> workstations but can't in it's present state.

Given the large number of bugs that have been fixed in the mainline source,
this probably isn't a good idea in any case.

J> Any suggestions?

Try Fvwm 2.1.11 (from ftp.fvwm.org/pub/fvwm/version-2/testing/) and see if
it solves your problem. If it doesn't, report the bugs and we will endeavor
to fix them.
