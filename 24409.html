  I've noticed a slight oddity with fvwm-2.2.2 under Tru64/Dec Unix/OSF1
when running on a 1280x1024 display; it works fine on 1024x768 displays
(which use the on-board video card as opposed to the machine with the
1280x1024 display which uses an add-on card from Digital).

  If I use the mouse to switch virtual windows by moving the mouse off
the right side of the screen and move the mouse too quickly, the pointer
goes off in to "null space" -- the virtual window does not switch. If I
do the same behavior, moving the mouse very slowly, then it switches with
no problem.

  As far as panning up, down, and left, I can move the mouse as fast as I
want to and it switches every time, but the right side is picky. I also
noticed that this same machine (with the 1280x1024 display) has two
displays, so I had to use fvwm2's '-s' option to prevent two instances of
fvwm from starting.

  My best guess is that the machine with the add-on video card (and the
1280x1024 display) is running a different X server than most of our other
Dec Unix running 1024x768.

I have enclosed the output from 'xdpyinfo -ext all' from the 1280x1024
screen in hopes it will be useful. Any thoughts on how to fix this
problem would be most appreciated.

Thanks,
-Jason
