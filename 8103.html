On 01 Mar 2003 22:26:50 +0100, Gert Brinkmann wrote:
>
> Mikhael Goikhman wrote:
>
> >The advice is to use FvwmButtons for this. Have menu or launch buttons,
> >swallow FvwmIconMan and any applet like your app inside FvwmButtons.
>
> Do I understand you right?
>
> I should build my own taskbar by using FvwmButtons that swallows all
> things I want (in my case a FvwmPager)?

Yes, developers prefer not to have a duplicating functionality. Ideally
there should be 2 very configurable modules, one for the dynamical list of
windows (let's call it FvwmIconMan for now) and one for the button layout
(let's call it FvwmButtons). FvwmPager is something different, but if you
prefer it to the list of windows - you may swallow it too.

> Can FvwmIconMan be configured that it behaves the same way as
> FvwmTaskBar (with onmouse-over-bubbles, etc.)?

Balloons are not available in FvwmButtons and FvwmIconMan, but you wanted
to add some code to FvwmTaskBar, am I right? So I suggest, instead, to
work on missing features of FvwmButtons and FvwmIconMan. I would think
adding balloons is easier than adding such features like button layout
and swallowing (that are already available in FvwmButtons).
