I went ahead and delved into the sources, and got the following into Andrews
cool patch:

1) New Directive - PixmapJustify
                   It takes "Left", "Right", or "Center"
                   (someone requested this and it wasn't difficult)

2) New Directive - PixmapTile
                   It doesn't take any arguements - by specifying it
                   you are saying "yes". The code responsible for the
                   manipulations is the same as my previous patch. This
                   overrides PixmapJustify.

The documentation on these new ones is missing, so save this e-mail!

So your .fvwm2rc will have a line (or three) resembling this:

TitleStyle flat leftjustified ActivePixmapUp denim.xpm ActivePixmapDown smiley.x
pm
TitleStyle InactivePixmap cool_blocks.xpm PixmapJustify Center

or

TitleStyle flat leftjustified ActivePixmapUp denim.xpm ActivePixmapDown smiley.x
pm
TitleStyle InactivePixmap cool_blocks.xpm PixmapTile


The patch itself is uuencoded, gziped, in context diff form (my thanks to
Lars Wirzenius for helping me figure diff out). It includes everything
that Andrew's did, except the changes in documentation and a couple of
very small changes in the flow.

If imitation is the most sincere form of flattery, then Andrew should
consider himself sincerely flattered =] Thank you a LOT, because without
your patch I'd have never started.

Any other suggestions? Would people be intrested in a pixmap tiled around
the border? It'd be a lot of work, but I'm learning tons by doing these.

Anyway, lemme know what you think!
