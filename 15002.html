When fvwm (2.0.39) is restarted, windows on screens other than
the upper left screen change slightly in position.
This only happens on the first restart after a particular window's creation.

To illustrate, given a 2x2 desktop, whose four screens are labeled:

+-------+-------+
| (0,0) | (0,1) |
+-------+-------+
| (1,0) | (1,1) |
+-------+-------+

If you start an xterm in the empty screen (1,0), it will
place itself at geometry 80x24+0+0. Both xwininfo and FvwmIdent report
this geometry.

Restart fvwm.

Recheck geometry of the xterm in screen (1,0).
Now xwininfo reports 80x24+-1+-1 while FvwmIdent still reports 80x24+0+0.

The window has truely moved - easily verified visually. Only one pixel,
but now you can see the top pixel line from that xterm at the bottom
of screen (0,0). Repeated restarts do not move the window further.

Anyone else observed this? Is it a Bug or a Feature(tm)? Anyone working
on it? If it's a problem up for grabs I could take a look, but I don't
want to duplicate effort. Chuck?
