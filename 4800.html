This is generally incorrect. Starting from 2.3.x it should be simply
"Restart". I can only guess what "Restart fvwm2" does on your sistem.
It depends on your $PATH if you have more than one installation. There
are also other issues described in the man page. Please use the correct
form "Restart". Unless you want to restart to a different WM, of course.

> enabled via pop up menus
> and If I use
> Restart --preserve-state
> that doesn't help either.

I hope you use 2.4.x or greater. I can't reproduce this problem.

Is it possible your home directory is not writable? This is needed to
create the restart state file.

Open FvwmConsole window and type in it:

  Exec xmessage $v # or "Exec echo $v" if you don't have xmessage
  Style FvwmConsole Sticky # this is good to have in .fvwm2rc
  GotoDeskAndPage 1 1 1
  Restart

Report results, including any message written to the standard error.
