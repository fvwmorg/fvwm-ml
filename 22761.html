Stepson <stepson_at_aosi.com> wrote:
> Hey all,
> I'm trying to install FVWM2 on my Sun Sparc 2, running Solaris 2.6. I
> have the latest gcc compiler (2.7.2 or the like?), and first gnu
> complains of an unrecognized language (cg92) but appears to continue to
> compile, but then I get a:
> Picture.c:33: X11/xpm.h: No such file or directory
>
> I checked in /usr/include/X11, and sure enough, its not there! I tried
> commenting out the line that calls the xpm header file, but that didn't
> help (I don't know much C to be honest). So, any help would be
> apprecaited. Resond privately, if possible.

Hi,
        The X11/xpm.h is part of the Xpm package that you can find in
        in ftp.x.org. I think that it is in the contrib directory.

        With respect to the gcc errors, they are due to the fact that
        you have configured your xmkmf for the Sun C compiler, if you
        want to stop the warnings you need to change the configuration
        of that.

Hope this helps.

Later, Juan.
