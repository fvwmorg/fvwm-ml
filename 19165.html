Both of the new FvwmIconMan versions (v0.9.1 and v0.9.2) are seg
faulting when emacs switches to the minibuffer *if* I have the
following line in my .fvwm2rc:

*FvwmIconMan*2*dontshow title=*Minibuf*

If I take it out, things seem to work fine. I've enclosed the
FvwmIconMan section of my .fvwm2rc below. I'm running 2.0.45 on an
AlphaStation 400 4/233 under DU 3.2g. Any suggestions?

*FvwmIconMan*Resolution global
*FvwmIconMan*background PowderBlue
*FvwmIconMan*foreground black
*FvwmIconMan*font 7x13
*FvwmIconMan*action Mouse 1 N sendcommand Iconify
#*FvwmIconMan*action Mouse 2 N sendcommand WarpToWindow
*FvwmIconMan*action Mouse 2 N sendcommand focus
*FvwmIconMan*action Mouse 3 N sendcommand "Module FvwmIdent FvwmIdent"
# *FvwmIconMan*action Select
*FvwmIconMan*action Key q N quit
*FvwmIconMan*followfocus true
*FvwmIconMan*sort true
*FvwmIconMan*usewinlist true
#*FvwmIconMan*drawicons true
#*FvwmIconMan*shape true
*FvwmIconMan*format "%i"
*FvwmIconMan*plainbutton up black PowderBlue
*FvwmIconMan*selectbutton down black PowderBlue
*FvwmIconMan*focusbutton up NavyBlue #E2E200000000
*FvwmIconMan*focusandselectButton down NavyBlue #E2E200000000

*FvwmIconMan*1*managergeometry 1x0-125+150
*FvwmIconMan*1*buttongeometry 125x0
#*FvwmIconMan*1*show resource=emacs resource=gemacs
*FvwmIconMan*1*show title=*INBOX* title=*vm* title=reply*
*FvwmIconMan*1*title "Emacs mail"
*FvwmIconMan*1*iconname "FvwmIconMan: Emacs Mail"

*FvwmIconMan*2*managergeometry 1x0-0+150
*FvwmIconMan*2*buttongeometry 125x0
*FvwmIconMan*2*show resource=emacs

*FvwmIconMan*2*iconname "FvwmIconMan: Emacs"

*FvwmIconMan*3*title "Windows"
*FvwmIconMan*3*iconname "FvwmIconMan: all"
