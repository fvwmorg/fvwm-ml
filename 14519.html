>>>>> Carl A Adams writes:

> The problem shows itself when I first log in when bringing
> up the first new window or two from my .xsession file. These
> windows sometimes do not have any of my styles; the color is
> some default light grey, and I have no buttons in my menu
> bar. [...]

Instead of starting them in .xsession, create an InitFunction in your
.fvwmrc and start them from there. If you must put them in .xsession,
you'll have to use sleep to wait before running the apps to give fvwm
a chance to start up.

> As long as I am on the subject, my $.02 about the config
> file is that I would not like to see fvwm depend on a copy
> dropped in /tmp for runtime. We run tmp cleaners that wack
> old stuff, so if I am logged in for several days this file
> goes away. (I just seem to remember some rumor floating
> about. Donno, maybe I am nuts.)

That's primarily a problem with fvwm 1.x. In 2.x, that requirement
will go away once all the modules are rewritten to use the fvwmlib
interface for reading the configuration. In fact, under 2.x I don't
believe the /tmp/fwmrcXXXXX file is saved at all unless you use FvwmM4
and turn on the debug switch so modules such as FvwmForm will work.
