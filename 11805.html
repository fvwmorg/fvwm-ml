Hello All,

I'm new to the mailing list, so please excuse me and let me know if I'm
breaching protocol for this list or doing anything incorrectly.


I am maintaining a Linux (Slackware 8.1) application in X (XFree86 v
4.2) using the fvwm2 window manager (v2.4.7). This application
creates windows that are hidden when created, but exposed after they
are initially drawn upon. I am having "BackingStore-ish" problems
with these windows using this version of fvwm2.

Using an earlier verson of fvwm2 (v2.2.5), there are no problems with
the application. The windows are created, drawn upon, exposed, and
the initial drawing is retained. However with the new version
(v2.4.7) the original drawing is not retained. However, I can still
draw on the window once it is exposed.


I've tried the following configuration options in X and fvwm2:

- "Option 'BackingStore'" is enabled in my XF86Config.

- "Style '*' BackingStore, SaveUnder" is in my .fvwm2rc file.


I've also tried a newer version of fvwm2 (v2.4.16) and that does not
work either. I'd like to use the new version of fvwm2 (>= 2.4.7) if I
can figure this issue out. Can anyone lend me a hand?
