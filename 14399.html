I tried fvwm2.0-pl36 and compiled it on Linux 1.3.20. (gcc-2.6.3)
With the system.fvwmrc it seems to work fine. However, using my own
.fvwmrc, fvwm hangs and all I can do is move the mouse.
pl33 works fine with this .fvwmrc (I am now using it). pl34 didn't work,
pl35 did not hang but some windows (TECPLOT) did not have an icon in the
FvwmIconBox.

In my .fvwmrc I use xload, xbiff, rclock, FvwmPager and FvwmIconBox
'swallowed' in my FvwmButtons module. I think that's causing the trouble.

My suggestion for fvwm2.0 is that the icons in FvwmIconBox do also highlight
if you are working in the corresponding window (just like in FvwmPager).
