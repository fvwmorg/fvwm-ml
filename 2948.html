INSTALL.fvwm contains more than a one line description for every configure
option and common situations.

> --with-gnome-includes location of GNOME headers
> --with-gnome-libs location of GNOME libs
> --with-gnome prefix for GNOME files (not needed for GNOME WM hints)

The stuff that generates these lines is taken from the standard gnome.m4.
The logic is not trivial, it has no sence to change names without changing
the logic. Since everything works well, there is no reason to change logic.

For most of users the defaults (no gnome options) should be more than ok,
because gnome-config returns the place of includes, libraries and more.
It is also not critical if FvwmGtk is built without gnome-libs.

The problem of the original poster is due to the FreeBSD port that
by default requires GNOME libs. It is not FVWM that requires this.
