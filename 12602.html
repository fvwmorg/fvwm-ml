Ruslan Kosolapov <rkosolapov_at_sw-soft.com> writes:
>
> Some time ago construction "Key -<key> A A <command>" was mean
> "execute <command> when <key> will be released".
>
> Now such construction return "[FVWM][ParseBinding]: <<ERROR>> No such
> key: -<key>"
>
> Question: how to bind action to key release?

That feature was added on 2002-11-14 as part of the 2.5.6 release.
It was removed on 2003-03-09 as part of the 2.5.7 release because
the feature wouldn't work reliably.

> Also in man FvwmProxy I see:
> "Key -Meta_L A A SendToModule FvwmProxy Hide"
>
> But this is not working - error received. Bug in documentation?

Yes, it is a bug in the documentation.

> I read something about it in this list, but I don't remember details
> and can't found no one messages in mailing list archive about it.

Look for the dates above.
