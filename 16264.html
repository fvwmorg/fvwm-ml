I agree. It is nice. It would be nice to keep it. But I gotta be able to
run compound commands.

Instead of having Exec parse the string, maybe we should just have to versions
of Exec. Exec which does the implicit exec, and Run which doesn't. I don't
much like the idea of trying to figure out from the command whether an exec
is possible. First off, you don't know if people are using Exec to send
commands to a real shell. They might be sending stuff something else for all
we know. Secondly, it could get tricky to get right, and trickier to explain
to somebody when it doesn't work. Adding a new command seems simpler, and
might even involve adding less code to fvwm.
