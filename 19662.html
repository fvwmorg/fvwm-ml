In my quest to get all the M_* packets documented for my Perl module, I have
been hunting down M_WINDOWSHADE and M_DEWINDOWSHADE.

These packets are sent when a window toggles between shaded states (duh),
assuming that you enabled shading at build-time. However, these packets don't
really conform to the established standard. Each packet contains only one
value, which looks like the X window ID.

So, why not have these packets return the same identifying triple that
packets like M_LOWER_WINDOW, M_DESTROY_WINDOW, etc. have, or that longer
packets like M_CONFIGURE_WINDOW use as their first three args to identify a
window?

(the code is in builtins.c)
