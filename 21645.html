peteg_at_gatwick.geco-prakla.slb.com said:
> On one login ( the first ) if I double click on (say) file.dat, only
> the "file" part is highlighted but on the second the whole "file.dat"
> is selected.

It's to do with how your terminal emulator is set up. For XTerm, the
charClass property controls this. For example, I've got the following in my
.Xdefaults file (which is read by "xrdb -merge $HOME/.Xdefaults" in .xinitrc:

#if 0
Make "/-~$" parts of words for double-clicking
#endif
XTerm*charClass: 47:48,45:48,126:48,36:48

Other terminal emulators may do it differently. XTerm's is certainly
documented in a file in the source tree; I think it's in the manpage too.

Nothing to do with fvwm, anyway.
