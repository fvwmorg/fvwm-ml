And gensch_at_zk3.dec.com spake, saying:
> On 13 Feb, Wesley Hart wrote:
>
> > It works as you describe. However, I have trouble getting 'Next' to
> > work with FvwmButtons - the following doesn't work as I expect. I
> > get additional instances of FvwmButtons each time the function gets
> > executed. Fvwm doesn't produce any error messages while defining or
> > executing the function. Any ideas why this doesn't work?
>
> A lot of configuration files set FvwmButtons to CirculateSkip, which
> means that "Next" won't find it. Try:
>
> "I" Next ["FvwmButtons" CirculateHit] Goto
> + "I" None ["FvwmButtons" CirculateHit] Module FvwmButtons
>

Looks like CirculateSkip was the culprit - it works now.
Thanks for the tip!
