Using xv 3.10a, fvwm 2.0.40, and Linux 1.2.13 on an i386-33, I verified
the reported bug (select Grab with HideWindows option; updates to xv
window don't happen until an event [reported example: move window]
occurs), but I found other events which triggered an update of the
display window. If I move to another desktop and back, or if I execute a
Recapture, the xv window is updated. If I move the window it is
updated. If I raise or lower the window, it is not updated. I would
guess that fvwm is not giving xv what it expects when it is grabbing in
HideWindows mode. Whether fvwm is giving xv all that it should give it,
and mwm/olwm are giving it extra info which it (incorrectly?) expects is
another question.
