There is the same X detection in 2.2.x and 2.4.x.

This code is generated by autoconf anyway, so we may only do a work around
if the problem is known.

The code is this:

  if mkdir conftestdir; then
    cd conftestdir

I guess (I don't have Tru64) that mkdir failed but still returned
successful 0 status. Then "cd" generates the error you see.
Do you have write permissions in the build directory?
Or mkdir on Tru64 does not return the error status when failed?

Please debug this problem, I don't think it is FVWM related.
