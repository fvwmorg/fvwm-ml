I think this is a large part of your problem.

> ModulePath /usr/local/fvwm

is definitely wrong. You should either remove the ModulePath setting
altogether (highly recommended; fvwm knows where to look!) or, if you
want to find new, nonstandard modules you should use the $FVWM_MODULEDIR
environment variable to represent the default path.

With this setting no modules can be found. Like, FvwmM4 for example.

Pls. see the NEWS file and the man page for more info on this.
