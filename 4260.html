On 12 Feb 2002 08:03:16 +0100, Andre Bonhote wrote:
>
> > And then start X with a one line .xinitrc running fvwm-2.4.5.
>
> Well, guess what: It worked (even with 2.4.4). And the funniest thing
> for me is that it now works with my current RC as well! The only fact
> that changed since yesterday is, that I always 'restart'ed fvwm2 until
> now, but never exited from X and startx'ed again.

Ok, then I have an explanation for you. You ones changed the layer of
logwindow's using something like "All (logwindow) Layer +2" command.
This changed layer of your windows (4 instead of 2) is preserved during
Restart regardless of StaysOnBottom. The solution is to put them back
using Layer -2.

> Sorry Mikhael for the work. But I really can't explain this. I know it
> sounds like the good ol' Windows helpdesk times: "I didn't change
> anything in the system configuration, but it doesn't work anymore".

We are always ready to a hidden, hard to reproduce bug, not just a user
failure. In this case, it is a feature, some people may like to preserve
a layer of windows during restarts that they changed dynamically.

There is always "Restart --dont-preserve-state" if you don't need this.
