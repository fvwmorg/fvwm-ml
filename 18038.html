Hi everyone,

I just subscribed to this list, so I don't know if this is
or has been a thread already...

I'm running fvwm-2.0.42 on NetBSD-1.1 (i386) and XFree86-3.1.2.
While hacking my ~/.fvwm2rc, I discovered that fvwm2 is pretty picky
about how the ColormapFocus is to be set. It turned out that
either exactly ONE space or exactly ONE tab must be used between
ColormapFocus and FollowsFocus or FollowsMouse. Otherwise, the code
in fvwm/builtins.c (procedure SetColormapFocus) doesn't find
either keyword. I'll dive into the code once I know this really
is a bug, and hasn't been fixed in the meantime.
This version of fvwm compiled right out of the box here, with
only four warnings about uncast pointer-integer assigments
in modules/FvwmForm/FvwmForm.c (all 'GetFontOrFixed' calls).

Great window manager, by the way; must be running it for over
2 years now.
