%l kernel >= 1.2.13, libc 4.6.27, XFree 3.1.2

Destroying modules makes the X display hang.

No customization, vanilla system.fvwmrc. I discovered this behaviour
while I loading a second ButtonBar via the Popup-Menu.

To reproduce the bug: choose "Destroy" from the menu and click on the
ButtonBar. fvwm wipes the ButtonPictures away but leave a somehow red
background. Now no input via mouse or keyboard isn't possible.

It also hangs, if I try the Banner module; when the Banner thinks it's
time to disappear, it freezes the display.

If you need more info, please aks. Best whishes, Karl.
