> I am running FVWM 2.0.46 and xterm 3.3.2.3 under FreeBSD 3.0.

[interesting stuff about active icons]

> It says in the man page that some window managers will allow you to type
> into the mini-screen icons. For example, twm will let you. But FVWM
> won't let me do this. Is it possible to set it up so that I can?

Under Linux, it seems to vary from app to app. I iconified xev and
rxvt, and both still get key evens when the icon has focus. But xemacs
doesn't - or perhaps, more accurately, it seems to ignore the events
when it is iconified. The "stock" xterm that comes with Debian Linux
also doesn't handle key events when the window is iconified.

I also ran FvwmIdent against xterm and rxvt while iconified. Both
report

Focus Policy: Passive
  - Input Field: True
  - WM_TAKE_FOCUS: Absent

> I hope that this is the correct place to send this message, but I don't
> know of any xterm mailing lists, and it works under twm.
