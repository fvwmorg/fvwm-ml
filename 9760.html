Mikhael Goikhman wrote:

>This is not currently possible. But why don't you just define 2 menu
>versions (they will be more readable then one bulky menu you suggest)
>and fire them on different key bindings (for example one with Alt-F5
>and another with Shift-F5):
>
> Key F5 A M Menu MySSHMenu
> Key F5 A S Menu MyTelnetMenu
>

  This would work. It isn't ideal. but... The comments on
complexity below still apply, however.

>With small tricks you may even define only one menu version, like this:
>
> Key F5 A M OpenMySSHMenu
> Key F5 A A OpenMyTelnetMenu
>
>[...]
>Not tested, but this technique should work.
>

  That wouldn't work for me, really. I don't want to change *all* of
the items in the menu. Just one or two. So, that would be a heck of
a lot of work just for part of a menu. I guess that it actually would
*work*, but is a lot of effort.

  Also, any of this would only allow me to get it if I thought of
it before opening the menu or submenu. And because I have
all of this built with M4 anyway, it'd be much more complicated
to do it that way.

  Can I request that this sort of feature be added in the future?
Having the menu entries change when modifiers are activated
and disactivated is kinda difficult, and configuring it is no small
feat either, but... It would be very useful in the situations where
it's desired. And doesn't hurt anything (except total code size
I guess) when it's not used...

                                             - Chris
