On Sun, Mar 02, 2003 at 03:10:19PM +0100, Jan Johansson wrote:
>This has changed between 2.4.8 and 2.4.14.
>
>When using 'Style * NoIcon' (or NoIconTitle) iconified windows
>no longer show up in FvwmIconMan.

For the archives.

The important part that changed is the default of '*FvwmIconMan:
resolution'. It used to be 'global' but is now 'page'.

Iconified windows with "NoIcon" defined does not stay on the
current page and as such does no show in the icon manager.

Setting '*FvwmIconMan: resolution' to 'global' or 'desk' make the
iconified windows show again.
