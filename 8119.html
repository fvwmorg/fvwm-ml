On Sat, Mar 01, Mikhael Goikhman wrote:
> On 01 Mar 2003 11:30:03 -0800, Tenebrae wrote:
> >
> > On Sat, 1 Mar 2003, Mikhael Goikhman wrote:
> >
> > > On 01 Mar 2003 16:20:23 +0530, Kingsly John wrote:
> > > >
> > > > +++ Michael Fischer [2003-02-28 20:37:02]:
> > > >
> > > > > What I want is a 1 column, 4 row FvwmButtons, where
> > > > > (counting from top to bottom), the sizes are
> > > > >
> > > > > 68x68
> > > > > 68x68
> > > > > 68x252
> > > > > 68x68
> > > >
> > > > I think Fvwm requires all the buttons to be in some kind of ratio.
> > > > (ie,252 is not a multiple of 68)
> > >
> > > Nothing like this is required. It is only that if it is a multiple of 68
> > > all proportions may be reduced to 1x1 and so on, but this is optional.
> > > The above may be reduced as well: 17x17, 17x63.
> > >
> > > I suggest to use ButtonGeometry to specify the exact size of 1x1 button.
> > > For example try:
> > >
> > > *FvwmButtons: ButtonGeometry 4x4-0+0
> > > *FvwmButtons: (17x17, ...)
> > > *FvwmButtons: (17x63, ...)


*MailBox: ButtonGeometry 4x4-1+1
*MailBoxBack black
*MailBoxColumns 17
*MailBox: (17x17, Swallow(UseOld,NoHints,Respawn) "xeyes" \
        `Exec exec xeyes -geometry 68x68`)

*MailBox: (17x17, Swallow(UseOld,NoHints,Respawn) "xclock" \
   `Exec exec xclock -bg black -fg turquoise -geometry 68x68 \
   -hd turquoise -hl turquoise -padding 0 -update 1`)

*MailBox: (17x63, Swallow(UseOld,NoHints,Respawn) "gkrellm" \
        `Exec exec gkrellm`)

*MailBox: (17x17, Swallow(UseOld,NoHints,Respawn) "kbiff" \
        `Exec exec kbiff -profile Inbox`)
