Dan Espen <dane_at_mk.telcordia.com> wrote:
> Bruce Burden <bruceb_at_austx.tandem.com> writes:
> > Hi all,
> >
> > I just got fvwm2 running on my Sun/Solaris and CDE 1.2. However,
> > I can't find a pixmaps directory. So, apparently, I can't use toys
> > like xclock, xbiff, and who knows what else.
>
> I've never seen any xpm files shipped with Solaris.
> Solaris does ship with black and white bitmaps which fvwm2 will use.
>
> Use this command to find them:
>
> find /usr/openwin -name '*.xbm'

Wrong directory -- if you install the CDE desktop stuff, Sun ships tons of
xpm files in /usr/dt/appconfig/icons/C -- all those *.pm files are xpm files
for all the cde stuff.

> > Is there some way to simply get rid of dtlogin/CDE, and simply
> > use a nice "startx" program? No, I really don't think CDE is a "good
> > idea".
>
> Caution, second-hand information follows:
>
> cd /etc/rc2.d
> mkdir tmp
> mv S99dtlogin tmp

Bad idea. Instead (as root) run:

        /usr/dt/bin/dtconfig -d

which will disable the CDE desktop auto-start, and then TO KILL YOUR CURRENT
CDE LOGIN SESSION ALTOGETHER, run

        /usr/dt/bin/dtconfig -kill

Of course, you might be better off adding fvwm2 to your list of available
desktop managers so that if you needed to run CDE for something (???), you
still could... but to each his own...

Hope this helps,
