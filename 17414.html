Stephen Wray <Steve.Wray_at_vuw.ac.nz> writes:

> I used to rely on FvwmWinList to warp the pointer to totally obscured
> windows. As of 2.0.43 this is no longer the case. WHATS MORE it doesn't
> seem to have an option to do this!!! What happened? How can I fix this
> *abberant* behavior???

You could try FvwmIconMan. It's like FvwmWinList, but I think it's better,
but I'm predjudiced since I wrote it.

Or, if you want to stay with FvwmWinList, which I admit is better for
transient use, I bet that the problem comes from the change in the fvwm Focus
function which makes it no longer warp the cursor. If you use the
FvwmWinListAction option to bind WarpToWindow to a click event, that'll
probably work.
