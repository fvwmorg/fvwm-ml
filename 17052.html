Hi all,

I'm using a Sparcstation 2 with Solaris 2.5 and OpenWindows 3.5 installed.
Recently, I compiled fvwm-2.0.42 and installed it (I used gcc 2.7.2).
The compilation and installation went smoothly enough, including such
niceties as rplay support, etc. However, when I ran it (from my .xinitrc
file), it started acting strangely. Before I describe what's wrong,
let me just say that I'm using the sample fvwmrc file supplied with
the package, twiddled somewhat for my site (the module, pixmap and bitmap
paths are different).

The problem is that, with the exception of surrounding borders and the
title bar, no other decorations are being put on any windows. What's
more, the mouse doesn't have any effect on the windows. Actually, this
is slightly misleading - I can use any mouse support that the application
has (such as cutting and pasting or xterm menus), but cannot use it
to access any fvwm features such as window dragging and resizing or
drop-down menus.

The first time I started fvwm2, I couldn't even exit X (I had no xterm up
and no way to get one either) so I had to reboot... Occassionally, when
I started fvwm2, I couldn't actually type into the xterm that came up -
however, I could still cut-and-paste to and from it, so I had to
cut-and-paste garbage from my bash$ prompt so that bash gave me an error.
This gave me more letters that I could 'type' with, and eventually I
was able to do a 'ps' and a 'kill' on my fvwm process. It was better
than rebooting, I suppose... :-)

It seems to me that no functions are being executed (the InitFunction
doesn't start either).

Has anyone else happened across this problem? Anyone got a solution?

Regards,
 Robert.
