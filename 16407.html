Maybe you are suggesting that similar options be available for the
titlebar buttons as well. This is somewhat understandable, and might
be a good idea. I'm in a bind this and next week, but I want to get
Charles some code optimizations and reductions first and see what he
thinks. BTW, it would be fairly easy to add as you suggest. The only
part is to reduce the number of needed style types to a minimum.

There could be an added button style or a few, which after the code
reduction should only incur a marginal code increase. However, the new
styles would have to be chosen carefully, and it might be best that it
is tested as a patch against 2.0.42 first so we can get feedback.

Regarding the maximize problem, the code says if MWMButtons is
specified than only the rightmost button gets reversed polarity when
maximized. An additional method might be to have extra style flag
bits for buttons (which would not increase structure size). What
comes to mind is to assign preferred buttons a button style flag of
MWMButton, which will act so on windows which have the MWMButtons
style flag.

The maximize problem would need to be worked on first. It is due to
only a few lines of code, and I am going to suggest a solution to this
(which also adds flexibility) Charles about this and see what he
thinks (he is working on something also). It involves using added
style flags for the button, while not increasing code size (--
actually, I wouldn't be surprised if Chuck has already done something
like this :). My god, I'm needlessly rambling again, sorry!
