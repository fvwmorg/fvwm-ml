I'm sorry you see it this way. The *sole* goal of my posts on this
subject has been to get the default .fvwm2rc file be set up for
ClickToFocus. Though I've been dragged into a little bickering with
the pointer focus zealots, even that has been just to emphasize the
point that this is not an issue that can just be ignored and that the
decision about the default is an important one.

> In any case I reiterate my _request_ that you keep the discussion strictly
> relevant. I believe that pointer focus is the default in most TWM
> derivatives (it certainly is for the TWM and TVTWM most of my users use),
> so you're probably better off complaining in a general group instead of
> missing most of the audience.

But I'm not suggesting we change any other WM: either they use
explicit focus by default or don't have a sufficient installed base
that it makes any difference. FVWM, mostly by virtue of its
popularity on Linux systems, is becoming common enough that the number
of bug reports we get relating to it has become an issue for us. This
also means that it's installed base is large enough that it's time to
start thinking about what's best for the largest number of users.

Unless you're willing recommend we change the default, or argue that
default focus policy doesn't matter, I think further discussion on
this issue is warranted.
