> just a quick question: I just installed fvwm2.0.46 and noticed a slight

> double-clicking to select text results in different selections under ncdwm
> and fvwm. Specifically, it looks like ncdwm has more "leniency" in
> selection text. It seems like (if this is how it's done), the regexp in
> fvwm doesn't include things like periods ("."), etc.
>
> I guess the question is: is this a wm related problem, or something else
> entirely. If it is fvwm related, I'm assuming the regexp would be simple
> to change to my liking, and recompile, but, what part of the code is it

Hi

This is totally unrelated to the window manager. The X server administers
the selection clipboard, and individual apps and toolkits are responsible
for selection policy.

In xterm this policy is set as the Xresource charClass. Read the section
labelled "CHARACTER CLASSES" in the xterm man page.
