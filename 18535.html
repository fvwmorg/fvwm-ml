    lar3ry> One comment that I can see coming up is that maybe have
    lar3ry> some support for a tile or cascade 'region' -- for
    lar3ry> instance, many fvwm95 users have a task bar on the screen
    lar3ry> that can get obscured by these modules. In addition,
    lar3ry> there are lots of people with button bars, pagers, and
    lar3ry> whatnot that they don't want obscured.

Are you sure this is not handled with the command line parameters for
each module? I think it already does this.

    lar3ry> This would limit the region in which the windows are
    lar3ry> placed to the rectangle defined by the corners (0,20) and
    lar3ry> (1280,1004), leaving the 20 pixels from the top and 20
    lar3ry> pixels from the bottom clear of any added windows
    lar3ry> (assuming a 1280x1024 screen size).

FvwmTile 0p 20p 1280p 1004p
FvwmCascade 0p 20p 1280p 1004p

I think these will do what you want. You read the man page? I'm
just curious since I thought it was stated fairly clear there, unless
this is something completely different.

btw, I just put 0.8a on there which omits Sticky windows unless they
are activated by a flag...
