No I do not have a libXpm.a . Also you probably do not need a libXpm.a .
I'm fairly sure that someone has already said this, but files with a ".a"
suffix are for building executeables with the library "staticly
(sp?) linked", while ".so" files are dynamicly linked libraries. All you
should need is a ".so" file unless you are trying to do some sort of code
devel. So, all in all, you probably just needed to do the install and run
ldconfig.
