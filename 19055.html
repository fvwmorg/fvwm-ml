On Fri, 24 Jan 1997, Christopher Wolf wrote:

> Not only do I see CleverPlacement avoiding the space once occupied by
> iconified windows, but if I open a series of small windows on a
> almost-blank screen, I notice CleverPlacement likes to stack them
> against the most-right and most-bottom areas of the screen, making
> them all look like the classic wind-blown snow in a large pane of
> glass view.

CleverPlacement places windows so that there is a minimum amount of
overlap, so since you have (or had before iconifying) windows in the upper
left corner of the window, the place with the least amount of overlap is
in the lower right corner. The real bug is that CleverPlacement doesn't
see that your window is iconified. It seems that somehow the ICONIFIED
flag isn't getting set.

Again, I can't duplicate this myself, so if anyone who sees this problem
can send me their .fvwm2rc file that would help greatly. (Hey, this is a
great way to collect some cool .fvwm2rc files :-))
