> > > rw> Are there any plans to add side pixmap support for menus, sumilar to

> > >
> > > It's already in there, in the latest betas. It'll be in the 2.2
> > > release, which should be out "soon" (really, though, we mean it this
> > > time!) The man page sez:
> > >
> > > AddToMenu menu-name [ menu-label action ]
> > > ...
> > > If the menu-label contains a sub-string which is set off by at
> > > signs ("_at_"), then the text between them is expected to be the
> > > name of an xpm or bitmap file to draw along the left side of the
> > > menu (a "side pixmap"). For example
> > >
> > > AddToMenu "StartMenu_at_linux-menu.xpm@"
> > >
> > > creates a menu with a picture in its bottom left corner.
> >
> > And of course you can have it a bit easier with the CVS source
> > for a few hours now: 'MenuStyle * SiedePix linux-menu.xpm'
