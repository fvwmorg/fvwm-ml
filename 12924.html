> I spent a few hours today getting my dual monitor setup under Linux
> (Debian Woody) and XFree86, with fvwm2.
>
> What I've managed at this stage is the two monitors being treated as
> seperate screens, running fvwm2 with thw -s switch so it only starts on

> The upshot of this is two monitors work seperately, I can drag windows

> Is there any way to do this currently? If not consider it a feature

(That was posted on April 3.
 A similar question -- "xinerama, but disjoint virtual desktops" --
 was posted on April 13.)

I was in the same dilemma running Openbox:
I wanted my screens tied together with Xinerama;
but I also wanted them to page independently.
So I scripted a simple pager to do the trick.
It works well for me.

It should work (maybe with some tweaking)
on any WM that is EWMH-compliant.
I have not tested it with fvwm.

Details are at:
http://www.zelea.com/project/screenpager/home.html
