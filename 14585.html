When executed 'Recapture' function, fvwm losts to see WM_HINTS hints.

You may eproduce this bug in a following way (I checked with pre2.0pl37):

1) Start fvwm.
2) do 'xterm -iconic'. This acts well, i.e., starts with iconic state.
3) exit xterm executed in 2).
4) do 'Recapture'
5) do 'xterm -iconic' again. But xterm starts with normal state.

Sorry I've not found where is a bug in fvwm source. Add to BUGS list ?
or easy to fix ?
