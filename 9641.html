On 06 Aug 2003 16:18:08 -0400, Adam Woodworth wrote:
>
> I've searched google, newsgroups, and #fvwm (unfortunately the fvwm
> mailing list appears to be unsearchable), but haven't found an answer to
> this:
>
> If I have an instance of mozilla running, and then I launch another
> instance of mozilla, the new mozilla window does not have my rightmost
> titlebar button (which is my Close button). This happens regardless of
> what my rightmost titlebar button is.
>
> If I open a new mozilla window from the first mozilla instance, then all
> the titlebar buttons are there. It's only when I execute the mozilla
> binary again does this problem appear.
>
> I'm using FVWM v2.4.6 and Mozilla 1.0.0 on Debian Linux 3.0.
>
> Using DecorateTransient doesn't fix this, and I can't figure out how to do
> fix it any other way.
>
> Any help would be greatly appreciated!

I neither can understand how this may happen with the main
(non-transient) browser window nor may reproduce this with my mozilla,
but I think this line should fix it:

  ButtonStyle All - Clear # remove all default titlebar button hints

Optionally, you may also want to setup correct hints for your buttons
(the default hints are for 2=Maximize, 4=Iconify, not what you have):

  ButtonStyle 1 - MwmDecorMenu
  ButtonStyle 6 - MwmDecorMin
  ButtonStyle 4 - MwmDecorMax
