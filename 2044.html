On Fri, Feb 02, 2001 at 04:00:40PM +0100, Kristian wrote:
> Hi!
>
> I have found a bug introduced in fvwm-2.3.27 concerning the KDE 1.1.2
> File Manager (started with "kfm -w"). If I right-click on a file and
> then on "Properties" the Popup-Window will stay lowered behind anything
> else.

Yikes! That's a bug of kwm. The popup window sets the
transientfor hint to indicate it's transient nature, but the hint
points to itself instead of its 'parent' window. Thus fvwm
chooses layer 0 to put the window on. I have coded a workaround
in fvwm. Could you please report this to the kfm developers?
