Yeah, it was too big for me too, I just deleted it.

I'm running Fvwm on a 400Mhz machine and I can opaque move
windows just as fast as the pointer moves. That's using either the
title or any other part of the window. I can't see any kind of lag.

Redeeman should see if he can reproduce this with a minimal
config. If so, I'd suspect some issue with saveunders or
backingstore. If not, he needs to trim down the config file
to find what is causing this. The posted config file looked
like it might be a fvwm-themes setup.
Maybe he can just tell us which theme this is.

I guess it's possible he has some kind of special binding for
move from the titlebar. If you do this in FvwmConsole:

Mouse 1 T A FuncFvwmRaiseLowerX Move

does moving get better?
