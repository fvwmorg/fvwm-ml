I'm having an odd problem with fvwm and a broken color map. I've
compiled fvwm 2.2 with xpm support on Solaris 2.6. The machine I'm
running it on is an Ultra 5 with a mach 64 derived video card (I can't
remember the specific model name). It has 4MB of video memory, and I
run it true color mode.

When I start fvwm2, some of the colors (but not all) are inverted.
Colors like black and white, and red and cyan, are switched. The
problem goes away when I run netscape -install and put the mouse
cursor inside the netscape window.

I don't think it's a problem with my X setup, since twm will run fine
without the color inversion. Also, the problem exists with the
default .fvwm2rc, and my own .fvwm2rc.

list at the moment.
