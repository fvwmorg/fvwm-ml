Graham Gough wrote:
>
> Is there any way in the latest fvwm to use styles to have autoraise
> turned on/off selectively for different classes of windows?

        Short answer: presently, no.

        Longer answer: maybe, or maybe later.

        Someone suggested trying the ClickToFocus style. This depends on how
you use your desktop. If you view the focusing and raising of windows as
one indivisible action, instead of two discrete ones, it might give you
an approximation of what you're looking for. The windows to which you
give the ClickToFocus style will not be automatically raised when the
mouse cursor rests over them; they also will not be automatically given
focus - you will have to click them to focus and raise them.

        If you want certain windows to be given focus but not raised when the
mouse cursor is over them, and other windows to be both focused and
autoraised, then you're out of luck, for now.

        To implement the latter would require adding an additional style, which
can't be done till after the Great Style Flag Rewrite (GSFR). It would
also require one of a) a modification to the module interface, or b) a
new core FVWM command like AutoRaise (raise this window only if it has
the AutoRaise style). I would opt for approach b.

        Would you mind entering a request for this feature into the FVWM bug
tracking system, so it doesn't fall through the cracks?
