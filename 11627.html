Hello everybody,

I have the following line in my .fvwm2rc:

MenuStyle black grey76 grey30 -adobe-helvetica-medium-r-*-*-*-120-*-*-*-*-*-* fvwm


According to the man page:

------------------------------------------------
MenuStyle stylename options
     Sets a new menu style or changes a previously defined style.
------------------------------------------------

As i understand it, you don't *define* a new menu style with that
command, but only set it (that is, if till now menu style was "x"
and you want to change it to "y", you say: "MenuStyle y". Also, style
called "y" have to exist in order for this to work, because MenuStyle
could not create new styles)

*) Contrary to my understanding, using "MenuStyle mwm", for example,
from FvwmConsole does not set menu style - but why? The only way i
found to change something in a menu style is to use "MenuStyle * <options>".
Is it supposed to be so or did i miss something?

1) How do you define a new menu style - do you just copy one of the
existing menu styles into a new one with, say:

"CopyMenuStyle fvwm my_style"

creating that new style along the way and than change some of it's
properties? (Of course if the answer to previous question is that
there is only one changeable style - "*", the question is irrelevant,
though an existence of CopyMenuStyle suggests that many menu styles
can be defined)


2)The line from .fvwm2rc i mentioned at the beginning, what it is
supposed to do? Apparently, it does not contain stylename's name -
"black" is the name of color, as are grey76 & grey30. Also, it is
said in the man page that "options is a comma separated list
containing some of the keywords", and the mentioned line contains
only values for some of the properties without properties' names, so
seemingly it is not structured correctly.


ddtl.
