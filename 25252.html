----- cut -----
Style "FvwmPager" NoTitle, NoHandles, Sticky, WindowListSkip,
BorderWidth 0, NakedTransient
Style "FvwmButtons" NoTitle, NoHandles, Sticky, WindowListSkip,
BorderWidth 5, StaysOnTop
Style "FvwmButtons" SloppyFocus
*FvwmButtons(2x1 Swallow "FvwmPager" "FvwmPager 0 0")
*FvwmEvent focus_change "function showhidebuttons"

AddToFunc showhidebuttons
 + "I" prev (CirculateHit FvwmButtons) Move -1019p -698p
 + "I" current (FvwmButtons) Raise
 + "I" current (FvwmButtons) Move +0p +0p
----- cut -----

This will put the pager in fvwmbuttons on the left top corner (on my
1024x768).
If the pager isnt active, only a few pixels will be visible.
Moving the mouse over the pager moves full pager on the desktop.
Now my problem: if fvwm2 changes the focus, the mouse cursor sometimes
is around a second busy.
No input is possible.
Has anyone a idea?
