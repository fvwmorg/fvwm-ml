Jason Kibblewhite wrote:
>
> ...I'm unable to get fvwm to read the changed menus using
> DynamicPopupAction.
>
> If I do a restart it reads it the first time but it never
> changes if I open the menu again, and I know that the info
> has been updated.

        I've not tried this feature, so I really don't know what I'm talking
about (so what's new?), but if I'm reading the man page right, here's
what's wrong.

        The first time you pop it up, you're destroying the DynamicPopupAction
along with the menu; from there on, it will never be executed, and
you'll just keep seeing the same old stuff.

        Try adding the optional "recreate" keyword to your DestroyMenu (in
.freshmeatmenu or whatever) and see if that corrects the problem.
Alternatively, right after the DestroyMenu, you need to add another
DynamicPopupAction command.
