Todd Fries writes:
> You know, if you really wanted to make tkgoodstuff irresistible and live
> up to it's name, someone should implement a configuration dialog for
> fvwm itself, so we could configure fvwm on the fly and then save it to an
> rc when we liked what we saw.

> That would be killer!

Ok, go for it! ;-)

Actually something like that has crossed my mind. Configuring styles,
options, and some modules ought to be fairly straightforward using the
tkgoodstuff configuration utility widgets. Indeed, there's no reason to have
much of an .fvwm2rc, since tkgoodstuff could send fvwm most of the rc file
commands.

But the tkgoodstuff configuration tool now does not do "on the fly"
configuration---it requires restarting tkgoodstuff. Anyway, to do a
reasonably general thing would be a lot of work, and right now I'm aiming for
a relatively robust version of the current feature set to be ready by when
fvwm2.1 is released (all bug reports welcome).

But I'd welcome a collaborator to work on this idea.
