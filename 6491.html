Hi Mikhael, et al.

> You say that you don't have any icon for Eterm at all or you have the
> wrong icon? Eterm supplies its own icon picture and you should see it.
> If no, something is wrong here.
>
> Anyway, if you want to override the icon that Eterm supplies do:
>
> Style Eterm* IconOverride
>
> BTW, Eterm is not the only application that provides icon picture hint.
> If you use unified icon set and don't want applications to add chaos
> (every application may have its own idea how its icon should look and of
> which size), you may do:
>
> Style * IconOverride
> Style XTerm NoActiveIconOverride # enable: xterm +ai
>
> Tell us whether this solved your problem or not.
>
> Regards,
> Mikhael.

Well, here's how I made out:

- I inserted the Style Eterm* IconOverride line with no avail. :(
  Still No icon whatsoever
- I deleted ~/.Eterm
- Re-ran Eterm, and configured it they way I like.
- Minimized Eterm - got the icon back!
- But... Although I have the icon back, it is still not sticking to the iconbox
  like all my other clients are... Eterm icon just goes to the top left of
  the root window. :(
