"mm" == Marco Mariani <m.mariani_at_imola.nettuno.it> writes:

mm> Does the -m4-squote option works? I've not been able to use it,
mm> and the example in the man page doesn't work.
  I don't know. I haven't tried this. As someone has already pointed
out, the man page example needs to use `fvwm -cmd' not `fvwm -f'.


mm> Is there a way to refer to $HOME within FvwmM4 or Fvwm ?

mm> I'd like to include(`~/.fvwm-whatever'), because I often launch X
mm> from a directory that is not $HOME.
  The FvwmM4 manpage shows `HOME' as a pre-defined symbol in the
EXAMPLE PROLOG, but does not mention it in the CONFIGURATION OPTIONS
section. I just tested which section is correct. It turns out `HOME'
is pre-defined.

  You could use

    m4_include(HOME`/.fvwm-whatever')
