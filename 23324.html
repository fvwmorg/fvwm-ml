  OK, I have to admit to not having a clue what you mean by 'CVS' - I can
see nothing by this name in the Web or FTP sites, nor the FAQ.

  /However/, I *am* now running the 'development' 2.1.6 I found on the FTP
site, and my Netscape placement problem's been solved! I can now start
'netscape -iconic', with 'StartsOnPage 0 1' and everything comes up in the
right place.


  Here's the 'but'. I'm still having a problem that I'd not mentioned
before, with starting Solaris's 'clock' & 'cm'. Now, I used to start these
command-line iconic, but I now apply the 'StartsIconic' style (their
WM_CLASSes are set appropriately).

  However, they don't appear in the correct place as specified on their
command line:

   toolwait clock -geometry 132x70-9+143 -WP 1078 9 +Wn
   toolwait cm -geometry 877x419-80+50 -WP 1081 80

  They both come up at (0,0) ... until clicked upon, when they run over to
where they're supposed to be!? Actually, it's a bit worse than that - that
was with the iconic parameter; with clock having 'StartsIconic', it's icon
is a blank white box till I double-click open it & re-iconise it. Hence:

   FvwmCommand 'Function Bounce-Icon clock' 2>&1 > /dev/null
   FvwmCommand 'Function Bounce-Icon cm' 2>&1 > /dev/null

  ... in my start-up script, and:

+ "I" Function Bounce-Icon clock
+ "I" Function Bounce-Icon cm

  ... in my RestartFunction, with:

AddToFunc Bounce-Icon \
   "I" Next [$0] Iconify -1
+ "I" Wait $0
+ "I" Next [$0] Iconify 1

  Not nice, but it works!


  The other potential work-arounds are: don't give icon locations on the
command-line (with netscape, I gave a leading IconBox style of exactly the
corrds to use), and don't use 'em! (I guess I could get what I want from
FvwmButtons, but I've not yet looked into that at all).
