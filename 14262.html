please excuse me if you've seen this before, mail has been acting up
here lately.

It's been some time since I've looked into AutoRaise under fvwm 2.0.
>From then, I remember that AutoRaise couldn't be enabled on a
per-window basis, it was all-or-nothing.

Now, I'm a great fan of AutoRaise on a per-window basis, and now that
AutoRaise is separated into a module, I think I could actually
implement this myself (including AutoLower, while I'm at it).
However, I don't know how to do this. In particular, how do I make a
module that somehow receives enter window and leave window
notifications including the window id?

Is there any precedent for modules that maintain a per-window state
during their execution, part of which is determined statically on
start up (such as: always auto-raise XTerm windows), part of which
changes during run-time (such as: I want this window here to
auto-raise from now on)?

I'll be grateful for any information regarding already implemented
features like this, or examples I could look at, or documentation I
could download.
