"Stephen P. Hill" wrote:
>
> Button 3 is too complicated and I don't see that there is any syntax
> that will handle it. If you could combine conditions then it
> could look similar to the following:
> AddToFunc NSMailFunction
> + "I" Next (MailThread) Iconify -1
> + "I" Next (MailThread) focus
> + "I" None (MailThread) Next(Netscape) Exec exec netscape -remote 'openInbox'
> + "I" None (MailThread) None(Netscape) Exec exec netscape -mail
>
> of course, that is not valid syntax.

Actually it is valid syntax (you might need a space between the next and
the "(" though) and it does work. Here's my functions for controlling
netscape. Note that netscape is not very good at handling mutiple remote
commands and sometimes loses them.


DestroyFunc my_netscape
AddToFunc my_netscape I Next (Netscape) Exec netscape -noraise -remote
openBrowser
+ I None (Netscape) Exec target netscape
+ I None (Netscape) Wait Netscape
+ I Next (CurrentDesk Netscape) my_flipfocus

DestroyFunc my_url
AddToFunc my_url I Next (Netscape) Next (Navigator) Exec netscape
-noraise -remote "openURL($0, new-window)"
+ I Next (Netscape) None (Navigator) Exec netscape
-noraise -remote "openURL($0)"
+ I None (Netscape) Exec target netscape $0
+ I Wait Netscape
+ I Next (CurrentDesk Netscape) my_flipfocus

DestroyFunc my_mailbox
AddToFunc my_mailbox I None (MailThread) Next (Netscape) Exec
netscape -noraise -remote openInbox
+ I None (MailThread) Next (Netscape) Wait
MailThread
+ I Next (Netscape) Exec netscape -noraise -remote
"openURL($0)"
+ I None (Netscape) Exec target netscape $0
+ I None (MailThread) Wait MailThread
+ I None (CurrentDesk MailThread) Next
(MailThread) MoveToDesk 0
+ I Next (MailThread) my_flipfocus

DestroyFunc my_mail
AddToFunc my_mail I my_mailbox IMAP://mailhost/Inbox

Cheers,
