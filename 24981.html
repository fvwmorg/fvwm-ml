On Sun, Oct 31, 1999 at 04:41:39PM -0500, dane_at_mk.telcordia.com wrote:
> Ilia Chipitsine <ilia_at_cgilh.chel.su> writes:
> > -----BEGIN PGP SIGNED MESSAGE-----
> >
> > Dear FVWM-team,
> >
> > I'm currently using fvwm2. What I exactly want is a window
> > manager which uses as less memory as possible (fvwm won that competition !)
> >
> > How to make it use even less memory ?
> > I want just manager with xpm and GNOME support, I do not want modules,
> > virtual desktops, buttons, etc. can I save some memory that way ?
>
> You don't say what release you are using. With 2.2.2,
> "configure" gives you some choices:
>
> --disable-modality disable modal dialogs for applications
> --disable-saveunder disable saveunders for pop-up menus
> --disable-shape disable shape extensions
> --disable-activedown-button disable ActiveDown button state
> --disable-inactive-button disable Inactive button state
> --disable-mini-icons disable MiniIcon style option
> --disable-vector-buttons disable vector button style
> --disable-pixmap-buttons disable pixmap button style
> --disable-gradient-buttons disable gradient button style
> --disable-multistyle disable stacked button styles
> --disable-extended-titlestyle disable styled title-bars
> --disable-borderstyle disable BorderStyle command
> --disable-usedecor disable UseDecor style option
> --disable-windowshade disable WindowShade function
>
> I can't say how much that will save you.

Only very few kilobytes (if at all). Most of these options will be gone
with the 2.4 release. I guess the best way to save even more memory is
to use an old release with a smaller binary. I'd try one of thr 1.24x
releases. If flexibility is not the thing you want the best place to
save memory is the size of the executable.
