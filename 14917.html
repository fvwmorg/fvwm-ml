Has anyone else seen any problems in clients calling XGrabButton or XGrabPointer
under fvwm? Whenever any client managed by fvwm calls eg. XGrabButton() it
reliably fails with a BadAccess XError (for me at least). Notably proctool
(solaris) and recent beta versions of netscape seem to do this, as well as my
own code making similar calls.

I'm running the 3.4 sun openwindows dps server with fvwm 2.0.39, but I get this
behavior regardless of which server I run (behaves same with X11R6/fvwm) --
it seems to only happen when I use recent fvwm 2.x versions, not with any
other window managers (olwm, twm, mwm). If nobody else has seen this I'll try
to track down what may be causing it in the code, but it'd be nice to get any
insights anyone may have first...
