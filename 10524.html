On Thu, Nov 06, 2003 at 03:38:24PM +0100, Marcin G?ogowski wrote:
> Hallo,
> Today I tried the OpaqueMoveSize and I noticed very strange behaviour.
> On my old Virge GX the windows with OpaqueMoveSize 100% are refreshed
> after move (maybe because of the speed of the card).
> On my new Radeon 9600 the windows are refreshed all the time, which
> cause very hard window moving.
> Is there a option in fvwm to refresh window contents after move?
> (such option is for example in sawfish).

It's the X server that refreshes the windows. If you have XFree,
try enabling Backing Store with the +bs option to xinit. If that
does not help, you try placing the feedback window during motion
in the top left screen corner with

  Emulate Mwm

or disable it altogether with

  HideGeometryWindow
