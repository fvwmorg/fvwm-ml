> > I've been lurking for a while now, and have the following question:
> >
> > I put a picture on the root window via the following:
> >
> > xv -rmode 5 -quit -root ~/bgrounds/hchaos.png
> >
> > I also start a transparent aterm via .xinitrc
>
> I use "wmsetbg" to set the desktop background. It seems to
> set the background in a different way to xv or fvwm-root
> since it changes the background of aterm, gnome-terminal
> automatically.
> It seems also to be stronger than xv and fvwm-root:
>
> aterm -tr
> wmsetbg image1.jpg
> # aterm bg changes to image1.jpg
> fvwm-root image2.jpg

This would be a future invocation like, but currently fvwm-root does not
suport jpg. Supposing this is a type, you should use:

  fvwm-root -r image2.png

> # aterm still got image1.jpg
> gnome-terminal --transparent
> # gnome-terminal uses image1.jpg as background

Then these terminals should use image2.png, I guess.
