On Tue, Nov 09, 1999 at 10:29:34PM -0600, matt_at_lehi.tamu.edu wrote:
> Hi,
>
> I recently installed the `potato' version of fvwm on my Debian box:
>
> % fvwm -version
> [FVWM][main]: Fvwm Version 2.2.3 compiled on Nov 2 1999 at 23:50:00
>
> I had been running 2.0.46 before.
>
> Now it seems that I can't get windows to come up where I want them
> to. I had been starting Netscape on another page by doing this:
>
> AddToFunc InitFunction ...
> ...
> + "I" Exec /usr/X11R6/bin/netscape -geometry 711x563+877+609
> ...
>
> Which put it on page 1 1 on my 800x600 screen. Now it is coming
> up on page 0 0.
>
> I've tried SmartPlacement, RandomPlacement, SmartPlacement *and*
> RandomPlacement, and even SmartPlacementIsReallySmart, but I can't
> seem to get the behavior I want.
>
> I'm sorry if this is a silly question, but I've scoured the man
> page and searched deja.com in vain. Any help is greatly appreciated.

The reason is a netscape bug. When you start netscape with an
explicit geometry, it still claims that the position is
'program specified' instead of 'user specified'. Usually fvwm
is set up to ignore all program specified positions. You can
disable this feature with the 'UsePPosistion' style:

  Style Navigator* UsePPosition

you may have to remove the 'NoPPosition' style from your config
file too. However, I'd recommend that you use one of the
StartsOn... styles instead.
