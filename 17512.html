That's because the examples use 'fvwm -f "<ModuleName> blah"'.
fvwm2.0.43 uses the '-cmd' option instead. see the ChangeLog:

       22. Changed behavior of '-f' to just specify config file (now use
           -cmd to perform an actual command, like -cmd 'FvwmM4 .fvwm2rc')

So fvwm tried to interpret the binary as a script.
