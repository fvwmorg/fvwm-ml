> I've been using it on irix machines for the last few months, and it
> works fine. I suspect the problem might be the compiler or bit-depth
> you're using.
>
> For my setup, I compiled it with gcc-2.8.0 with the regular "-32"-bit
> ABI (not "-n32" or "-64". It works on the 32-bit and 64-bit machines I
> use (irix 6.2, 6.3, and 6.4). If you used the 64-bit model, or used cc
> instead of gcc, that may be the source of the problem.
>
> It may also be an X server problem. Xsgi runs by default in pseudocolor
> mode. Its real display depth is 24-bit or 32-bit, but it acts like an
> 8-bit server so that it will be compatible with older programs. To get
> truecolor, programs have to ask for it specifically, in some sort of
> non-standard way. You can fix this by modifying the startup script for
> the X server. I forget what you need to modify right now, but I can
> mail you instructions later this week when I go back to work.
>
> Oh, yeah. It might be the distribution you're using. The plain 2.0.46
> from the Fvwm page worked for me; I haven't tried fvwm-smr-2.0.46.4 yet.
