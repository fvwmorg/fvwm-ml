Well, how about GNU autoconf solution ... ?

Here is a sample, patch to 1.24c. What it did is:

* create configure.in, sysdep.h.in (by autoheader) and configure (by
autoconf)
* modify source code to use sysdep.h information.

So, making fvwm procedure is

        a) do 'sh ./configure' to create sysdep.h
        b) xmkmf
        c) make Makefiles
        d) make

b) to d) is the same of current procedure.

(Well, using 'sysdep.h' is a little bit ugly, merging to configure.h
is better...)
