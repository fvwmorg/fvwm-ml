Is there any way in fvwm to change the geometry of a window based on its
class, using the Style mechanism?

We have an application that opens up lots of windows all the time, all
of which have a +0+0 geometry specified by the program itself. This
gets very annoying as every single window from the smallest dialog, file
requesters, alerts etc. all popping up on the same spot.

If I could specify where on the screen various important windows with known
classes would appear this would solve a large part of the problem.
