Quoting Bill McGee (bill.mcgee_at_amd.com):
> Has anyone implemented a solution to this yet? I am interested
> in some form of equivalent to ctwm's f$warptoscreen(#) function.

I wrote a little program to do this (if I understand your problem
correctly).

Blurb from the web page:

If you have a multihead display, with multiple display/screens instead of
one giant virtual screen, this program is useful. To use it, you should
bind it to some key sequence in your window manager. The program allows you
to warp the pointer to a different screen, so you can move the mouse easily
between montiors with just a single keystroke (keeping the same X and Y
positions).

http://www.ishiboo.com/~nirva/Projects/xwarppointer/

Fvwmrc bdining example and usage is in the README included in the pkg.

Hope it helps!
