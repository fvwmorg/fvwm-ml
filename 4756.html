FVWM version 2.5.1 compiled on Mar 26 2002 at 20:20:35
with support for: ReadLine, Stroke, XPM, GNOME WM hints, Shape, SM,
Multibyte, Xinerama

I've noticed that on applications that set their layer themselfs that
FvwmIconMan will use the layer they have. Example I've used is rox
filter, http://rox.sf.net. It can bring up desktop icons ala gnome/kde
and sets it's layer as 0, FvwmIconMan also trys to use this layer which
generally hides it behind all the other windows on the screen which
default to 4
