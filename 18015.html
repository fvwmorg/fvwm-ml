> Oh good. But I've decided to try and make everyone happy on this
> one and actually add the mini icon support. Since it's #ifdef'ed in
> there, I guess it won't hurt anything. As I've said in the past I
> don't like all these #ifdef's in there for various features, but
> I've decided that it's the only way to get closer to pleasing
> everyone.

An idea struck me that might be possible; but I'm not sure how much
rewrite it would take (most likely a _lot_). Since #ifdef's are the
Achilles Heel of programming (and nobody really likes them), would it
be possible to rewrite some or all of FVWM in an "Apache" manner,
i.e., using "modules" to implement different features? I don't recall
too many #ifdefs; Apache just "knows" there's a module built in. In
fact, it would allow one to link in, say mini-icons, without
rebuilding the entire FVWM.

Obviously, something of this magnitude would have to wait for the
Great Rewrite (2.1), if it were even considered. I've never dived too
deep into the Apache nor the FVWM2 code, but would be willing.
Unfortunately, my knowledge of X is very, very limited. :(

Comments? Flames? Scapegoats?
