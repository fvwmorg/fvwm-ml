On Wed, Sep 25, 2002 at 08:58:40PM -0500, Victor Eijkhout wrote:
> > > > >> How do I cycle through windows so that Next takes the one with next
> >> > >> highest x coordinate?
> >> > >
> > > > >Try the "Direction" command.
>
> Could someone look at the source for obvious bugs in this? I set up
> Shift-{Left,Right} to do Direction {West,East} Focus, and if there is
> a window that is (totally, or almost so) contained within the outline
> of another (but not with coincident centres) it gets focus only one
> of the two directions. For instance, if it's offset to the right, it
> only gets focus if I'm arrowing left.

Sure, that's exactly how it's described in the man page. If the
center of the new window is left of the center of the old window,
only "W" works. In 2.5.4, there is a new direction "C"enter that
just works on the closest window in any direction.
