That's just a debug message. You problem has nothing to do with
the window being shaped.

gkrellm has the nasty habit of storing its window position between
to starts. Once the window appears, gkrellm moves it to some
off-page location. That usually happens if you kill the window
while you are viewing a dfifferent page. For example, if the
window is at +0+0 on the top left page, and you are viewing page
(1 0), then kill gkrellm, it's real position on the display is
+<-display width>+0. When you start the application next time, it
moves itself to the left of the actual screen.

You should disable this "feature" in gkrellm (right click on the
gkrellm title, the disable "Remember screen location ...".
