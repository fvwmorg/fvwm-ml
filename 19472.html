Orlando Andico <orly_at_gibson.eee.upd.edu.ph> writes:

> Hello:
>
> I'm using FvwmWinList as a (sort of) icon manager. I use NoIcon so that
> them icky pixmaps don't appear, they just become a list in FvwmWinList.
> Now I set the minimum width == the maximum width so it only resizes
> vertically. Now the bad habit is that, the FvwmWinList has a tendency to
> wander downward and to the right after it resizes. I'm using 2.0.43; is
> this documented bug behavior?
>
> (I realize this is dumb); I'm actually using a modified FvwmWinList, but
> the modification is trivial -- I added some XTextWidth(...) code in
> make_name() so that the window names get clipped at character boundaries
> instead of getting clipped at the edge of the module window (which looks
> messy). Maybe this causes the module drifting?
>

I know little about the internals of FvwmWinList, but I wrote FvwmIconMan
which is very similar and is sometimes plagued with the same problem.

Quick question: do you have your FvwmWinList set up so that it grows to the
left or up? That is, is the corner which is fixed (assuming the wandering
wasn't happening) any other that the top left corner? If so, then this
operation is impossible to do in X without occasional wandering. The best you
can do is minimize the risk of its occurance, which I've been trying to do for
two years in FvwmIconMan.

It's very unlikely that your change affected anything.

bpp display or are running ANY webbrowser other than netscape. I don't know
