julian_at_echoes-online.de writes:
> Hiya
>
> have a problem at my friend's computer. I gave him all my
> fvwm-config, he unpacked it into .fvwm (just as needed) and the
> user-rights are okay, too. If he loads fvwm (either via xterm
> within x or via the xdm-config-file) it gets loaded but doesn't load
> the configs. There are some errors but the are more the "cannot load
> pixmap..." - type than the critical one. Does anybody have any idea
> what the problem could be? Or can anybody give me a hint where to
> look?

I don't think I understand the question.

Are you trying to say fvwm isn't reading $HOME/.fvwm/.fvwm2rc?

If you use a command line like this to start fvwm:

fvwm -f /tmp/config

Then thats the only config file fvwm will look for during startup.

Recent beta releases prefer to read $HOME/.fvwm/config.

Find out the release of fvwm, find out the command line
used to start fvwm then if you still have questions,
ask again.
