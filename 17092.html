On Mon, 10 Jun 1996, Yves Arrouye wrote:

[...]
> I have a bug report for fvwm2 too, which I send you here so that you can
> forward it upstream. The bug is with windows refresh. To exhibit it, launch
> xv, select Grab, choose the Hide XV Windows checkbox and grab something:
> the window still displays whatever was there before the grab; if you move
> the window, its contents are correctly updated.
> This bug does not appear with the mwm window manager, so I think it is
> related to fvwm2.

Could people please check whether their 'xv' shows this bug? I can
reproduce it with xv 3.10 and 3.10a.

I'm increasingly of the opinon that it isn't entirely fvwm's fault. The
bug is reproducable with fvwm1.24, fvwm2 (2.0.42) and twm, but not
with olwm. I haven't got access to mwm to verify Yves's report.

The common point seems to be that fvwm and twm all use ActivePlacement
policies, where as olvm (and presumably mwm) use something like
RandomPlacement. It isn't as simple as this, though: if I put fvwm
into RandomPlacement mode, I still can't get correct behaviour.

Any ideas, chaps (and chapesses!) ?
