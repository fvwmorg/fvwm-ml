Greg Fall <gmf_at_dweezil.dyn.ml.org> writes:

> On 15 Oct 1997, Manfred Bartz wrote:
>
> > The centering can be achieved with m4 pre-processing (m4_eval), but I
> > think you still need to know the width of the button bar.
>
> I don't see how M4 can do any better for you than just hard-coding
> *FvwmButtonsGeometry in there. Sure, you can make some kind of
> BUTTONBARWIDTH macro, but that really makes it no easier.

Well, if you have two systems with different size screens, or you use
different screen resolutions for different users, m4 pre-processing is
a real help. As long as the size of the object is fixed, you can then
calculate the necessary offsets for centering based on the ``HEIGHT''
and ``WIDTH'' values of the screen.

This may not be applicable for you.
