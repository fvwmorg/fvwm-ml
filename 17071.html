> I, for one (maybe the only one) would most appreciate a way to specify an
> ordering in either the FvwmWinList module or in the internal WindowList.
> I use many windows in a typical session, spanning several pages, and it
> would be very nice for me to be able to specify
>
> 1) (*minimal wish*) alphabetic ordering, or
>
> 2) a specified ordering, perhaps of the form found in the Style
> directives, where the matches to the first window name would occor first,
> then to the second name form, and so on, defaulting to alphabetic at the
> end.
>
> This would be of great assistance to me, as I often have 20 windows open,
> and reading down the list to find a name buried somewhere in there is
> unnecessarily time-consuming (but !!!MUCH BETTER!!! than the HP-VUE
> default of having to remember which page the window is on, and whether it
> is iconized or buried under another window, or destroyed, or ....)

Have you tried the IconManager?

I use it all the time to manage a large number of
windows on several desktops; it's far better than using icons
or relying on the FvwmWinList.

You can have several window lists (xterm windows in one list,
emacs/vi windows in another list, etc.),
specify alphabetical ordering, iconify/deiconify/warp to windows
by using the mouse on the window lists, and more.

Check out the FvwmIconMan man page for details.
