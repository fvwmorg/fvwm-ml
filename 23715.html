On Wed, 24 Feb 1999, Jim Nicholson wrote:
: This has been bugging me since 2.1.5.
:
: Make install puts the modules into
: ${PREFIX}/libexec/fvwm/${FVWMVERSION}/ now, rather than just plain old
: ${PREFIX}/lib/fvwm. I can understand doing this to support having
: multiple versions of fvwm installed, but frankly, there doesn't appear
: to be any other specific provision for multiple versions in the
: install process (i.e., making ${PREFIX}/bin/fvwm2 a symlink to
: fvwm2.2, etc.) So the net effect of the current install is that it
: breaks .fvwm2rc each time fvwm2 is updated, since ModulePath needs to
: be altered.

I believe that the correct solutions it to install fvwm as;
${PREFIX}/bin/fvwm${FVWMVERSION} and linking it to ${PREFIX}/bin/fvwm2.

This has buged me and I and others have asked for this a couple months ago.

Randall
