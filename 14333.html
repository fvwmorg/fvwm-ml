>>>>> "dwood" == R H Wood <dwood_at_spdmail.spd.dsccc.com> writes:

dwood> humm...

dwood> didn't the original idea include the suggestion that these
dwood> multiple-segment .fvwmrc "files" be put in a "hidden"
dwood> subdirectory? like ~/.fvwm/ perhaps?

I believe that one of the original suggestions had that, yes.
Regardless, I still don't think that the default should be changed to
this.

All (most?) other window managers have one rc file, and I don't want
to break that tradition just yet. Fvwm is already powerful enough to
provide this multiple rc file functionalilty if you want, but I don't
think that it should be the default.

Now, if someone out there wanted to provide a sample system.fvmwrc
file to be included that helps facilitate this by reading multiple
local files, along w/ a README about it, I'd be happy to include it in
the sample.fvwmrc directory so everyone that installs it knows that
option is available. Perhaps I'll take a stab at making one this
weekend, but since I don't split up my rc file, perhaps I'm not the
best one to provide this.

....
dwood> Perhaps fvwm could benefit down the road with having a "place
dwood> of its own" to put who-knows-what-have-you stuff that might be
dwood> needed for future enhancements/features.

dwood> just another man's $.02 worth (maybe less) ;-)
