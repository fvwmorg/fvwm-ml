I've a developer accessing a SPARC20 workstation running Solaris2.6 using exceed
(ver. 6.1).
User is logged on to the workstation running fvwm and then accesses workstation
using eXceed running in
single window mode. He is running window manager now on PC and workstation.
He claims once in a while when he kills his eXceed session by clicking on "x"
causes his workstation window manager to die and
everything he types in his UNIX windows shows up on Console screen. Once in a
long while it does this on UNIX workstation (Ultra1)
without eXceed also.
He can open new xterms from the toolbar on UNIX but can't type into any of the
windows.
This goes away when he logs out and logs back in.

Is this a bug? How do I fix this so next time window manager doesn't die on
UNIX when he kills eXceed session on PC.
How does one restart xserver and get control of the windows without logging out?
