On 01 Jan 2004 08:43:29 +0100, Joolz wrote:
>
> My FVWM setup gives me 4 screens (I hope this is the correct term).

X uses the term "screen" to mean something different.

Looking into your screenshot from another message, you use one virtual
desk with 4 virtual pages each (or more precisely, 2x2 pages).

> I arrange tasks so they are logically grouped on the different screens,
> and I can switch between the screens with key combinations.
>
> One problem: I'm on screen 1, typing in mutt, goto screen 2 to in
> order to lookup something in Google, start typing and noting happens
> (well maybe I'm destroying stuff on screen 1 :-)

If you switch from the one desk with 4 pages layout to the 4 desks with
one page layout (arranged as 2x2 in FvwmPager, with or without desk
labels), you get the focused window remembered for each desk separately.

> IIRC there used to be a tool called FvwmTrackFocus.pl that would
> enable you to automatically set focus to a window on the screen you're
> on (I never got it working though). Google doesn't tell me a lot about
> this, does it still exist? (or better yet) has it been integrated?

If you speak about the module that Google returns on "FvwmTrackFocus"
query, then you need 2.5.x in order to run it. It is not integrated.

Just tried it with 4 desks, 2x2 pages each, seems to work, although I
don't quite like the fact that the window under pointer may end up non
focused, instead the previously focused window on that page is forced.
Probably good if you use keyboard to switch desks/pages, but not mouse.

Please always specify your fvwm version in every message. Do not imply
that others remember/didn't delete your other messages. Another reason is
that you may run several fvwm versions at home and at work, and you may
just switch fvwm versions between your messages to the list, noone knows.
