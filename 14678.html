I've just done sample implementation, to apply configure's feature to
fvwm-2.0.38 in last weekend. This implementation does replace 'Do it
yourself OS supporting stuff' section of configure.h. It does NOT do
configure where to install, where is the XPM library, which feature to
select, and so on. This MUST BE done by Imakefile or by
user. configure must support OS specific library existence check ONLY.

Well, I found that libs/strncmp.c is the source of mystrncasecmp(). So
I think it should be renamed to libs/mystrncasecmp.c. This renaming
causes Imakefile (SRCS= section) also.
