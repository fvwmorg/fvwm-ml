  To: Raymond Toy <toy_at_rtp.ericsson.se>
  cc: fvwm_at_fvwm.org
  Subject: Re: FVWM: FVWM 2.2.4 vs Rational Rose (Solaris)

  Date: Wed, 06 Sep 2000 09:20:08 -0400
  From: Dan Espen <dane_at_mk.telcordia.com>
  
  Raymond Toy <toy_at_rtp.ericsson.se> writes:
>
> I've been using FVWM for ages and it's always worked for me just
> fine. However, I've started using Rational Rose for Solaris. FVWM
> seems to get completely confused.
  
  Rational Rose uses some software from Mainsoft that lets MS Windows
  applications be recompiled to run on UNIX.
  
  There seem to be 2 Mainsoft Knowledgebase pages that relate to UNIX
  window managers:
  
  http://www.mainsoft.com/nbase/mainwin/kb/kbmw0027.htm
  http://www.mainsoft.com/nbase/mainwin/kb/kbmw0034.htm
  
  The second page looks like it might help, try
  exporting MWWM=allwm before you run your application.
  
  You might also try MWWM=MWM. With this it would make sense to tell
  fvwm to respect MWM decor hints.
  
  I think the fvwm-workers list would be interested in any results you
  can tell us about.
  
  --
  Dan Espen


Ray then reported that MWWM=allwm seemed to work best.

I think you are correct in your guess that you may see more of these
applications. Make sure you let Rational know how you feel about
their application defaults.
