On 23 Oct 2002 13:40:36 -0700, Denis Perelyubskiy wrote:
>
> Is it possible (perhaps in 2.5.x series, if not in 2.4.x) to
> NOT pass the placement click/keypress to the applications?
>
> I did find the Click... options, but those seem to apply to
> mouse only, and only to 'Focus'. Or are they supposed to
> work for placement clicks/keypresses as well?

I think the intention is to never pass the placement clicks/keypresses to
an application regardless of any style options.

> I am using the smartplacement style, and some windows go
> away right after I place them, since I think placement
> click/keypress (i hit enter sometimes) is passed to the app.

Denis, this is the second time you say you use SmartPlacement (BTW, it is
better to avoid this obsolete name and use the MinOverlapPlacement name)
when you actually use ManualPlacement. Please be more attentive.

I can't reproduce the problem with this config:

  Style ManuallyPlacedXTerm ManualPlacement
  Exec xterm -name ManuallyPlacedXTerm

Pressing Enter to place this xterm is not passed to it.
Probably you use something like "Exec xterm -e date" and "date" or other
similar command exits immediatelly, so xterm exits too.
