>
> In practice using -DMODALITY_IS_EVIL makes live a little dangerous.
> Today, the 'default' fvwm2 seems to make no difference between
> FULL_APPLICATION_MODAL and APPLICATION_MODAL dialogs which is, I
> would say, to restrictive.
>
> Has anyone some remarks, hints or (best of all) a solution.
>
There was a flame-fest a while back about modality. There was almost consensus
that system-modality is evil. Personally speaking, I don't want Fvwm
complicated with modality handling code. If an application crashes because
it is relying on the window manager to enforce modality then the appplication
is defective IMHO. There may be scope after 2.1 and the great style flag
rewrite to build a module that manages modality (maybe a SmartFocus module
that would allow sloppy-focus to be controlled such that moving the mouse
to an application window would move the focus to the relevant dialog box etc.)

Cheers,
Tim Phipps.
