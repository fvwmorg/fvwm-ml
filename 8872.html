On Sat, 2003-05-10 at 21:46, wang yin wrote:
> Hi,
>
> Why FVWM grab the server whenever I move or resize windows?
> All the other applications stopped updating the display that
> time.
>
> I think that's not good to grab the server. How can FVWM not to
> do it?
>
> Thanks.

FVWM needs to grab the X server to draw the XOR'd frame type it normally
uses. You can change the move/resize style by putting this in your
fvwm2rc file:

Style "*" ResizeOpaque
OpaqueMoveSize -1
