Steve Swamp wrote:
> That's right. I'm running both 8 and 24, but the 8 is the default. And I can't
> run 24 as the default, because it breaks older applications (believe me, we've
> tried). So that's why I need fvwm to start with 24. An Xresources option would be
> nice, or just some code to autoselect.
It would have to be a command line option. Fvwm doesn't do Xresources
and auto selecting a non-defualt would probably cause problems for most
users. I counted a total of 33 CopyFromParent's in fvwm source and some
of these are probably what would need changing. It would be good too if
the modules could be told of the visual to use, that would mean a change
to the module protocol and recoding all the modules.
