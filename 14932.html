>>>>> Chad Coulliette asks why FvmwForm doesn't work for him...

Do you also use FvwmM4? If so, run FvwmM4 with the -debug option.
The problem is that FvwmForm hasn't been updated to do use some of the
newer features for module communications in pre2.0. It still relies
on looking for its configuration in the file specified on its command
line. If you use FvwmM4, that file will be a temp file, but unless
FvwmM4 is run with the -debug switch, it removes the file so FvwmForm
can't get its config.
