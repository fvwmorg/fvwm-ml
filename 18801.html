Hi,

I'd like to make an addition to the wish list: An enhanced
StaysOnTop/StaysPut style option that had more than two independent
window stacks; at least three, but preferably N. This could be used to
implement the StaysOnBottom command, for one thing, and probably could
lead to further extensions for grouping windows and applying actions to
particular stacks. You could have syntax like

Stacks 3
Style "*" Stack 2
Style "FvwmButtons" Stack 3
Style "xosview" Stack 3
Style "xbiff" Stack 3
Style "Reminder" Stack 1
Style "Notification" Stack 1

"Stacks" would default to 2, and the parser would translate
StaysOnTop/StaysPut to "Stack 1"/"Stack 2". Commands could be added to
raise/lower windows from one stack to another. Etc.

Is anybody doing this or has plans to do this? I'll look and see if I
can do it, but I'm no Fvwm internals expert. Does anyone have any
additional ideas or comments? Should I wait for the Semi-mythical Great
Style Flag Rewrite?

--Xris
