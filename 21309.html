Hi everybody!

I have a problem with FVWM2 that I don't have with other window
managers such as FVWM, Afterstep, twm, Exceed and HP X-Terminals.

I run a commercial application (HP Design and Drafting System for
Cabling, DDS-C) remotely and display it on my Linux-box. At some
point in the application a have a list from which I can choose
different components. If I click with the right mouse button a
component name, a window opens which displays some technical info
about the component and the graphical symbols associated with the
component. The problem is that with FVMW2 this windows is really huge
(1596x6384 pixels) when it is supposed to be something like 533x399
pixels (533*3=1599, 399*16=6384).

I have really no clue what the problem is ;< I have read the FAQ, TODO,
BUGS, searched the www-site and also looked at the man-page but found
no indication of a similar behaviour. This is why I'm writing to this
mailing list.

I can guarantee the reproduction of this bug. Unfortunately not too
many people have access to DDS-C so I don't know how anybody else
could repeat it. But I'll be happy to dig up more information if
someone tells me how.

Here's what I came up with:
- Fvwm Version 2.0.45 compiled on May 8 1997 (from debian-release 1.3
  precompiled binary)
- personal configurationfiles make no difference
- some library info:
  /lib/ld-linux.so.1, libXpm.so.4, libXext.so.6, libX11.so.6, libc.so.5
- I'm running Linux 2.0.30 with AcceleratedX v.3.1

I guess that's all... I can always go back to FVWM1 but I already have
started to enjoy the more advanced features of FVWM2 :)

Thanks in advance,

Jaakko Puurunen
