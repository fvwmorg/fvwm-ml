On Fri, 31 May 1996, Erik Rissanen wrote:

> The problem is that fvwm2 crashes when I switch to an another
> virtual console with ctrl+alt+Fn (Fn is any F-key). This seems to
> happen at random times, but very often. To produce it I just
> do something in X and switch from and back until it happens.
> It always happens when I switch from X to text, never the opposite.
>
> Here is the output:
>
> Fatal server error:
> Caught signal 11. Server aborting
>
> ...
>
> It is the SVGA server.
> I run it with a Cirrus Logic 5434 card with 2 Mb RAM in 16bpp.

Seems to be a problem of the X server, anyway. I have experienced the same
problem some time ago when I was using a Cirrus Logic card (with either
fvwm-1.24 or olvwm, Linux 1.2.8). As soon as I changed the card, the
problem dissapeared. You may want to try changing/disabling some of the
server options, see the README.cirrus for more details.
