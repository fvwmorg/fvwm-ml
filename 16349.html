I compiled 2.0.42 this weekend. It almost compiled without a hitch. In my
previous compilation (.39, I think), I had installed libfvwm.a (I've no idea
why). When linkink fvwm, -L/usr/X386/lib appeas before -L../libs so the
system library was linked in instead of the distribution library. It should
be fixed to avoid these types of problems. Though I'm not really sure why the
library was installed in the first place.
