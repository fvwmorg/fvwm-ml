> > That means, if I select window B after having selected window A before, I'd
> > like the toggling function to pop me back into window A, and from there to

> > could be achieved by the new `Prev'-Function in Fvwm 2, but it doesn't work
> > for me.
>
> The line
>
> Key F22 A N Prev [CurrentPage] FlipFocus
>
> /should/ do what you want, but there have been many reports from
> people for whom it did not work. Originally, I experienced similar
> problems, but I could finally work around them by playing with the

Thanks - I tried it and it works - sometimes. If it works, it works
fine: I can toggle ad infinitum between the two selected windows with
a single keystroke.
But mostly, nothing happens at all - the text cursor in Xterm windows
is blanked out for a moment (as if the window would loose focus, but
no other window gets the focus instead. What's worse, toggling to
another window never works if the window which had the focus previously
is iconified.
Also, it isn't clear to me why I have to use `Prev [*] FlipFocus' in-
stead of just `FlipFocus'?

As Fvwm clearly has some idea of which window had the focus previously,
I think there should be a possibility to choose that window again.
Is the current behavior of FlipFocus a bit buggy or would some kind
of extension be necessary (e.g. a builtin function PreviousWindowID
returning the id of the former selected window?
