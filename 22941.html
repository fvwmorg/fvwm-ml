"Nagle, Adrian" <anagle_at_ball.com> wrote:
> This may not be the appropriate place to ask but I'll make it quick.
>
> I'm trying to figure out what color depth I'm running FVWM2 in. I've
> started looks at pictures with xv and they do not get all of the picture's
> color palette. My graphics card can handle many colors (at least 32bit).
> Is there a way to check the setting after loading FVWM2? I was looking in
> the XF86Config file and it looks like it is set to 32bit, but that doesn't
> make sense with the strange colors I'm experiencing. Title bar gradient
> fill wouldn't do that would it?

        Titlebar gradients shouldn't cause this problem. First, as somebody
else mentioned, try running xdpyinfo to find out what visuals your X server
supports. Then try experimenting with xv's -visual command line option.

        xv tries to determine the best visual to use, but doesn't always
make the best choice with some of the newer server/card combinations.
I'd start by trying

                xv -visual default

(that's what I use with AcceleratedX). I'll bet that works, but if it
doesn't give you the right results, try -visual TrueColor or -visual
DirectColor (assuming xdpyinfo says the server supports those).
