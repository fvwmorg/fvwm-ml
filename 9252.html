On Fri, Jun 13, 2003 at 03:55:52PM -0400, Gregory L. Shank wrote:
>
> I'm running fvwm-2.4.16 on Solaris 8 and am running into the following
> problem:
>
> There is one app that opens some new windows outside the physical screen
> area. I can't get to the title bar to move it into view. I've turned off
> multi screen mode and edge scroll mode and have set my desktop to 1x1
> like so:
>
> EdgeScroll 0 0
> DeskTopSize 1x1
>
> Can I force apps to always open windows on the physical desktop and if
> so, how?

If you know the name, resource or class of the faulty
application's windows, you can force them on a particular page
with

  Style application_name StartsOnPage 0 0, SkipMapping

(you may or may not want the SkipMapping style).
