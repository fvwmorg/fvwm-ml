În data de 13-09-2002, la 15h 08'49", Andy Arbon (andy) scria cãtre fvwm, despre "FVWM: Xterm titles":
>
> Hello,
>
> I am looking for a way to make it easier to tell the many xterms I
> always end up running apart in the window lists. What I am currently
> thinking of doing is changing the title of each xterm to something like
> "xterm (<username>) - <last issued command>"

The way I do it is that I open each xterm for a purpose. If it is a
local terminal then I call it xterm_at_localhost and the rest I call it
with the names of the computers I log into. One xterm is for the e-mail.
I have mutt rename it with this lines in .muttrc:

 set xterm_set_titles=yes
 set xterm_title="Mail"

So I have no need to make changes in the xterm name. If you have few
xterms oppened for the same computer it can be tricky, I never had more
than 2-3, except for the localhost. In that case I have special defined
few xterms (different colors and names) and it never happend to have
more than those oppened.

You can try the function which change the title for xterm, but what if
the last command on all xterm is 'ls -ltr'?
