>>>>> "Ed" == Ed Carraway <ecarrawa_at_hilliary.nosc.mil> writes:

Ed> Is their any way (apart from writing your own drag&drop functions) to
Ed> use Motif drag&drop to drag objects to another fvwm viewport (by
Ed> dragging to the edge of the screen to activate EdgeScroll)?

I don't think so. I believe Motif D&D is grabbing the server during
these drags and stopping all wm functions.

Setting the following X resource:

          *DragInitiatorProtocolStyle: drag_dynamic

seems to help somewhat - the program I checked didn't grab the server
so some fvwm actions happened normally (like windows appearing) but it
still wouldn't flip pages.

Unfortunately I don't have any more time to play with it right now.
I'll try to look into it.
