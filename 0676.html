> I have. I have some minor problem with FvwmPager. As I've written earlier I
> have four desks and move through them using keyboard (Alt+right and
> Alt+left keys). I have several windows stiky (like irc, icq, console).
>
> I have several windows on one desk, some of them are stiky. I move one
> non-stiky window to the top of all of them, it is shown correctly on the
> pager. Then I move to another desk and return back. On the screen I see the
> non-stiky window still on the top of all. On the pager I see the stiky
> windows on top of non-stiky ones... Here are the pictures describing what I
> see. Console and irc are stiky, mail is not.
>
> On the screen:
>
> +----------------------------------------------+
> |+-Console----------+ |
> || +-mail---------+ |
> || | | |
> |+-------| | |
> | | | |
> | | | |
> | | | |
> | +-irc--| | |
> | | | | |
> | | +--------------+ |
> | | | |
> | | | |
> | +----------+ |
> +----------------------------------------------+
>
> On FvwmPager:
>
> +----------------------------------------------+
> |+-Console----------+ |
> || |---+ |
> || | | |
> |+------------------+ | |
> | | | |
> | | | |
> | | | |
> | +-irc------+ | |
> | | | | |
> | | |----------+ |
> | | | |
> | | | |
> | +----------+ |
> +----------------------------------------------+
>
> Is it a bug or an intended feature and if it's intended how can I make
> pager to show me the true positions of the windows?

It is not a feature, pager should show the real world, so it is a bug.
Please always tell us the fvwm version when reporting problems (you should
test over the latest stable or unstable one). When reporting bugs, it is
very helpful if you show how it is possible to reproduce them. Thank you.
