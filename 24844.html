>>
>> > > AddToFunc FocusAndWarp "I" Focus

>> >
>> > That works if you use MouseFocus, but not ClickToFocus.
>>
>> Of course it works with ClickToFocus. This has nothing to do with
>> focus policy.
>
>Really? The documentation I found for "WarpToWindow" just says that it
>will put the cursor in the window. It doesn't say that any focus policy
>is implied by this, so I assumed that your current focus policy would
>still apply. Putting the cursor in the window, while using ClickToFocus
>policy, should NOT cause focus to change, IMO.

WarpToWindow does what the docs say. The effect of WarpToWindow does
depend on the focus policy. However, that's not the point. The example
above works regardless of Focus policy because there's a line to give
focus and a line to warp. One will always be extraneous, but it's
guaranteed to work independent of focus policy. I think you must have
not seen the first line.
