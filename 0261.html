Why are you setting display to hostname:0.0 instead of just :0.0?

The former will always uses sockets to communicate between the clients
and the X server, while the latter will use shared memory which is
_much_ faster.

It sounds to me like you don't have permissions to access the server
using the hostname:0.0 method. I've seen this before (just the other
day someone was using the elm backgrounder and it kept removing these
permissions!)

If you must stay with hostname:0.0, try using:

  xhost +hostname

before you invoke the window manager.
