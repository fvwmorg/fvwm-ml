Elizabeth Schwartz wrote:

> There's a feature of X11R6 called "xinerama" which supports using multiple
> monitors as one virtual screen. I have not used this yet but I am looking
> forwards to it!
>
> Since this is an X11 feature it should be below the level of the window
> manager and
> transparent to it.
>
> We run fvwm on Solaris 2.6 and xinerama is supported with Solaris 2.8 (and
> some versions of 2.7)

    Sorry, what do you mean -- "FVWM works with Xinerama", or "FVWM supports
Xinerama"? By the latter I mean -- FVWM knows that it e.g. shouldn't center
popups on the global screen, making them split between physical screens.
We are using 2x2 screens Xinerama setup under XFree 4.0.2, and FVWM's
Xinerama-ignorant behaviour is a bit irritating.

> For our dual-head systems on 2.6, we use "x-big-x" from x-software in
> germany (www.x-software.com ) This works quite well, windows dragging
> across monitors and all.
