> Is it the panel itself that keeps forcing the layer like that? Is there
> any way to customize the layer used from the panel's perspective?

Panel uses gnome hints to convince fvwm to change its layer dynamically.
So this should be configured in the panel.

Thanks to Dominik, now it is possible to do: Style panel GNOMEIgnoreHints
This sounds a bit silly (disable gnome hints from gnome panel), but
this is a good answer to the panel developers forcing this behaviour. :)

There are other solutions with layers depending on whether you want panel
to be always on the same layer as other windows or always above or below.
