fvwm-2.5.4

I've been trying to rearrange my fvwm configuration so that .fvwm2rc
essentially has a PipeRead command that looks for and reads any files it
finds with a certain extension. Carrying this over to menu
configuration, I tried having one file per application and having in
each file both Style information and an entry to add it to the menu.
What happens is PipeRead reads in, for example, mail.men first then
term.men --- with the first file having information for mutt (which is
running inside an xterm) and the second for xterm. The problem is that
since term.men gets read secondly, its ``Style XTerm Icon terminal.png''
overrides the ``Style mutt Icon mutt.png'' command, even though the
window containing mutt has ``mutt'' as its title --- not ``xterm.'' If
I rename the files so that term.men gets read first (by prepending a
zero to the file name), then mutt has the icon it's supposed to have.
Is there a way to ensure that a window will have the ``right'' icon,
regardless of the order in which fvwm reads its Style command?

Thanks for any help.
