On 07 Mar 2002 16:36:35 -0800, Elliot Sowadsky wrote:
>
> I'm trying to blow away internet explorer spam windows. Presently i'm using
> add_event and WindowID $0 ("*Iexplorer*") to trigger a close. Problem is
> that the window has already started going thru drawing, so it takes several
> seconds to die. Is there an earlier event to trigger on or another method?

No, I believe, there is no other way to do this except for asking explorer
itself not to open spam windows. I don't know what do you mean by "spam
windows", because I don't run explorer, but if you mean javascript
windows, recent Mozilla and some other unix browsers are configurable
about blocking popup windows and other javascript abuse.

You may also (on add_window event) do something quick before Close, like
Iconify or MoveToDesk 0 10.
