If you really want to have fun with windows, look into DPS. If you have
Solaris or SGI, or better yet, NeXT, you are set.

Since there is a full PostScript interpreter on board the display server,
anything that you can do in PS you can put on screen, even to animating
the window decorations (Shaped!). Mostly useless when it comes down to
it.

If you just want the funky decorations, look into ImLib, the image
processing engine that E uses. It is easy enough to create shaped windows
in X. It would be easier to add shaped borders to Fvwm than it would be
to fold the two together.

I am sure that if you get a framework going, folks will want to jump in
and help coding. One big problem you are going to have: Fvwm has only 1
border width, not seperate widths for top/bottom/left/right... Also, the
positions of reparented client windows are hard coded in many places in
the code.

Also, you are going to have to be able to specify each border seperately,
and turn off the top border. Corner handles will have to have independent
sizes.

This is not an easy task, and may have to wait for 3.x, but probably well
worth doing.

ericb
