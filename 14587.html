Thanks god, there is anybody alive!
So my congratulation to make a new
distribution release, i've been waiting a long time for.

So here my ideas for the Fvwm 3.0 Release:

        - seperate the Window Border and Menue Outline,
          to get the bowman direction back and to make
          it much easier to make a new outline (Lost95)

        - add drag&drop to root window and modules
          ,especialy FvwmButton, with Motif Compatibility
          I hope that the lesstif people will make a library,
          so that having a drop target in fvwm will not
          require Motif.
          An other way to integrate drag&drop is to use the
          rdd library.
          
        - DrawOut Module, an evolutionary step of the
          FvwmButton Module, to save screen space
        
        - RollOut Module, nearly the same as DrawOut
        
        - FvwmCosy Module for having an Cosy like
          enhanced interface:
          in the middle you select with the button the
          desired Desk or can toggle to see the Pager of the
          Desk

        - FvwmClippbord Module, (part of FvwmCosy) where to
          drag images out of the web browser, switch the
          desk to another application and take it from the
          clippboard

        - Make a Fvwm-library, that every application using
          this will easy be able to use Fvwm features.
        
        - Support Session Management through Fvwm library


I think these steps will make the Fvwm more than only a
Window Manager. So let's talk about the next release, now

                Gruss
                        
                        Kay
