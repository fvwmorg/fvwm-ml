> I don't suppose there's an undocumented StaysOnBottom window style

No, sorry. StayOnTop uses a flag and there aren't any spares

>
> So, if there isn't such a beast, *could* there be, oh ye mighty
> developers out there? :)

One day. After the great style flag rewrite. There is a hack that will
keep just one window at the bottom. FvwmAuto can call any arbitrary
function and you can use that to keep lowering a particular window e.g.

Module FvwmAuto AutoLower
AddToFunc AutoLower "I" Next [ ConsoleWindow ] Lower

If you start your eterm with the name ConsoleWindow it will get lowered
whenever you change focus.
