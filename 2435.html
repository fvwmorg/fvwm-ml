On Thu, 3 May 2001 11:40:54 +0200, Dominik Vogt <fvwm_at_fvwm.org> wrote:
> On Tue, May 01, 2001 at 11:32:24PM +0100, Adye, TJ (Tim) wrote:
> > 3) In my FvwmButtons (edited a little to conform to the new syntax) I
have
> > buttons like
> >
> > *BarButtons(Title XTerm, Icon rterm.xpm, Action `Exec "XTerm" exec
> > xterm`)
> >
> > which I think should mean that the button remains shown as depressed
until
> > the XTerm window appears (this is what used to happen). Now the button
is
> > only shown depressed as long as mouse button is down. Is this a new bug,
or
> > am I using the wrong syntax?
>
> FvwmButtons does not have the functionality you describe. You
> are probably confusing this with the 'Swallow' option.

If that's true, it must have been removed recently. It used to do this in
2.3.6, and it's still in the (2.3.31) FvwmButtons man page:-

               Any fvwm command is recognized by FvwmButtons.
               See fvwm2(1) for more infomation. The Exec com-
               mand has a small extension when used in Actions,
               its syntax is:

                 Exec ["hangon"] command

               When FvwmButtons finds such an Exec command, the
               button remains pushed in until a window whose name
               or class matches the quoted portion of the command
               is encountered. This is intended to provide visual
               feedback to the user that the action he has
               requested will be performed. If the quoted portion
               contains no characters, then the button will pop
               out immediately. Note that users can continue
               pressing the button, and re-executing the command,
               even when it looks "pressed in."

I rather liked this functionality, but I agree that it's not a critical
feature. If it was decided to remove it (when did that happen - I didn't
notice it in the news?), then I guess the documentation should be updated.

In any case, this is just a minor quibble (sorry if I'm too much of a
pedant). My main problems are the spurious window raising and full-screen
refresh that I mentioned in my original message.

Thanks,
Tim.
