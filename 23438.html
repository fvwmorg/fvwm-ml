Sorry for overloading this post with 3 issues.
I am using FVWM 2.1.7 on a Sun running Solaris 2.5.1.

1. When I run the FvwmTaskBar module, *FvwmTaskBarDeskOnly doesn't
   work (although it does in Fvwm95 if I run that). Is there a bug in
   the code, or is it in my configuration somewhere?

2. I have been trying to activate a menu from a FvwmButtons button in
   such a way as to look a little like a "panel". That means
   positioning the menu specifically relative to the button - without
   having to know exactly where the button is or the exact size of the
   menu. I want it to sit just above the button in the same way a
   panel does.
   I spent some time playing with the position options to the Menu
   command, but could not get it to work. Does anyone have any advice?

3. For a future version of FVWM2, would it be possible to add extra
   switches to the command line to enable parameters to be passed to the
   .fvwmrc file to be used in a similar fashion to parameters in FVWM
   functions (eg. fvwm2 -p1 abc -p2 def ... -p9 "Last Parameter").
   This could allow, for example, the default decor to be selected from
   the command line.
