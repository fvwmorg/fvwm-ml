>>>>> "Brady" == Brady Montz <bradym_at_cs.arizona.edu> writes:

Brady> Stephen Wray <Steve.Wray_at_vuw.ac.nz> writes:
>> I used to rely on FvwmWinList to warp the pointer to totally obscured
>> windows. As of 2.0.43 this is no longer the case. WHATS MORE it doesn't
>> seem to have an option to do this!!! What happened? How can I fix this
>> *abberant* behavior???

Brady> You could try FvwmIconMan. It's like FvwmWinList, but I think
Brady> it's better, but I'm predjudiced since I wrote it.

:)

Brady> Or, if you want to stay with FvwmWinList, which I admit is
Brady> better for transient use, I bet that the problem comes from the
Brady> change in the fvwm Focus function which makes it no longer warp
Brady> the cursor.

Correct. For those who don't recall, this was a much requested
feature (for Focus not to warp the pointer, that is).

Brady> If you use the FvwmWinListAction option to bind WarpToWindow to
Brady> a click event, that'll probably work.

Actually, better would be to use something like the following
function:

  AddToFunc FocusAndWarp "I" Focus
  + "I" WarpToWindow 2p 2p

and bind that to the click event (because WarpToWindow won't actually
change the focus on it's own in click to focus mode).
