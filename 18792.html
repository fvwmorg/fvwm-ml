|> Hello,

|> I'm using fvwm 2.0.43, compiled on an HPUX 9.05 station using the
|> libraries for X11R6.1. My Xpm lib is 3.4g. For some reason, no matter
|> what i do, I can't get iconized windows to use pixmaps instead of
|> bitmaps. I tried changing the paths, adding a slash at the end of the
|> name, etc. It works with bitmaps but I can't get pixmaps to work and I
|> don't know why. I'm guessing it could be a colormap problem but I'm not
|> sure how to verify this. Xv brings up the pixmaps with no problem. I'm
|> really at my wits' ends.

Which path are you using? In order to use full colour pixmaps your pixmap
path must be set (the pixmap icons don't go in the IconPath). I mention
this because when I first saw it in FVWM it seemed slightly illogical.
There IS a reason of course but putting pixmaps in the IconPath won't
work.

So the PixmapPath should look like this:

PixmapPath /usr/X11/include/X11/pixmaps/:/usr/local/icons

Or wherever you keep your pixmaps.
