Chris> }Todd> the button would stay 'pressed in' as the man page suggests...if

Chris> }I've been thinking about that, and I think a different way should
Chris> }probably be used. I've been toying with a generic parameter, either
Chris> }to be given to the ButtonStyle command or as a separate one, to make
Chris> }individual buttons act with a toggle behavior. That way they could
Chris> }work with any bindings (not just Maximize).

Chris> now what would be really nice, is along with that, you could specify
Chris> an alternate pixmap to use instead of having it "pressed in". that
Chris> way, you could, for instance, after maximizing, replace the maximize
Chris> button with a restore button, a la win95 or somesuch.

2.0.42 basically has that capability now. If using Pixmaps as the
button style, if you specify 2 pixmaps, the second is the "pressed in"
one, but you only see it while it's actually pressed in. Using
FullPixmap removes the border, so this toggle ability that I was
referring to would be all that would be needed to get what you want.

BTW - keep in mind that mentioning 'win95' along with any idea is
enough to turn me off of it... :)
