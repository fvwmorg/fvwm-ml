> Does
>

No, not with the port. The Makefile has a variable that appears to be
hardcoded, to yes (WANT_GNOME=yes), and then checks to see if another
is defined (HAVE_GNOME). It isn't real clear what make parameter will
tell the port to use the --disable-gnome config directive.

Another suggestion in this thread was "make WANT_GNOME=no", but
WANT_GNOME is just redefined back to yes and gnome-libs is still
included.

Maybe I'll have to hack the Makefile myself. Perhaps the port
maintainer is listening?

I'll probably send directly to the maintainer if I can't get around it
correctly.
