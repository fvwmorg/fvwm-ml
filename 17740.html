>>>>> "AJS" == A J Smith <A.J.Smith_at_nortel.co.uk> writes:

AJS> I'm not sure this is the right place for this, but I need some help
AJS> installing any version (1.24+) of fvwm.
....
AJS> The most used window manager here is tvtwm, but I didn't like it and
AJS> wanted to go back to fvwm. We can install basically what we like, but I
AJS> don't have root. If I know exactly what has to be done then I can
AJS> probably get someone who is root to do it for me.

You don't need to really install fvwm to get it run. It'll run out of
your local directories just fine. You just have to set up the paths
properly in your .fvwmrc/.fvwm2rc file (eg ModulePath, PixmapPath).

Now, if you do want to really install it, then the directions are all
pretty much in the tar file with the source. See the new official web
page to find where to get the latest version:

                     http://www.hpc.uh.edu/fvwm/

AJS> I'm not that experienced at installing unix software, so I don't know if I
AJS> have everything I need. I just thought, "if tvtwm can work, surely fvwm
AJS> will?"

It sure should. I know people on this list have compiled and run it
on the HPs too, so as long as you have Imake installed, you shouldn't
have any problems.

AJS> Anybody willing to help?

I'm sure someone will who has done this on HPs before, if you have any
more difficulties.

AJS> I still have all my old configurations for .fvwmrc at university, so I
AJS> don't need to write those from scratch, though if that version of fvwm is
AJS> not compatible with whatever version I install there could be problems.

The latest beta version is 2.0.43, and the next version will be the
first official 2.xx version (2.1.0). There have been no other
official releases since 1.24r. The 2.xx rc file syntax is not
compatable with the 1.24r style, but a script is included in the tar
file that can give you a hand in converting it, plus a list of changes
in the FAQ can help you convert by hand as well. It isn't (or rather
shouldn't be) that bad.
