On Mon, May 05, 2003 at 01:13:54PM +0400, Michael Lashkevich wrote:
> Dear Developers,
>
> I installed "fvwm-2.4.15" as an "rpm" package under Linux RedHat 7.2.
> There is a bug: from time to time, while launching the Nedit editor FVWM
> produces the window without the title bar, and with WindowListSkip and
> Sticky properties turned on.
>
> There were no such problem with previous versions of FVWM, so I attribute
> this bug to FVWM rather than to Nedit.

Might "from time to time" have something to do with the window
title? If you edit a file, it's name shows up in nedit's title
bar. If that title matches a style definition, that style is
applied to the window title. E.g. if you have

  Style FvwmButtons NoTitle, Sticky, WindowListSkip

and run

  $ nedit FvwmButtons

this would cause the described effect.
