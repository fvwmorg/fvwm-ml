Wrong. All your assumptions are right :-) However, if the application
really wants to, it can find out about the frame and the border
(FvwmButtons is one of these naughty applications). To answer your
initial question: I don't know. Aterm seems to be quite buggy since
its behaviour seems to be random.

> I get similar behaviour - transparency works fine with borders, does not
> work without borders (either the whole window or parts where there is some
> text are no longer transparent).
>
> anybody knows a good implementation of transparent xterm-clone? (I intend
> to use it on solaris).

Try rxvt. It's implementation of transparency was copied by most
(probably all) terminal emulators that have it. I spent some time
discussing/debugging that feature with rxvt's authors and it should
work well now (but make sure you have the latest version!).
