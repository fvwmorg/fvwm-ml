I've also seen this with fvwm-1.24r. It seems related to stopping the fvwm
process. For instance, in my .xsession, I am starting fvwm in the background
and the last line in the .xsession file is a foreground xterm. When I logout
of the xterm, I expect the fvwm process to die, too. Sometimes it does;
sometimes it doesn't.

When fvwm does not exit, it goes into some runaway mode. The next time I
login, my .xsession starts a new fvwm process. Now I have one runaway (from
previous login) and this new fvwm process. If I don't manually kill the old
process, it just sits there consuming CPU cycles (and upsets a lot of my
colleagues!)

Has anyone found a solution to this or know the cause?
