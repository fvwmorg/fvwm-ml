Alas! Shao Zhang spake thus:
> Hi,
>
> Sorry if this bug is already mentioned. I have the following key binding
> in my fvwm config:
>
> Key quoteleft A M Module FvwmPager -transient 0 3 Warp
>
> It worked fine in fvwm 2.4.*. In fvwm 2.5.2, only the first desktop is
> shown correctly, for desktop 1, 2 and 3, no virtual screens are shown,
> only a blank rectangle with the desktop title.
>
> If my description is not clear enough, I can supply a screen shot.

Yes, this is a known problem. FvwmPager is pretty much totally borked in
2.5.x ;)

But, if you don't use a transient pager, and you don't make it sticky,
switching desktops and switching back will make it render the other 3
desktops properly ;)

For now, do what I did: make your desktop bigger (more pages), and then
just use the one. If I remember correctly, it's fixed in CVS, though.
