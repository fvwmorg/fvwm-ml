Previously from Neil Zanella
>I cannot see that Piperead would be any more useful than Exec.
>Could anyone give me an example of a useful Piperead command
>that cannot be implemented as an Exec command?

FVWM listens for commands on the stdout of the PipeRead command. So
you can use powerful scripting tools like perl or python to do
intelligent things with PipeRead. With Exec, you're fundamentally
limited to what FVWM knows how to do (FVWM doesn't internally support
iterative constructs, conditional clauses, etc.). You'd never be able
to implement something like fvwm-menu-directory without PipeRead.
