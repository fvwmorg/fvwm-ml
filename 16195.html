    Pieter> 1. In Fvwm.tmpl the variable CFLAGS is set to the value of
    Pieter> FvwmCompilerFlags if the latter has been defined. This
    Pieter> overwrites the old value which included $(ALLDEFINES) and
    Pieter> thus removes any -I and -D switches to the compiler.
    Pieter> Using CCOPTIONS instead of CFLAGS seems more appropriate
    Pieter> to me.

This is only my opinion, but I think CDEBUGFLAGS would be even better,
since if it is defined it would override any optimization flags. It
is handy for setting debugging flags, and even gets passed down the
environment (however, since Fvwm.tmpl is included, it might not be
neccessary) if you specify the following in the Imakefile (BTW Chuck
knows about this already):

#define PassCDebugFlags 'CDEBUGFLAGS=$(CDEBUGFLAGS)'

About the strerror, I couldn't even find str_errlist on SunOS 4.1.1
(!) so I just replaced it with "sorry!" ;) so it would compile.
