It seems the compiler options specified are generated by a standard Solaris
2.5.1
imake configuration file, which assumes Sun CC to be present. either use Sun
CC,
or adjust your /usr/openwin/lib/config/site.def to include HasGcc2
(see /usr/openwin/lib/config/sun.cf to see why).
If both of these aren't valid options, use your own xmkmf pointing to your
own
configuration files. If you don't know how to do this, there should be a
script to
generate makefiles for fvwm which will help you out in this case, but
doesn't with
other xmkmf requiering applications.
