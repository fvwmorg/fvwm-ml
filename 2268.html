Hello Tim,

Tim Hanson wrote:
>
> How do I get FvwmButtons to Swallow xosview? I'm trying to get it to
> use two frames. I don't want to use Frame0 because that's where
> FvwmPager sits.

here's a part of my ~/.fvwm2rc. Hope that helps:

--- snip ---
*FvwmButtonsFont 6x13
*FvwmButtonsFore Black
*FvwmButtonsBack grey67
*FvwmButtonsGeometry +0+0
*FvwmButtonsRows 1
*FvwmButtons (2x1 Frame 0 Swallow(UseOld) "FvwmPager" "Module FvwmPager
0 0")
#*FvwmButtons - - Swallow "FvwmXeyes" Exec xeyes -name "FvwmXeyes"
-geometry +0+0 -bg grey67 &
*FvwmButtons - - Swallow "FvwmXclock" Exec xclock -name "FvwmXclock"
-geometry +0+0 -padding 1 -bg grey67 &
*FvwmButtons (2x1 Frame 0 Swallow "FvwmXosview" 'Exec xosview -geometry
+0+0 -name FvwmXosview -load +battery -net -page +disk -int -xrm
"xosview*captions:Flase xosview*background:grey67" &' )
*FvwmButtons Fvwm2 window3d.xpm Function barthiswmpopupfunc
*FvwmButtons Tools box_full_3d.xpm Function bartoolspopupfunc
*FvwmButtons Shells Shell.xpm Function barshellspopupfunc
*FvwmButtons (7x1 Frame 0 Swallow "goodstuffterm" "Exec xterm -geometry
+0+0 -name goodstuffterm -title goodstuffterm -sb -bg grey67 &" )
--- snap ---

It includes (form left to right): pager, clock, xosview, menues, xterm
