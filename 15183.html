This is a long shot, but I'm hoping there are some people out there
using fvwm under IRIX 5.3 who might know about this.

The X client programs xwsh, wsh, and winterm all seem to set the property:

        WM_ICON_NAME(COMPOUND_TEXT)

which contains the icon title in hex. xterm, on the other hand, sets

        WM_ICON_NAME(STRING)

to the icon title in ASCII. 4Dwm (SGI IRIX window manager) correctly
displays the icon titles for xwsh, etc. But fvwm's icon titles for
these clients are blank! Is there an easy way to get fvwm to convert
the WM_ICON_NAME(COMPOUND_TEXT) to the icon title and use that instead
of looking for WM_ICON_NAME(COMPOUND_TEXT) (which I'm guessing it's
doing now)?

Any help would be appreciated!
