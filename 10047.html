On Sun, 21 Sep 2003, Mikhael Goikhman wrote:

> On 21 Sep 2003 15:43:41 -0400, Ian Hutchinson wrote:
> >
> > netscape -iconic
> >
> > works fine under metacity (for example) but not fvwm (using 4.79 version
> > of netscape for speed reasons under Red Hat Linux 8.0).
> >
> > Under fvwm [fvwm 2.5.7 compiled on May 30 2003 at 20:29:15
> > with support for: ReadLine, Stroke, XPM, PNG, Shape, XShm, SM, Bidi text,
> > Xinerama, XRender, XFT, NLS] the netscape always pops open its first
> > window. It looks as if it tries to start iconic but then for some reason
> > deiconifies straight away.
> >
> > I read that the fvwm Inconic setting does not work for netscape, but that
> > netscape -iconic does. I have tried all combinations and none works.
> >
> > Anyone have a fix?
>
> With fvwm-2.5.8 (from cvs) and Netscape 3, "netscape -iconic" works
> without problems. It is started iconic.
>
> If you start fvwm without any configuration, like "fvwm -f no-config",
> does "netscape -iconic" work? f it does not work, it is possible that you
> run some netscape wrapper script that discards its arguments.
>
> Alternativelly, it is possible to start any netscape iconic by adding:
>
> Style Navigator StartIconic
>
> But it causes all new subwindows to be started iconified too.
>
