I am using "Fvwm Version 2.0 pl 37 compiled on Oct 12 1995 at
13:20:26", and have found a bug that did not exist in "Fvwm Version
1.24r".

I am running Solaris 2.3 on a sparc 5. I start "fvwm" with "openwin".

The problem I found, or should I say bug, is that on certain windows I
have "NoIconTitle" set and when I start Fred, (that's my name for
fvwm), any window I have this set for starts in the upper left corner,
not in their designated location. That is until I click on the icon,
then it moves to the correct spot. Also the identify module will not
identify any of these icons with no titles.

With Fred version 1.24r these icons where placed in their correct spots
but the identify module still did not work.

If anyone can help me out I would be greatly appreciative.
