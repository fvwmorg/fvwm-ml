Dan Espen wrote:
>
> "Andrew Morton" <morton_at_nortelnetworks.com> writes:
> >
> > I could have been misleading people. I normally run fvwm2-2.2-1 and I
> > had that installed when I decided to have a quick hack at it. I did a
> > 'make' and a 'make install', but it's possible I picked up an old
> > module??
> >
> > The only module I use is FvwmPager, and that's from $HOME/.fvwm2rc. I
> > just 'exec fvwm2' from my .xinitrc.

You must have picked up something strange because version 2.2 doesn't
have this error message in it. Version 2.3.0 was prone to pausing,
version 2.3.1 onwards is less likely to pause (you have to do `fvwm -cmd
FvwmButtons' to get a pause).


>
> Even when FvwmPager is started from the .fvwm2rc, it now has to be
> started this way to avoid the delay:
>
> AddToFunc InitFunction "I" Module FvwmPager ...
>
> not
>
> Module FvwmPager
>
Not any more. Version 2.3.1 onwards only waits for modules started from
the command line and ignores modules started from inside rc files. If
this isn't the case let me know because I think I fixed it.
