Chuck,

Since you said that, in the future, there will be a higher number of
possible styles, I'd like to make a suggestions. How about user-defined
styles?

That is, right now, the only way to group styles is for "*" or common Class
or name strings. It would be nice if I could group them differently. For
example:

AddToStyle Info NoHandles
+ NoTitle
+ Sticky
+ WindowListSkip
+ BorderWidth 0

I have this combination of styles on for several applications and it would
be cleaner to just say something like:

Style "Perfmeter" Info
