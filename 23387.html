Hello all!

        I would like to suggest a new IconBox feature: AutoArrange.

This would work like in e.g. Window Maker. When a window is deiconified or
destroyed the iconfied windows would rearrange and move to fill the box
from where it "starts". This way icons in the IconBox would not end up
scattered all over after a while.

I know that this is a feature a lot of people miss in fvwm*, even me myself
know a lot of people who have whined about it. It shouldn't be to hard to
implement either, IMHO, just scanning through the list of existing windows
each time a window is deiconified or destroyed and then rearrange the ones
that are still iconified.

Me myself don't feel competent enough to make a flawless patch and then keep
it up-to-date with the current /fast/ development, so I'm hoping one of the
developers read this and finds my suggestion reasonable. It would make fvwm2
/even/ more complete.

Thanks for listening. Your making a great job. Keep it up. /Linus Nilsson
