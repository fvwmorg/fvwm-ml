dear folks,

i would like to report an apparent bug in fvwm2 that extends up to
2.0.45. the problem occurrs with the program Maelstrom running on an X
server managed by fvwm2 with libc-5.4.x. this bug manifests itself in
Maelstrom having a window if indefinately large size that extends to the
right and below. in 2.0.43 the window would also extend to cover any
virtual desktops that were to the right and below the place from which
the program was initially run. in 2.0.45, this window doesn't extend to
any other desktops than the one it is run on. Maelstrom only occurs
within the upper left 640x480 space of this window (it normally only
runs in a 640x480 window).

Maelstrom works correctly on a server managed by fvwm1 with libc-5.4.x
and on a server managed by fvwm2 with libc-5.2.18.

if it is pertinent, this behavior has persisted through almost all of
the recent 2.0.x kernel series, running XFree86 3.2 and Accelerated-X,
and on at least three different machines (a 486, pentium, and a PPro).
in my experience, this behavior has not been exhibited with any other
programs.

happy bug hunting.
