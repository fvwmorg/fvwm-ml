  na> Okay, what's the correct approach? There seam to be many files
  na> that need the permissions changed (directories and executables,
  na> etc.). Is there a file list or a better installation procedure?

First, note that this mailing list is not really the appropriate place
for these problems: this is for discussion of the fvwm window manager,
which is generic and runs on any version of UNIX and X and has nothing
to do with how to start the XFree86 server or the permissions of its
files. You should probably continue this on the comp.os.linux.x or
comp.windows.x.i386unix newsgroups.

Second, all the permissions should be correct as-is on an installation.
Can you provide an example of permissions that you felt were incorrect,
and why you thought so (what didn't work that made you suspect a problem
with permissions)? Whenever I install things they always have the right
permissions out-of-the-box.
