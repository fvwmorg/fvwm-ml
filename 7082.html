> I've just compiled fvwm-2.5.4 on sparc/solaris, partly because I'm

There is probably a bug in 2.5.x for rendering images on a
machine with bigendian byte ordering (and no XRender ?).

Can you see the following line just after png configure check:

when you run ./configure?

Also, as you run Solaris I imagine that at the end of configure
you can see:

  With Xrender image support? no: ...

yes?
