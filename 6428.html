On Sun, Oct 06, 2002 at 05:01:19PM +0000, Mikhael Goikhman wrote:
> On 06 Oct 2002 12:47:42 -0400, Victor Eijkhout wrote:
> >
> > If I invoke FvwmIdent from the console, it waits for me to click in a
> > window (fvwm 2.4.11); if I invoke it as
> >
> > AddToMenu fvwm
> > + "Source fvwmrc" Read /Users/victorle/.fvwm2rc
> > + "Restart fvwm" Restart
> > + "Console" Module FvwmConsole
> > + "Ident" Module FvwmIdent
> >
> > then it give the information immediately. Is there a way to invoke
> > from a menu and still have it wait for me to click somewhere?
>
> You should be invoking the "fvwm" menu from decorations and not
> as the root menu, otherwise you would not have see behaviour.
>
> To trick the window context (i.e. to remove it), you may use:
>
> None ("abracadabra") Module FvwmIdent
>
> I plan to add a more relable NoWindow to remove the window context.

Good idea. However, it's already possible to delete the window
context with the WindowId command:

  WindowId Root Module FvwmIdent
