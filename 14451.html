AGH! Please dont mess with this one - I use it all of the time!
Basically it is a style that allows you selectively turn on/off
switching desktops when a window is being mapped.

For instance, I have two desktops: Main and Mail. When I start fvwm,
an xtermand some other stuff it started on Main. On Mail, I start
emacs and exmh. I have the SkipMapping style set for emacs and exmh
so when they start, fvwm doesnt jump to the Mail desktop and then back
to the Main desktop. It makes them start up "in the background". I
thought the jumping back and forth when windows are being mapped is
real ugly.

Here is the style line for my exmh and emacs:

Style "?macs" Icon gnu-animal.xpm, StartsOnDesk 1, NoButton 2,
  SkipMapping
Style "?xmh" Icon mail1.xpm, StartsOnDesk 1, NoButton 2, SkipMapping

Here is an excerpt from two messages from Rob Nation (2nd and 3rd of
August) on this issue:

>You need to set the SHOW_ON_MAP flag which exists in fvwm. There seems to
>be no way to set this in current versions of fvwm, other than by changing the
>source code.

and...

>>Do you think it would be possible to add this to a future release of
>>fvwm so you can do this in the .fvwmrc rather than modifying the
>>source code? It would be nice to have the option to SHOW_MAP or
>>SKIP_MAP on a selective basis!

>Actually, its obscure, but it was already there. There were
>styles ShowMapping and SkipMapping that did the job.
