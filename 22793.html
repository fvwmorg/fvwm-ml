> Wouldn't it be better if IconPath would be renamed to "BitmapPath"
> in view of the fact that there is a "PixmapPath"? After all Fvwm2
> is already incompatible with Fvwm1 so this extra naming fix should
> not be a big change in view of what has already been changed.

Only if it were currently implemented as an alias, and then perhaps have
"IconPath" deprecated post-2.2. Many of us are using IconPath currently,
and we shouldn't implement a change that breaks an existing config file,
except on an early development track (earlier than the late-beta track Fvwm2
is currently on).
