On Wed, Sep 01, 1999 at 09:18:20AM -0600, Mark Crocker wrote:
> Actually, this used to be the way it worked. I have three Icon
> managers defined in my .fvwm2rc and have had it this way for a long
> time over several different versions of fvwm2. With the newer
> versions, the icon manager will display a blank placeholder if there
> are no windows that correspond to that manager. Older versions simply
> didn't appear in that case.
>
 

Your correct it did use to work this way. I questioned Brady Montz (the
FvwmIconMan mainter at the time) about it and he said that it was a very
unreliable hack to map and unmap the managers, so he removed it. I
still have the old code and if time permits I'll try to diff it out and
see if I can make sense of it now. But my feeling is if Brady said it
was a bad hack then it probablly was. My advice to you as his was to me
is get used to it, it won't take long. And now two years later I just
ignore the empty managers floating around.
