Viorel ANGHEL wrote:
>
> Sometimes, my FvwmButtons crahs on startup (init or restart) with errors
> like:
>
> X Error of failed request: BadWindow (invalid Window parameter)
> Major opcode of failed request: 18 (X_ChangeProperty)
> Resource id in failed request: 0x3400002
> Serial number of failed request: 41
> Current serial number in output stream: 46
> X Error of failed request: BadDrawable (invalid Pixmap or Window parameter)
> Major opcode of failed request: 62 (X_CopyArea)
> Resource id in failed request: 0x1000020
> Serial number of failed request: 119
> Current serial number in output stream: 124
>
> What's that?
>

I used to get similar error messages when starting up FvwmButtons,
except with me it happened every time. I finally traced my problem to
the version of libXpm that I was using, it was libXpm-4.7. When I
replaced this version of libXpm with the earlier 4.6 version,
FvwmButtons worked fine, and I never had any mor eproblems with it.

You can, of course, check the version of libXpm that your FvwmButtons is
using by "ldd FvwmButtons"

Hope this helps.
