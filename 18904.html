On Fri, 17 Jan 1997, lar3ry gensch wrote:

> Well, I had been using a hacked fvwm-2.0.43 / fvwm95-2.0.41a (basically
> just adding the stuff that gives it the Win95 look and feel). After
> hearing about the BorderStyle and related patches (my sources were too
> hacked up to patch correctly, so I waited PATIENTLY for 2.0.44), I've
> decided to put together a Win95 appearance for fvwm.
>
> So far... It's PRETTY CLOSE... but still no joy.

Looks good.

The nice thing is that you can make it look BETTER than Win95 by putting
stuff like gradients and stuff in. On my (outdated) BorderStyle patch
page, I had a Win95 similar window that had a gradient on it (slight, but
there :-).

> Style "*" UseDecor win95, ForeColor Black
>
> The win95-*.xpm files are pix of the Win95 buttons from the fvwm95
> distribution, and grey75.xpm is a 1-pixel XPM file that contains a
> single grey75 pixel (BorderStyle could possibly use a COLOR command!).
> All these .xpm files are attached to this message via MIME.
>
>
> Also, the fvwm2 man page could do some explaining about 'ActiveUp' and
> 'ActiveDown' and how they apply to the TitleStyle and other functions.

It's explained in ButtonStyle, but I guess there should be references to
it (this applies to every command where it says state can be one of ...,
a notable exception is BorderStyle which only has two state as explained
in Fvwm.tmpl):
              state refers to which button state should be set.
              Button states are defined as follows: "ActiveUp"
              and "ActiveDown" refer to the unpressed and pressed
              states for buttons on active windows; while the
              "Inactive" state denotes buttons on inactive win-
              dows.

> We could then make a pop-up menu, or FvwmButtons display that will allow
> the user to select the desired window display style... this would make
> a powerful enhancement to the system.fvwm2rc file.
>
> Anybody game for such a project?

Sounds good. You might consider incorporating ChangeDecor+Recapture too
for dynamic style changing. On my page with those three different styled
windows at the time I was able to dynamically change the styles and decors
individually on each window with the ChangeDecor+Recapture.
