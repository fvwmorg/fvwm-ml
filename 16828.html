It says all :-)

I've already posted a patches to this list twice (for version 1.?? and
version 2.0.??, sorry I forgot), which make fvwm to configure the OS
specific section (ex. use sysconf or not) with GNU autoconf.

But, IMHO (comes from the experiences of making patches),

a) Most of all applications for X use imake and that's maybe the way
   of X. There is no reason that fvwm denies this manner.

b) Switching to GNU autoconf is not suitable. Though GNU autoconf is
   great, imake knows ALL about development environment for X
   application and GNU autoconf (maybe) don't, sometimes mistake
   (That's the reason that patch for 1.xx was not included to fvwm).

c) Using imake and configure (yes, my patches do so) is tired :-)
   (We should run both 'configure' and 'xmkmf' before 'make')

We should carefully consider the usage of GNU configure to fvwm.
