Hi again,

I've investigated my problem a little closer, and have discovered that
the only difference - as far as the environment is concerned - between
starting an application from the CLI and from the window manager, is
that the windowmanager can't supply a TTY. Starting ImageMagick
without a tty (ie: program < /dev/null > /dev/null ) results in silent
failure.

So - is there a way to have fvwm2.041 supply a tty to a program that
it executes? (pty's would be just as good I expect)
