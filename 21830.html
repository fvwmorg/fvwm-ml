John Culleton <john_at_ccpl.carr.lib.md.us> writes:
> The normal cursor on my system is an "x" outside a window and an arrow
> inside a window. However when I perform a function frequently the cursor
> changes to a circle. I have to click on any empty space to get the "X"
> cursor back. The system then beeps.

Fvwm changes the cursor into a circle when you select an operation
that requires a window, but you are not in a window.

For example if you select "iconify" from an fvwm menu, the cursor
changes to a cirle until you click on a window.

> If I forget to perform this little routine then the screen saver won't
> work.

Because fvwm is suspending everything else until you click on something.

> Any hints on how to get rid of this annoying little circle
> automatically? Assistance appreciated.

Perhaps the source code could be changed to make the little circle
time out...
