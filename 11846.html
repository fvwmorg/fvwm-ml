Hello Manjo and fvwm-Users,

Avter fighting some days with fvwm 2.5.8 I have compiled it without
any Errors for "Debian GNU/Linux 3.0 WOODY".

I have changed the configuration to:

cparams = --without-gnome --disable-xft --disable-xfttest \
--with-rplay-library=yes --with-imagepath=/usr/share/pixmaps,/usr/share/icons

and goten this

michelle_at_michelle1:~ > fvwm -V
fvwm 2.5.8 compiled on Mar 25 2004 at 15:43:22
with support for: ReadLine, RPlay, Stroke, XPM, PNG, Shape, XShm, SM, Bidi text, Xinerama, XRender, XFT, NLS

redistribute copies of fvwm under the terms of the GNU General Public License.
For more information about these matters, see the file named COPYING.


OK, I have installed it on my Workstation and curently (!!!) it is
working fine. With one exeption:

It requires libfreetype6 2.1.7-2 from UNSTABLE which is not acceptable,
because it conflicts with the openoffice.org1.1 backport from

deb ftp://ftp.freenet.de/pub/debian-openoffice/ woody main contrib

but I am working on this.


For now I need some testers !!!
