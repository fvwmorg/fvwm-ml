hi folks,

here's what i'd be super happy to have happen. i'm using fvwm 2.0.45.

i'm trying to avoid tkgoodstuff because, to me, it seems like a huge
memory hog compared with FvwmButtons and also uses up a ton of cpu
cycles.

anyway, i use xbiff to indicate new mail (any better ones out
there?). i have FvwmButtons swallow xbiff. i want to be able to
click on this as well and have it start vm in XEmacs
(using gnudoit -q '(vm)'. i can run the gnudoit part as a script,
that's not the problem. the problem is convincing FvwmButtons that
this is a button.

i've tried this:

*FvwmButtons(Swallow "xbiff" Exec '~/bin/vm.sh')

but Fvwm is not happy with it at all.

any solutions?
