Hello !

Next and Prev are working fine but in my case I want to go to window on
page to which I am switching to (Next [CurrentPage !iconic] focus) .
But if there is no window meeting the condition then while switching back
to previous screen page I get to other window then the last acive.
Maybe there should be some empty position always on top of list meaning no
active window (root active?).

While I am changing active window from WindowList or using Next or Prev
commands positions on list are unchanged but while doing that by clicking
on a window that window gets number 1 on the list.
In my opinion app window should get its number while it is created and
release it when closed. It will be nice to get to the window just by
calling WindowList and then the same number/letter. Now it is not very
user friendly because number by which I activate window may change many
times and I must read windows list to find my app.

Question: is somebody working on a module which switches windows like
W95 does (acivates window while ALT or another modifier is released
so we can press ALT and then Tab many times to get to correct window
(using IconName and title to display info) and then activating it by
releasing ALT; ESC shuld cancel action) ?

Sometimes I will prefer window with no decor at all. I want maximize my
app window to full screen with no decor. But I want it to be done by
setting a flag so I can revert to normal/previous state (with previous
decor). Because decor was build on window creation I don't want to build
it again just switch on/off. Is there simple solution to this ?
