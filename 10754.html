> Yeah, that was the problem. What's the difference between the two? Is
> *FvwmPager setting options within the Pager, and Style telling the WM how
> to handle that app? (Sort of...?)

"*FvwmPager: ..." is a module option that is passed only by the
module binary itself.

"Style FvwmPager ..." defines a style that applies to all
*windows* with the name FvwmPager - which is usually the case for
pager windows.

In this specific case, the distinction is unclear because (a)
ParentalRelativity is a bad style name (it's almost impossible to
guess what it stands for) and (b) you have to set options in both,
the module and fvwm itself.
