On 15 Dec 2003 16:58:10 +0200, stu wrote:
>
> I have three individual konsole sessions running. Whenever I restart
> fvwm, the konsole sessions are re-arranged ie :
>
> Before Restart
> --------------
> DesktopPage 3 : konsole session 1
> DesktopPage 4 : konsole session 2
> DesktopPage 10 : konsole session 3
>
> After Restart
> -------------
> DesktopPage 3 : konsole session 3
> DesktopPage 4 : konsole session 1
> DesktopPage 10 : konsole session 2
>
> The re-arranging does not seem to follow any pattern, except that it
> seems to happen after every restart (using fvwm's 'Restart' command).
>
> Why is this happening?
> How would I prevent this from happening?
>
> I have one desktop with a size of 10x2.
>
> I am running Debian sid, fvwm 2.5.8, XFree86 version 4.3.0 .

It seems like you run fvwm under some session manager, like ksmserver.

So either don't do this, i.e. put "fvwm" as the only program of your
~/.xinitrc, or ask konsole developers to add XSMP support that konsole
seems to lack (otherwise there would be the unique id for each konsole
window and no heuristical guess-by-name is needed).

Another solution is to use unique window name for your konsole
windows, but as I said, this is not needed if konsole supports
XSMP properly.

It is also possible that something just does not work well together
(fvwm, ksmserver, konsole etc). In this case you may debug the problem
if possible and report your results.

Note that if you indeed use a session manager than you should experience
the same problem (I think) when you exit your the session manager and
start it again, then the saved konsole windows may be randomly switched.
But I can only guess here, there is no enough information.
