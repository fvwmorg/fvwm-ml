My Linux system is running a reasonalby recent version of FVWM based on
a reasonably recent version of X-Window. There is the following strange
behaviour.

If I shut down an application and then shut down FVWM before the
application actually exits then the whole system freezes. It happens
frequently with Netscape but yesterday it happened when a scanner
operation was still in the process of exiting. The distinguishing
characteristic seems to be an application that takes its own sweet time
about shutting down. My current workaround is to wait (impatiently)
until all disk activity ceases.

I need to exit FVWM/X Window frequently because my system has only 16MB
of real memory and I do run out on occasion.

Any thoughts? Is there another list I should be bugging about this?

Any replies appreciated.
