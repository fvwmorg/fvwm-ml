I'd never been able to get that FvwmConf module to work satisfactorily,
so I downloaded the latest version posted at the above link and tried it out
again. It still doesn't work. Probably not with Fvwm 2.2 but maybe it's me.
If anyone can tell me how to get it working with the latest "stable" Fvwm 2.2
let me know. Until then I'll keep working on mine.
FvwmConf is a module, mine is a complete set of rc files for fvwm2
configuration.
My package (so far) only has 5 decors. Contributions and suggestions
would be welcome. It is not necessary to restart fvwm2 to change decors.
Each decor can be selected via menu. Individual windows can be assigned
different decors by menu selection also.
Different module configurations are loaded for each decor.
FvwmEvent configs can now be changed "on the fly". (without restarting fvwm2)
You can even specify an FvwmEvent config file to load and use. (e.g. change to
a config with debug commands!)
Individual FvwmEvents "events" can also be changed. (not fully tested yet)
Andrew Laden contributed a Resize Desktop routine to alter number of pages
per desk. (eg 2x2 -> 5x5) Nice!
Backgrounds can be changed by input with the full path to the file.
I'm planning many more features, but it all takes time.
There's probably still a lot of bugs I have to find but I'm learning.
Hope this explains more.
