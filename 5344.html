This depends on your system setup. The error messages are usually in the
same place the X errors are. But some distributions redirect fvwm2 output
to something like ~/.FVWM2_errors.

I start X using this command:

  % xinit [params] 2>&1 | tee $HOME/x.out 2>&1

So I see X errors (and FVWM errors) in both virtual console and ~/x.out.
You may just redirect xinit or startx to a file without "tee" command.
