On Mon, 21 Oct 1996, Rob Whapham wrote:

> don't have to type xinit or openwin from a command prompt). I know this
> doesn't pertain directly to Fvwm, but can anyone give me any help in setting
> up the following OS's to use Fvwm:
>
> Irix 5.3
> HPUX 9.04 & 10.01
> AIX 3.2 & 4.1
> OSF (Digital Unix) 3.2
> DG/UX 5.4 (RISC) & 4.11 (x86)

In case any of you haven't already found the resource for HP-UX, putting

vuesession*wmStartupCommand: /<PATH>/fvwm<whatever>

(I think I got it correctly; check your documentation, and if all else
fails, try it out) into your vue.resources file will fire up fvwm in place
of vuewm. The downside is that this takes longer (!) to start up fvwm
than if you let vuewm come up, kill it, and then start fvwm. This is a
great mystery to me. Another downside is for those of you who have begun
to use vuewm's myriad resource editing gadgets. Starting fvwm this way
keeps you from accessing them. I think there is a vue-lite option, from
which it is also possible to start alternate wm, but I never did that.
As always, YMMV.
