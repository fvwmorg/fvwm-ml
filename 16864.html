As long as the discussion is on startup files, as a sysadmin, I like to
give my users a default environment that they can further customize.

The way this usually works is that people copy the system default file
to their own directory, then edit it like crazy.

Often the problem is that the default changes. E.g. suppose I want to
replace xterm with xtm. I'd like to do this only once inthe system
default file.

After a couple of years of this, we have 113 different user
environments, which stiffles communication between users.

With tcsh, my default user shell, the shell sources /etc/csh.login,
/etc/csh.cshrc ~/.login and ~/.cshrc. So changes that are put in the
first two act as globabl changes, which users can choose to overwrite.

How difficult would it be with fvwm for it to read system.fvwmrc, AND
~/.fvwmrc?
