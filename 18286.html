> SR> This is a good point. The reason why 2) isn't an accurate description
> SR> is because the most common way a for a dialog window to get the focus
> SR> in an explicit focus environment is for the window manager to give it
> SR> the focus automatically (or to tell it to take the focus) when the
> SR> window opens.
>
> Are you trying to say that in ClickToFocus mode a window manager would
> automatically give the focus to any window which opens? You're kidding,
> right? This would drive me completely insane.

Must be a lot insane people out there then, because this is the way
all Mac and Windows applications work (disregarding floating palettes
and menus, of course, the former of which are unfortunately not
supported by any X11 WM that I know of). It is also the way mwm works
in explicit focus mode (which is the default specified in the Motif
style guide).

> I've been staying out of this, but I need to say one thing: this is a
> mailing list for FVWM. I believe that there's an entire newsgroup for
> window management issues, and if this isn't FVWM specific then I'd
> appreciate everyone saving my bandwidth and moving to a more appropriate
> place. (I host this list.)

I disagree with this: the focus policy that should be specified in the
default .fvwm2rc file is an important issue that should be discussed
on this list. fvwm2's violations of the Motif style guide that I have
reported previously also deserve more complete coverage:

1) windows must autoraise when given the focus (Section 5.2.2)
2) enforcing modal dialog (Section 5.2.2)
