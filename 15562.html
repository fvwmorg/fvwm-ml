Solaris 2.5 defines several M_XXX things in <sys/stream.h> which is
included by <netinet/in.h>. Unfortunately, one of these is M_ERROR
which is used by fvwm.

This is only a problem with the FvwmAudio module with builtin rplay
support. The rplay header file includes <netinet/in.h>.

Can M_ be renamed to something else? How about MODULE_? I'd be
willing to write a perl script to do the conversion.
