>>>>> "Dan" == Dan Niles <dan_at_more.net> writes:

Dan> Here is my CurrentScreen patch for fvwm 2.0.43. This patch makes
Dan> CurrentScreen (in the Next/Prev commands) work more intuitive.

Dan> Before this patch, CurrentDesk selects windows on the current
Dan> desk, and CurrentScreen selects window on the current page. To
Dan> select only windows on the current screen (view port), both
Dan> CurrentDesk and CurrentScreen must be specified.

Dan> After the patch:

Dan> CurrentDesk selects windows on the current virtual desk
Dan> CurrentPage selects windows on the current virtual page
Dan> CurrentScreen selects windows on the current screen, or view port.

Actually, I don't think the distinction between Page and Screen is
necessary. I do think that CurrentScreen should be renamed to be
CurrentPage to match the semantics of Page and Desk in fvwm and Screen
in X11 (although I'd probably keep CurrentScreen around as an
undocumented synonym so as to not blow existing users out of the
water), and that it should always refer to just the visible page (and
not the same page on different desks) as that is really what makes the
most sense. If the ability to select things on the same page in
different desks is truly needed, I think that should be called
SamePage.

Chuck
