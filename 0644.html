On Tue, Mar 07, 2000 at 10:34:09PM +0000, Andrey wrote:
> Eterm has a nice -x option, which causes it to have no borders, title or any
> other 'beautification' from the window manager. This option worked OK up until
> 2.3.13 (which is the second to last of the devels I have compiled). I just
> compiled 2.3.16 and the option ceased to work.
>
> Also I tried using "Style <mytitle> NoHandles" shich did not work...
> "Style <mytitle> NoTitle" left the border intact, even though it removed the
> title (as it should)...

Tim Phipps wrote:

> The fact the Eterm could make borders disappear in 2.3.13 is a
> mis-feature of Eterm. It used to work because the window borders were a
> direct ancestor of the Eterm window and it fails now because they are
> now an aunt.
>
> If an X program wants a borderless window it can use the Motif or
> OpenLook window manager hints (which have been working for years).
> Admittedly, it can't switch the borders on and off once it has started.

I finally had the time to check this myself. The latest
cvs sources (fvwm) do grant Eterm's -x option. Furthermore
I checked the latest stable Eterm versions (0.8.9 and 0.8.10).
Eterm *does* use Motif hints, so I think we don't have to do
anything here. The problem probably was temporarily caused by
some of my 'Style' changes. Or are there any reasons why this
still is a problem?
