>
> >>>>> Tim Phipps writes:
>
> J> Also I don't understand why the examples of UseOld in the manual
> J> all combine it with NoKill, when "NoKill" is the default anyway,
> J> isn't it? Even with UseOld, is I restart fvwm2 a couple of
> J> times, it seems to miss xload's, swisswatch regularly :-(
>
> I think Jarl meant NoClose to be used in conjunction with UseOld, in order
> to swallow old apps/unswallow them on exit.
>
> But this only works, when I do a "killmodule FvwmButtons" from FvwmTalk.

This feature (the UseOld/NoKill) option allows a user to start programs
from their .xsession file and then Swallow them into FvwmButtons. I
orignally suggested this idea to Jarl because I occasionally have to use
multiple window managers (HP VUE, CDE, et al.) and I only want to have
to start my clocks and xbiffs once. Using these attributes lets
FvwmButtons grab a window that was running previous to the start of
FvwmButtons and then Release the window when it closes.
