My conclusions was:

1) Convert all our man page into XML "FVWM DocBook" format. So, the
new source doc will be fvwm/fvwm.xml,modules/FvwmButtons/FvwmButtons.xml,
bin/fvwm-root.xml ...etc. Use a new dtd (DocBook + some restriction + some
special tags for fvwm style, fvwm command ...etc.). docbook-xml as is,
is not what we/I want we should customize it.

2) We should distribute our man page (other format is an other story)
so we should convert the xml docs into man pages. There are _no_ tools
that allows to do the conversion from XML-DB to height quality manpage
(as we have now). So, we should do it ourself. I've started to write a
xslt stylesheet that can do this.

3) Other formats: for html docbook-xml + some customization can give
good result. It is not so easy to get good ps/pdf output (via tex or
fo), there are various tools (e.g., jade, passivetex and fop). I am
sure that we can get good info page.

4) All the active fvwm workers should be happy with editing the new .xml
files.

5) All these need a lot of work. I do not have the time now, but if
someone wants to do it, (s)he is welcome. I may help at the beginning.

> > Something like docbook2info would create want you want automatically.
> >

What I've noted about all these tools that convert a format to an other
is that I'am rarely happy with the result. In general you need either
to reparse the result or add some custumization.
