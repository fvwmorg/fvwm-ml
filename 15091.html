> - control over 3d look of borders (like width of shadow)
> - separate control over border width and control handle width

I'm not sure what the 2nd option in this list means, but please
note that the mwm-style has some items (titlebar) with a 1 pixel highlight/
shadow on the top and sides, and a 2 pixel shadow on the bottom.

Also, note that any dynamic configuration (while generally good) will slow
down the redraw operatation just a little.

Now, I've been thinking about the "module to do decorations". You
could have fvwm re-parent the application window (use the 2-level
reparenting that it uses now, but leave out the title-bar, buttons,
side-bars, and corners. This would leave a decor-less frame in which
a module would work. The module would tell fvwm how big to make the
frame, and where to put the window inside the frame, and the module
could create its own windows within the frame for decoration. This would not
be unnecessarily slow, since the module would do its own redraws, and get
expose events for its own windows. Note that the module would also have
to handle receipt of and processing for all the actions that are bound
to window decorations.

This is a very complicated undertaking, of course.
