1) I know this is too late for the official release but I've just finished
   moving over from CTWM to fvwm and the one thing I am missing is the
   f.movepack instruction. What this does is a move but the other windows
   on the screen are treated like the screen edge so moving over them has
   some resistance. It makes it very easy to tidy up the display.

   It could be implemented as a separate function or as part of Move if you
   specify a non-zero BorderResistance value (c.f. EdgeResistance).
   There wasn't an f.resizepack in CTWM but it would have been useful.

   Any thoughts on the feasibility of this?

2) I would quite like a MoveRelative command so I could jump windows around
   the screen in steps. How easy would this be?
   
Cheers,
Tim.
