Agreed. I would vote for more explicit syntax.

I don't actually use the beta version, I use one of the
old 1.23 releases.

But the actual writing of the fvwmrc is not a big deal.
The use of any syntax which is more explicit, if a little
more verbose is to be commended.

Having to try and figure out how fvwm might parse a command
or having to come back 6 months later to change a configuration
and having no idea what the hell it was doing really outweigh the
extra time taken to code the command in the first place.

I would also vote for the more explicit syntax. It is crystal clear
if not as concise as the other version.
