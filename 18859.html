On Wed, 15 Jan 1997 16:49:17 EST, "Charles Hines" writes:
>
>Ok, at long last 2.0.44 is ready and available from the official ftp
>site:

And there was much rejoicing!

> 25. Changed SmartPlacement behavior to be "stubborn" (like 1.xx
> StubbornPlacement option) and not place over icons (if you
> don't want this you can compile with NO_STUBBORN_PLACEMENT).
[...]
> 37. Added (GlobalOpts) global option
> SmartPlacementIsReallySmart (may become parm to
> SmartPlacement, or whole new CleverPlacement style after
> the Great Style Flag Rewrite) which uses clever placement
> code from Anthony Martin

...but unfortunately, SmartPlacement is not "stubborn" if
SmartPlacementIsReallySmart is set. (I use the FvwmPager module, in the
top left corner of the screen, sticky and in its iconified state in
order to make it show the current desktop, so the stubborn behaviour
would be particularly useful for me; right now, pretty the first place a
new window usually decides to nest is under my stays-on-top FvwmPager
icon.) Adding a non-NO_STUBBORN_PLACEMENT behaviour into
CleverPlacement should be an easy change, yes?

Also, another pet bug of mine which doesn't appear to have gone away: I
use FvwmButtons, with this:

*FvwmButtons Restart icon.xpm Restart fvwm2

...but whenever I use the button, a warning message is produced:

[FVWM][Done]: <<ERROR>> Call of 'fvwm2
' failed!!!!

...presumably because FvwmButtons is including the newline from the end
of the configuration line in the command it forwards to fvwm2, and fvwm2
evidently is failing to remove the newline as well--note the newline
before the trailing quote in the error message. (Is this a FvwmButtons
but or a fvwm2 bug?)
