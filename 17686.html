> I just noticed what appears to be a small bug in the menus. When the menu
> title + icon is the longest line in the menu, the title is not displayed
> correctly. The icon covers the first part of it. Can anyone confirm this?

I found another bug in menus. It really isn't serious, but here's the problem
anyway: Open a menu, move cursor over a submenu entry which isn't defined, move
cursor back to parent menu and you see the problem: submenus pop up in wrong
place and can't be seen all the time. If parent menu is itself a submenu, the
problem disappers when you move cursor over its own parent. Otherwise you must
escape from menus or select something.

Another thing is what to do with these menu entries which don't do anything. Should
they be shadowed or should they disappear? Anyway we shouldn't be able to select
something that does nothing. I would like to see something like global option

DisabledShadowing <percentage>

where percentage defines transparency of dummy entries, and if it's zero we don't
even see them. They don't really disappear from internals and if we define this
missing submenu or function, it can be seen again. I think this breaks or needs changes
to too many things now, but maybe later?

Still one thing: moving with cursor keys in menus sucks. We really shouldn't be
able to go anywhere on the screen with cursor keys and pressing right arrow on
a submenu should open that menu and move us there, not go 10 pixels right. I
tried to fix this one a couple of times, but I don't understand events and X...
this one should be easy if you know what to do. I don't ;(

Hmm, still one thing. I think triangles in menus indicating a submenu should
go to left side, if we expand menu to the left, also
_________ ____________
|Dummy ||<| Sub Dummy|
| ||<| Other |
|________||____________|
          |Kill Me |
          |____________|

Well that's really enough.
