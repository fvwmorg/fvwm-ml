On Sun, Jul 04, 1999 at 04:46:48AM -0400, FoulDragon_at_aol.com wrote:
> I appreciate the fact that fvwm2 does windowshading.... but, as a matter of
> personal taste, I notice the shaded windows look odd ... They seem like they
> are just the title chopped off abruptly. In some cases, I'd think it might
> look better if it had the appropriate bevel at the bottom edge, giving it a
> 'complete' look.

That's because they are indeed chopped off at the bottom :-) Personally
I don't care how this loows, but the bottom border would consume space
unnecessarily. If someone makes a patch, fine, but I don't think one of
the maintainers will take action.

> In other thoughts, it might be neat to have an 'include' sort of thing for
> menus. Ideally, you could put the window list (to make a flwm style menu, or
> one like the GeoWorks express menu) or a submenu into a menu rather than
> making it an offshoot. Let me try to give an example:
> Menu A has items a, b, c
> Menu B has items d, e, include menu A, and f, so it displays as d,e,a,b,c,f.

Nice idea, but hard to implement. Perhaps in the release after 2.4, but I
can't make any promises.
 
> Also, are pinned menus planned?

Yes, but it is much more difficult than I expected first (the code isn't
designed to allow easy implementation of pinned menus).
