> > > Also I have Alt+F4 bound to "Close", this is intercepted ny the
> > > "surrounding" Windows window, and asks if I really want to kill the
> > > X-Server (I don't).
> > >
> > > Is there some way around this?
> >
> > Use another key?
>
> Well, I guess so...

I suspect that is the only way. Windoze will interpret AltF4 as
"close current window", and the application (Cygwin) will get a
Close event which will be indistinguishable from what it would
see if you had selected Close from the (Windoze) File menu.

It's conceivable that there's some registry hack to have Windoze
use something else instead of AltF4 -- if so, it might be mentioned
in some of the Cygwin or VNC docs/FAQs/whatever -- or you could put
in a request to the Cygwin developers for Cygwin to

* automatically respond "no", instead of asking whether to kill the
  X-Server, and

* Pass the Close event to the X-server as an Alt+F4.

Either way, there's not much FVWM can do about it :(
