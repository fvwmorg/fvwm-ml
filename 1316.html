Hello,

   I'm running FVWM-2.3.21 under RedHat Linux 6.0.

   I have some slight problems with the 'Resize' command.

   ----------------------------------------------------------------------------

   I'm attempting to resize a terminal window (xterm or rxvt)
   by a single character, by moving its bottom right corner.

   The two commands:

      Resize bottomright w+1c keep

      Resize bottomright keep w-1c

   both work correctly, making the window one character wider
   and one character shallower, respectively.

   However, the command:

      Resize bottomright w-1c keep

   makes the window *two* characters narrower, rather than just one.
   Furthermore, the command:

      Resize bottomright keep w+1c

   has no effect on the window, instead of making it one character deeper.

   ----------------------------------------------------------------------------

   Some other problems arise when attempting to resize windows created by
   "pixel-oriented" programs such as xdvi, gv, or ee (Electric Eyes).

   In these cases:

      Resize bottomright w+1c keep makes the window 10 pixels wider
      Resize bottomright w-1c keep makes the window 11 pixels narrower
      Resize bottomright keep w+1c makes the window 7 pixels deeper
      Resize bottomright keep w-1c makes the window 8 pixels shallower

   From the FVWM documentation, I would have expected that here, '1c'
   should mean 1 pixel. Or at any rate, that all four resize commands
   should change the window size by the same amount in each case.

   ----------------------------------------------------------------------------

   I wonder if my understanding of what's supposed to happen is incorrect,
   or if there's a bug in the code for resizing?

   I would be grateful if one of the developers could take a look at this.

Thanks! -- Joseph
