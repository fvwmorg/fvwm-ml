If a module does not respond anymore, fvwm freezes for up to 30
seconds. The same might happen if an application grabs the X
server and does not release it (try killing off all applications
from the console). Other than that, all hangs/crashes are bugs.

If there is a crash, can you look for the core file and fetch a
stack trace from it:

  $ gdb fvwm core
  (gdb) where

and post the output to fvwm-workers_at_fvwm.org (we need to know the
fvwm version too).
