>>>>> "MA" == Mohit Aron <aron_at_cs.rice.edu> writes:

MA> I tried compiling fvwm-2.2 and noticed that it still produces the
MA> binary named fvwm2.

MA> The name was understandable while version 2 was still in beta, but now
MA> that an official version has been released why not revert back to the
MA> good old "fvwm" ?

Because lots of places have users still using fvwm1 and expect that when
they put "fvwm" in a script, they get something that works with all of
their old config files. Since fvwm2 is incompatible with the old syntax,
it must have a different name.

We (the fvwm-workers) will probably at some point introduce a configure
flag to install it as just "fvwm".
