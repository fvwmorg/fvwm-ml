Hi Chuck,

Thanks for the modified xcolormap. It works like a charm except for when
the colourmap is about full, at which time it dumps with the message:

X Error of failed request: BadValue (integer parameter out of range for operation)
  Major opcode of failed request: 86 (X_AllocColorCells)
  Value in failed request: 0x0
  Serial number of failed request: 5520
  Current serial number in output stream: 5520

when one pushes the left mouse button.

Using this program, I was able to track down my colour leaks though.
After startup I have about 203 colours free. Instantiating netscape
I have about 130 free. Destroying netscape restores my count to 203. The
same happens for various other programs. Ghostview, xview etc. etc.
I can run a program, and after exit my colourmap is back to 203.

BUT, this does not happen with the latest release of xdvi. After I
run it, and exit, the colourmap program reports about 120 colours free.
And they don't come back.

My question is then this:

[1] Whose responsibility is it to retrieve those colours?
                         ie.
[2] Is it xdvi's fault, or fvwm's fault?
