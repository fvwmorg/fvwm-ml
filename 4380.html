On Tue, Mar 05, 2002 at 11:10:55PM +0100, Jerome Breviere wrote:
> I had the same problem some month ago when upgrading from 2.3.16 to 2.4.x.
> I finally found out that 2.4.x came with new placement policy options.
> Everything went back to normal with :
>
> Style "XMMS*" UsePPosition

The 'new' options are rather the old ones merged into a single
line of options (instead of the various combinations you could
have before). Little new functionality was added. Anyway, it is
definitely xmms that is or was doing funny things old versions
moved their own windows instead of providing the proper placement
hints to fvwm. This circumvented the NoPPosition option. Also,
xmms 1.24 moved the window to strange and random positions. So in
the end, fvwm had nothing to do with the problem. The xmms
ChangeLog sais something about having fixed placement of xmms
windows with a title under certain window managers (1.25). I
tried the latest release and it behaves as it should.

> ( i use : Style "*" NoPPosition in my general desktop entry)
