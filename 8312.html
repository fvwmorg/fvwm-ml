Hi there,

this morning I experienced a problem using the following
key binding with version 2.5.6

DestroyFunc FocusRaise
AddToFunc FocusRaise I Raise
+ I Focus
+ I Iconify off

Key Tab A 4 Prev ("CurrentPage" "!Iconic" !"Fvwm*") FocusRaise

In version 2.5.3 and previous this keybinding circled
between _all_ windows on the current page starting with the
previous one.

In version 2.5.6 the keybinding toogles between the last
(prev) and the actual window. It seems that the order of
the WindowList is effected by the function above, which
should not be happen since I used "Focus" instead of
"FlipFocus".

Any ideas?
