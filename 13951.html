I've just upgraded my primary desktop system from an HP-UX workstation
running fvwm 2.4.18 to a SuSE 9.2 system running fvwm 2.5.10. Along the
way, the default behavior of Resize seems to have changed.

Using the binding:

  Mouse 3 TFS A Resize

I'm accostomed to being able to press-and-hold mouse button 3 to initiate
a resize, dragging out to the new size, and then releasing button 3 to
complete the operation.

Under 2.5.10, I must press-and-release button 3, drag to the new window
border, and then press-and-release button 1 to confirm the operation.
I frequently find myself trying to confirm the resize with button 3, which
maddeningly cancels the resize.

I'm trying to train myself to expect the new behavior, but seven years
of "finger memory" is difficult to overcome. Is it possible to configure
version 2.5 for the old behavior?


TIA!
