>>>>> "BM" == Brady Montz <bradym_at_cs.arizona.edu> writes:

BM> Modules can send arbitrary command to fvwm. I think that it would be
BM> nice if fvwm could send commands to modules. Perhaps something like:

BM> Send "FvwmModule" "command string"

BM> which would send the string "command string" to all instances of
BM> FvwmModule which are currently running. And then fvwm could send a
BM> packet to the appropriate module which contained the text.
....
BM> Any thoughts?

I've been thinking about ways to add to & improve the module
interface, and this seems like it might be a good addition. I've
added this idea to the TO-DO list.

I thought it would be useful to be able to send messages to a running
module to implement things like new & different menu styles in a
module (for those people who wanted different menu styles, like say
Pie menus) without bloating fvwm. And as you (and others) started
pointing out, there could be other potential uses.
