"David M. Ronis" <ronis_at_onsager.chem.mcgill.ca> writes:
> I'm using 2.0.46 under linux 2.0.31 with XR6.3. Things work as
> expected, but I've come across the following problem: I sometimes use
> the IRC client BitchX (sorry but I'm not responsible for the name). It
> comes with an icon that takes up most of my default colormap. This is
> no problem for applications that use their own ones, but is a problem
> for those that don't. So far this is as expected and isn't an fvwm
> issue. The problem is that even after BitchX finishes, the WM default
> color map is still messed up. The only thing I can do is to restart
> fvwm (from within X though). Is there some way to refresh/reinitialize
> the colormap?

Currently, fvwm2 never frees the colors used in icons. I've done some
experimenting, and so far, the technique to free the colors used by
the Xpm library eludes me.

However, I did write a patch to fvwm2 a while back that limits the
colors used in icons. This patch is supposed to appear in the next
version of fvwm2. Let me know if you want it before then.
