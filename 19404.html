Hi,

Yesterday I stumbled accross a problem with FVWM. My first thought was
that I make a mistake in my application but now I see that FVWM just
handles modal dialogs differently than mwm and dtwm.

I got three windows:

        -------------------------------------------
        ! Main Window !
        ! !
        ! ------------------------------------- !
        ! ! Dialog Window ! !
        ! ! ------------------------------ ! !
        ! ! ! Sub Dialog Window ! ! !
        ! ! ! ! ! !
        ! ! ! ! ! !
        ! ! ! ! ! !
        ! ! ------------------------------ ! !
        ! !-----------------------------------! !
        -------------------------------------------

I have the window hierarchy constructed like this:

       Main Window -> Dialog Window -> Sub Dialog Window

All Dialog Windows have XmDIALOG_FULL_APPLICATION_MODAL set.

Now what I get with FVWM 2.0.44 is this: When all Windows are managed and
I click on the Main Window, the "Sub Dialog" window disappears. However
the "Dialog Window" stays on top.

This is *NOT* how it is supposed to work. Both dialog windows should
stay on top when I click on the Main Window. This works with both, Mwm
and Dtwm and I consider this a bug in FVWM. Or is there some fvwm2 setting
which can make it resemble the behaviour of Mwm in this respect.

Thanks a lot -- Tom
