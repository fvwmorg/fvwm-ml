Someone recently pointed out a bug to me, and I wanted to make sure
that it gets fixed.

Fvwm compares the return value from XGetWMName and XGetWMIconName to
the define "Success",ie if(XGetWMName(...) != Success)return;

We should compare to zero, according to the man page.

The above comparison occurs several times, just getp XGetWM in the
source to find it.
