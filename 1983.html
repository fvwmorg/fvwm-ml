> Can someone tell me how to remap the Alt-tab keys functionality to a new set
> of keys? I have tried to change it through the system.fvwm2rc file, but it
> does not seem to have an affect ( I don't have a personal .fvwm2rc file in
> my home directory). I need to do this so fvwm doesn't intercept the signal
> before it gets to my ICA client running a windows NT profile. So that I can
> tab between applications in the ICA client and use a different key mapping
> to tab between applications on the X desktop. Thanks.

You should mention the release you are using. I'm going to assume
its a current beta.

You can remove the built in binding like this:

Key Tab A M -

You can assign the window list to Control Tab like this:

Key Tab A C WindowList Root c c NoDeskSort

Unfortunately control-tab doesn't seem to be a good key to use
with the SelectOnRelease feature in the current betas. Maybe someone
else has a better key to use.
