Dominik Vogt wrote:
> In any case, a transient being closed should transfer the focus back
> to its 'parent'.

Only if you want it to. Could we make this a style option?

> Whit mouse/sloppy focus this can't happen. If the pointer is over some
> different window than the transient with the focus this must be because
> the focus was grabbed by the transient. This can only happen with the
> focus on its 'parent' (at least with yesterday's patch). In this case
> the user will most likely expect to get the focus back where it was.

In mousefocus mode you can still change focus without using the mouse.
I use the Direction function to do this and you don't have to warp the
mouse. So it is possible to have the mouse over one window with the
focus on another (possibly transient) window. In this case closing the
focus window should switch back to the mouse-containing window
(probably).
