The more correct name would be fvwm-beta, the package name is fvwm.

> my fvwm2rc to the new syntax. All the errors were dumped to STDERR, so it
> was easy to catch, except for the problems because of the new way panels
> are done in FvwmButtons, but that wasn't too hard. One thing, though, was

I can't reproduce any problem by using StaysPut (this is a default)
or StaysOnTop, StaysOnBottom in the current CVS version.
If you execute Style Stays* commands dynamically (say in FvwmConsole) you
should also execute Recapture, since they only affect an initial placement.

If you think there are problems, post here (or to fvwm-workers_at_ list)
the output of: egrep 'Stays|Layer' .fvwm2rc. Try to use the daily snapshot.
