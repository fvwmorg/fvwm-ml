Hi All,

I'm trying to configure the development brach of fvwm-2.5.6
under the gnome2/nautilus environment provided by Debian/unstable
(the fvwm package by debian maintainer A. Kotelnikov).

All work nicely but I have problem with associating of some
actions. I'm accustomed to use of the root window menu on X window
root and mouse strokes by this way:

Mouse 1 R A Menu "LongRootMenu" Nop
Mouse 3 R 2
Stroke N654 3 R 2 Desk -1 0
Stroke N456 3 R 2 Desk +1 0

The presented functions works nicely in plain Xwindow envirnoment
as well as in gnome2 without nautilus running. The configuration
doesn't work when nautilus manages of Xroot window (It show some
beautiful rectangles and nautilus menu, but it isn't what I want).

There is a way to not propagate events from fvwm to nautilus or to
mask the xroot nautilus window for fvwm as a plain xroot window?
Is this question for fvwm comunity people or for nautilus developers?
(Or there is a another file manager for gnome2 with support of icons
on the desktop? :-))
