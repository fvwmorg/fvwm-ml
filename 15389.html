        I had the same problem with the Icon Manager at first too. It did the
exact same thing: It ran in the background but didn't appear on the
screen. To fix it I had to install a later version of FVWM. You have to be
running FVWM v1.24r or later.(I was using 1.21c). You also have to properly
indicate/make the config.h file. (Examples are in the config directory that
come with the .tar) If you compile on an SGI (I did) it doesn't come with a
release but, I have a config.h file that worked (for me) under IRIX 5.3.
If you want it, e-mail me and I'll post it OR e-mail Brady Montz at:
  bradym_at_cs.arizona.edu

He was very helpful when I had similar problems. Since I got it working
on the SGI I have compiled it under Linux on my Dad's PC for home use and
had no problems there either.

        Jon Watson
PS - I noticed I was kinda vague in my last message about the Icon
manager and if it supported the multiple window lists. Yes it does support
multiple managers showing specific things, (ie. 1 manager shows emacs, 1
shows xterms) but I don't use those features. There is an example of how
to do that in the documentation tho.
