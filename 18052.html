Me> Um... it IS drawn with XOR. You may have to play with XORValue. From
Me> the man page:

Me> XORvalue number
Me> Changes the value with which bits are XOR'ed when doing
Me> rubber-band window moving or resizing. Setting this
Me> value is a trial-and-error process.

Me> Try 1 (which I use with a black background), then 255 (the default
Me> actually), as I think these are the best bets (unless you have a
Me> TrueColor/DirectColor visual _at_ 24 bpp, in which case try 250, which
Me> worked for someone I know).

I decided to change the default to 1 (which I was setting in my
..fvwm2rc anyways) since it does seem to work better for darker
backgrounds. That'll be in the next beta.
