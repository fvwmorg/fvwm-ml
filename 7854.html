On 12 Feb 2003 18:56:20 +0100, Markus Grunwald wrote:
>
> I use "TitleStyle RightJustified" for my aterm windows. I set the
> window title to the actual directory, so that I have more room in the
> window.
>
> Now when I have really long paths and thus really long window titles,
> the end of the title is cut off in spite of the "TitleStyle". I thought,
> in the RightJustified mode, text would be cut off on the left side...
>
> Am I doing something wrong or is this fvwms fault ?

None of these. It always cuts from the end of the string. And this is
usually ok, think about long browser titles.

It would be nice to have a configurable title format (filter) with a
powerful string manipulation on per window match, but it is not currently
available. Maybe someone (me?) will implement this after 2.6.

I suggest you to solve this with your shell. For example, with tcsh the
string "%C05" includes only 5 tail subdirectory levels in the prompt.
There are other solutions to keep CWD short in tcsh and other shells.
