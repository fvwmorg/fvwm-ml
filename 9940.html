On Wed, 10 Sep 2003 18:25:59 +0200
Dominik Vogt <fvwm_at_fvwm.org> wrote:

> The usual way to emulate this is to use FvwmButtons as the base
> container, define a button for the menu and swallow FvwmIconMan
> for the application button. Any number of additional buttons and
> swallowed apps can be defined easily.
>
> I'm not sure whether a config for this fairly complex setup is
> floating around here.

#
# Taskbar
#
KillModule FvwmButtonsTaskbr
DestroyModuleConfig FvwmButtonsTaskbr: *
*FvwmButtonsTaskbr: Frame 0
*FvwmButtonsTaskbr: Colorset 6
*FvwmButtonsTaskbr: Geometry 1152x29+0-0
*FvwmButtonsTaskbr: Columns 24
*FvwmButtonsTaskbr: BoxSize Fixed
*FvwmButtonsTaskbr: Frame 1

*FvwmButtonsTaskbr: (Title Start, Action "Menu MyMainMenu \
Rectangle +$left+$top 0 -100m")
*FvwmButtonsTaskbr: (18x1,Swallo (UseOld) "FvwmIconMan" \
"Module FvwmIconMan")
*FvwmButtonsTaskbr: (Swallow (UseOld) "xeyes" "Exec exec xeyes")
*FvwmButtonsTaskbr: (4x1,Swallow (UseOld) "FvwmPager" \
"Module FvwmPager 0 3")

Be carefull with the "\", and set the "Geometry" and "Columns"-Options
to your needs.
