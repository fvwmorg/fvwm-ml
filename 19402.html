Choksing Taweponsomkiat <ct_at_ccoast.com> writes:

> Hi,
>
> I have 2 questions
>
> 1) How do I configure FvwmPager to display both mini-icons and small
> font title? It seems to be able to do just either one of them.
>
> 2) I have FvwmIconMan 0.9.7. (If it possible, )How do I configure it to
> have one manager manage emacs windows of that page and another manager
> manage emacs windows in all other pages/desks?
>

Not sure exactly what you want. Do you want one manager to show the emacs
windows on the current page, and the second to show all the rest, and as you
switch pages they get updated correspondingly? It can't do that. That would be
cool though. There's a simple change that would make that work, but I'll have
to think about the ramifications of it.

If you have one special page that contains emacs windows that need their own
manager, and this page doesn't change, that would work.

Simplest way is if you could have the first emacs set the icon string to
contain some special string, and then have a manager set to catch all windows
whose icon names contain that string. You can then set the format option to
something like "%t" so it displays the title string and not the icon. Since
these emacs windows seem special (or why else have them in their own manager"
it might work.

Example, make those emacs windows' icon names contain "bubba" and use this:

*FvwmIconMan*1*format "%t"
*FvwmIconMan*1*resolution global
*FvwmIconMan*1*show icon=*bubba*

*FvwmIconMan*2*format "%t"
*FvwmIconMan*2*resolution global
*FvwmIconMan*2*show resource=emacs resource=gemacs


Manager 1 catches the special emacs's and #2 gets all the rest.
