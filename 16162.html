Dan> You don't seem to mention more built-in defaults, but they are
Dan> there.

I didn't mention them yet because the real reason I added them was in
an effort to get it to be able to run without an rc file, for those
people who forget to copy the rc file into their home directory. The
additions were not enough (it still locks up), and I didn't have more
time to investigate it further.

Perhaps better (or rather, easier) would be to make it yell and scream
a lot if it can't find the file it's trying to read, which it
currently doesn't (actually, it should probably do that regardless).

Dan> The default style for "color" produces icon titles that are unreadable
Dan> due to lack of contrast:

Really? It's not much contrast, but still very readable for me.

Dan> fvwm.c line 347
Dan> ExecuteFunction("Style \"*\" Color black/dimgrey, Title, RandomPlacement",

Dan> I think the "dimgrey" is a good choice, it certainly suggests a
Dan> non-selected window to me.

I thought so.

Dan> May I suggest "white/dimgrey" to get decent contrast.

That certainly does give better contrast, but I don't think I like it
as much. The 'white' is too much contrast. Perhaps 'wheat' or
'lightgrey' for the foreground would be better. How do these look on
your system (particularily the latter, which I like only slightly less
than the black).
