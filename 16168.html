At the moment, fvwm reports errors by calling fvwm_err(). This put the
message out on stderr, where it will usually get lost lying in
~/.xsession-errors.

How about sending the text of the error message to an
'FvwmErrorDelivery' module, which could pop up a window containing the
text. Alternatively, if an FvwmConsole is open, then the errors should
probably go there.

What do people think ?
