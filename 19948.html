Hi,
        I have a major problem here. One of our people has written an
applications which takes image frames from a camera and displays them.
For displaying the images, it has to create a colormap of it's own,
as the default colormap doesn't have enough space left - normally.

        The problem I am facing is, that fvwm is not able to switch
to the private colormap - try what I may. I have tried by changing
ColormapFocus and ClickToFocus etc. but to no avail. Strangely, this
problem does not occur in other applications that we have written.
By the way, we have also installed an event handler inside the applications
that installs the private colormap when the mouse enters the window.
But the moment it installs the colormap, somebody immediately installs
back the default one :(

        Please throw light. We are using fvwm2.0.45 on a Sun Ultra
running SunOS 5.5.1

thanks,
sumeet
