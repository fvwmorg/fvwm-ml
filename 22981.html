I have recently tried to start using FvwmM4 as a preprocessor for my
rc file, and have had some problems.

First though, the reason I opted for M4 (instead of Cpp) is that I
need multi-line macro definitions. I don't think this is possible
with Cpp, but please correct me if I am wrong...

Now, if I tell my FvwmConsole to

        FvwmM4 fvwm2rc.m4

it will happily parse the file correctly, and add all the relevant
definitions to the current fvwm2 state. However, when I try to either
start or Restart with

        fvwm2 -cmd "FvwmM4 fvwm2rc.m4"

I get

        [FVWM][Done]: <<ERROR>> Call of 'fvwm2 -cmd "FvwmM4 FVWMRC.m4"'
        failed!!!! (restarting 'fvwm2' instead)

and I cannot figure out why. I am using 2.0.46, and I have been
unable to find anything wrong with m4 or the rc file which might be
the problem. Any ideas why the -cmd invocation bombs?

Thanks,

CJW
